### 1. `Schema.Mock` — JSON Schema/OpenAPI 명세로 가짜 데이터 즉시 생성기

**카테고리**: Dev/Tools · 오프라인 데스크탑
**핵심 기능**: JSON Schema 또는 OpenAPI 3.x 명세를 붙여넣으면 규칙에 맞는 가짜(Faker) 데이터를 즉시 생성해 JSON/CSV로 내보낸다. 프론트엔드 개발·API 테스트·목업 데이터 준비에서 반복적으로 겪는 "테스트 데이터 수작업 입력" 문제를 해결한다.

**상세 기능**:
- JSON Schema Draft 7/2019-09/2020-12 파싱 및 완전 지원
- OpenAPI 3.x `components/schemas` 전체 순회·자동 참조 해결
- 필드별 Faker 전략 선택: `string→name/email/lorem/uuid`, `integer→range`, `enum→random` 등
- 생성 건수 지정 (1~10,000건), JSON Array / NDJSON / CSV 출력
- 중첩 객체·배열·`$ref` 재귀 생성 (최대 깊이 제한 설정 가능)
- `format: date-time/email/uri/uuid` 표준 준수 값 자동 생성
- 필드별 수동 오버라이드 (특정 필드를 고정값·범위·정규식 패턴으로 지정)
- 생성된 데이터를 클립보드 복사 / 파일 저장 (UTF-8 BOM)
- 스키마 파일 드래그&드롭 지원

**기술 스택**:
- .NET 10 / WPF / C# 12
- `System.Text.Json` — JSON Schema 파싱·검증
- `NJsonSchema` (NuGet) — JSON Schema 로딩
- `Bogus` (NuGet) — Faker 데이터 생성 라이브러리
- 자체 OpenAPI 3.x `$ref` 해결기

**수익 모델**: 완전 무료 오픈소스 (개발자 생산성 도구)
**차별화**: Api.Probe(수동 요청), Schema.View(다이어그램), Schema.Diff(두 버전 비교)와 모두 다름 — Schema.Mock은 명세→데이터 **생성** 방향에 특화.

### 2. `Hook.Lab` — Webhook 수신 로컬 터널 & 실시간 페이로드 인스펙터

**카테고리**: Dev/Network · 오프라인+ngrok 대체
**핵심 기능**: GitHub/GitLab/Slack/Stripe 등 외부 서비스가 보내는 Webhook을 로컬 포트에서 수신하고, 페이로드를 실시간으로 파싱·검사·재전송할 수 있는 GUI 도구. ngrok 의존 없이 로컬 터널(SSH 리버스 포워딩 또는 cloudflared) 설정을 원클릭으로 처리한다.

**상세 기능**:
- 로컬 HTTP 리스너 포트 설정 + 수신 요청 즉시 트리 뷰 표시
- cloudflared/SSH 터널 원클릭 연결 (공개 HTTPS URL 자동 획득)
- 수신된 모든 요청 히스토리 (헤더·바디·타임스탬프 상세)
- JSON 바디 → 구문 강조 트리 뷰 자동 렌더링
- HMAC-SHA256 시그니처 검증 (GitHub/Stripe/Slack 규칙 지원)
- 저장된 요청을 로컬 엔드포인트 또는 지정 URL로 재전송 (replay)
- 수신 요청 조건 필터 (헤더 값, JSON 경로, 이벤트 타입)
- 설정된 시나리오 자동 응답 (200/400/500 코드 + 커스텀 바디)

**기술 스택**:
- .NET 10 / WPF
- `System.Net.HttpListener` — 로컬 HTTP 서버
- `cloudflared.exe` 번들 (프리번들 없이 다운로드 감지) 또는 SSH 리버스 터널
- `System.Text.Json` — JSON 파싱
- `System.Security.Cryptography.HMACSHA256` — 시그니처 검증

**수익 모델**: 무료 (개발자 개인 도구)
**차별화**: Mock.Server(응답 정의 중심) ≠ Hook.Lab(수신·인스펙션·재전송 중심). Api.Probe(요청 송신 도구) 와 반대 방향.

### 3. `Dep.Graph` — 프로젝트 의존성 버전 충돌 그래프 시각화기

**카테고리**: Dev/Analyzer · 로컬 파일 분석
**핵심 기능**: NuGet(.csproj), npm(package.json), pip(requirements.txt/pyproject.toml) 파일을 읽어 전이 의존성 전체를 방향 그래프로 시각화하고, 버전 충돌·다이아몬드 의존성·취약 버전을 한눈에 표시한다. 복잡한 모노레포나 마이크로서비스 환경에서 "왜 이 버전이 설치됐는지" 추적하는 시간을 절약한다.

**상세 기능**:
- NuGet/npm/pip 세 패키지 시스템 지원 (`.csproj`, `packages.lock.json`, `package-lock.json`, `poetry.lock`)
- D3 Force-directed 방향 그래프 (WebView2 렌더링)
- 충돌 노드 자동 강조: 같은 패키지가 다른 버전으로 요구될 때 빨간색 표시
- 경로 추적: 특정 패키지가 왜 포함됐는지 루트→리프 경로 하이라이트
- 취약 버전 경고 (OSV 오프라인 데이터베이스 또는 HTTP 조회)
- 버전 고정 제안 (충돌 해결 권장 버전 계산)
- 의존성 트리 텍스트 내보내기 + PNG/SVG 그래프 내보내기
- 필터: 직접 의존성만 / 전이 포함 / 특정 패키지 서브트리

**기술 스택**:
- .NET 10 / WPF
- `WebView2` — D3.js 그래프 렌더링
- `System.Text.Json` / XML 파서 — 패키지 파일 파싱
- `OSV API` (선택적 HTTP) — 취약점 데이터
- 자체 위상 정렬 + 충돌 감지 알고리즘

**수익 모델**: 무료 오픈소스
**차별화**: Git.Stats(커밋 통계), Schema.Diff(스키마 버전 비교) 와 다름 — 런타임 패키지 의존성 그래프에 특화.

### 4. `VM.Cast` — Hyper-V / WSL2 / VirtualBox 인스턴스 트레이 관리자

**카테고리**: System/Virtualization · 트레이 앱
**핵심 기능**: Windows Hyper-V, WSL2, VirtualBox VM 인스턴스를 하나의 트레이 메뉴에서 일괄 관리한다. 현재 실행 중인 VM 목록, 원클릭 시작/중지/스냅샷, CPU/메모리 사용 현황을 시스템 트레이에서 즉시 확인할 수 있다.

**상세 기능**:
- Hyper-V (`Get-VM` PowerShell), WSL2 (`wsl --list`), VirtualBox (`VBoxManage`) 통합 감지
- 트레이 메뉴: VM 목록 + 상태 (실행중/중지/일시정지) + 원클릭 Start/Stop/Pause
- 스냅샷 생성·복원 (Hyper-V 체크포인트, VirtualBox 스냅샷)
- VM별 CPU/메모리 사용량 실시간 표시
- 빠른 접속: RDP(Hyper-V), SSH(WSL2/원격), VirtualBox GUI 직접 열기
- 자동 시작 설정 (부팅 시 특정 VM 자동 시작)
- 실행 시 풍선 알림 (시작/중지 이벤트)
- 설정 창 중복 방지 (이미 열려있으면 Activate)

**기술 스택**:
- .NET 10 / WinForms (트레이 앱)
- `System.Management` — WMI Hyper-V 쿼리
- PowerShell Runspace (`Hyper-V` 모듈 cmdlet)
- WSL2: `wsl.exe` 프로세스 래핑
- VirtualBox: `VBoxManage.exe` COM API

**수익 모델**: 무료
**차별화**: App.Temp(샌드박스 변경 롤백) ≠ Vhd.Cast(VHD/VHDX 마운트) — VM.Cast는 가상머신 라이프사이클 관리에 특화.

### 6. `Ping.Cast` — 다중 호스트 ICMP 핑 연속 모니터 + 응답시간 히스토리 트레이

**카테고리**: Network · 트레이 앱
**핵심 기능**: 사용자 정의 호스트 목록을 지속적으로 ICMP 핑하고, 응답시간을 실시간 히스토리 그래프로 시각화하는 트레이 앱. 인터넷 연결 품질, 회사 내부 서버 응답성, 게임 서버 핑을 24시간 추적한다.

**상세 기능**:
- 호스트 목록 관리 (이름·IP/도메인·태그·그룹)
- 1초~60초 간격 설정 가능한 ICMP 핑 (TTL, 패킷 크기 설정)
- 실시간 스파크라인 그래프 (최근 60회 응답시간)
- 이동평균·최솟값·최댓값·패킷 손실률 표시
- 임계값 초과 시 Toast 알림 (응답시간 > N ms, 패킷 손실 > N%)
- 수동 트레이스라운트 보조 (특정 호스트 tracert 원클릭 실행)
- 24시간 히스토리 SQLite 저장 + CSV 내보내기
- DarkMenuRenderer 트레이 컨텍스트 메뉴

**기술 스택**:
- .NET 10 / WinForms (트레이)
- `System.Net.NetworkInformation.Ping` — ICMP
- `SQLite-net` (NuGet) — 히스토리 저장
- WinForms `UserControl` 커스텀 스파크라인 그래프

**수익 모델**: 무료
**차별화**: Ping.Map(CDN 세계지도 히트맵·1회성) ≠ Uptime.Eye(HTTP 폴링) ≠ Net.Scan(LAN ARP 스캔) — Ping.Cast는 **사용자 정의 호스트 연속 핑 히스토리** 전용.

### 9. `Proc.Bench` — CPU / 메모리 / 스토리지 종합 벤치마크 스위트

**카테고리**: System/Benchmark
**핵심 기능**: 순정 Windows 환경에서 실행 가능한 종합 성능 벤치마크. CPU 단일·다중 코어, 메모리 대역폭·레이턴시, 스토리지 순차·랜덤 I/O를 측정하고 결과를 이전 측정치·커뮤니티 통계와 비교한다.

**상세 기능**:
- CPU: 단일스레드(SHA-256, AES, 소수 판별), 멀티스레드(병렬 FFT, 행렬 곱)
- 메모리: 대역폭(STREAM 포트), 레이턴시(랜덤 접근 패턴), L1/L2/L3 캐시 구분
- 스토리지: 순차 읽기/쓰기, 4KB 랜덤 읽기/쓰기, IOPS, S.M.A.R.T 상태
- 테스트별 점수 + 그래프 (레이더 차트 + 바 차트)
- 결과 내보내기 (JSON + HTML 보고서)
- 과거 측정치 비교 (로컬 SQLite 히스토리)
- 커스텀 테스트 시간 설정 (빠른 30초 / 표준 2분 / 심화 10분)
- 하드웨어 정보 자동 수집 (WMI: CPU 모델, 코어 수, RAM, OS 버전)

- WPF VirtualizingStackPanel — 수만 줄 가상화 렌더링

**수익 모델**: 무료
**차별화**: Log.Lens ✅(정적 로그 파일 분석 뷰어) ≠ Log.Merge(다중 로그 병합) ≠ Win.Event(Windows EVTX) — Log.Struct는 **JSON 구조화 로그 라이브 스트리밍 + JQ 필터** 특화.

### 6. `Sand.Craft` — 모래성 쌓기 방어 게임 (조수 침식 vs 건축)

**장르**: Sandbox / Builder / Survival
**핵심 메커닉**: 밀물이 밀려올 때마다 조금씩 깎여나가는 모래성을 마우스 드래그 스컬팅으로 쌓고 강화한다. 물의 침식 물리와 구조 강도 시뮬레이션이 핵심으로, 어떤 형태의 성이 파도를 가장 잘 견디는지 실험하는 재미가 있다.

- 모래 입자 시뮬레이션: 쌓이면 경사각 초과 시 흘러내림(안식각)

- 샌드박스 자유 모드: 파도 높이·주기 설정, 재료 무제한
- 물리 시뮬: 입자 기반 물 + 모래 이산 시뮬레이션

**기술**: 입자 낙하 시뮬레이션 (Sand.Fall류), 간소화 SPH 물 시뮬레이션, 침식 모델 (파력 × 재료 강도), 2D 자유 스컬팅 (픽셀 쓰기), WPF WriteableBitmap

**매력**: 자유롭게 쌓은 것이 파도에 부서지는 시각적 쾌감. 최적 형태를 찾는 실험 과정의 탐구심. "이번에는 더 잘 버티겠지" 한 번 더를 유발하는 반복 매력.

**차별화**: Tide.Hunt(조수 간만 타이밍 탐험 어드벤처) ≠ Sand.Storm(모래폭풍 방벽 생존 탈출) ≠ Wobble.Stack(소프트바디 타워 쌓기) — Sand.Craft는 **침식 물리 vs 자유 스컬팅 모래성 건축** 샌드박스.

### 7. `Dream.Shift` — 꿈속 중력 방향 변환 플랫포머

**장르**: Platformer / Puzzle / Surreal
**핵심 메커닉**: 점프할 때마다 중력이 점프 방향으로 회전하는 꿈속 물리 플랫포머. 단순히 위아래로 뒤집히는 것이 아니라 8방향 중 원하는 방향으로 중력을 설정해 벽을 바닥으로, 천장을 발판으로 사용하며 퍼즐을 해결한다.

**게임플레이**:
- WASD 이동 + Space 점프 (점프 방향 = 새 중력 방향)
- 각 방에 별(수집 목표)과 출구가 있어 중력을 조작해 도달
- 환경 요소: 역중력 패드(특정 방향 강제), 중력 고정 구역, 방향 잠금 함정
- 모멘텀 보존: 이전 중력 방향 속도가 새 중력 전환 후에도 영향
- 시각 효과: 중력 전환 시 방 전체가 부드럽게 회전하는 연출
- 스테이지별 테마: 물 위 꿈(잔물결 배경), 미로 꿈(무한 복도), 추락 꿈(끝없는 하강)
- 타임 트라이얼: 같은 방을 최소 중력 전환 횟수로 클리어
- 드림 저널: 클리어한 방이 꿈 일기 형태로 기록

**기술**: 8방향 중력 벡터, 리지드바디 2D 물리 (각운동량 전환 계산), 부드러운 카메라 회전 (보간), 절차 생성 방 (씨드 기반)

**매력**: 방 전체가 회전하는 독특한 시각적 경험. 중력 방향을 예측하고 연속 전환하는 "중력 콤보"의 리듬감. 꿈의 초현실적 분위기가 이상한 물리를 자연스럽게 받아들이게 함.

**차별화**: Gravity.Flip ✅(이진 상하 반전) ≠ Rift.Jump(공간 균열 순간이동) ≠ Magnet.Jump(N/S 극성 반발) — Dream.Shift는 **점프 방향=중력 방향 회전** 8방향 연속 전환이 핵심 메커닉.

### 8. `Morse.Run` — 모스 부호 해독 + 러너 동시 조작 게임

**장르**: Rhythm / Educational / Runner
**핵심 메커닉**: 위에서 장애물이 내려오는 사이드스크롤 러너를 조종하면서 동시에 화면 한쪽에서 재생되는 모스 부호(소리+시각)를 해독해 정답을 입력한다. 두 작업을 동시에 처리하는 인지 분할 트레이닝이 핵심 재미다.

**게임플레이**:
- 왼쪽: 사이드스크롤 러너 (Space/Up 점프, Down 슬라이드)
- 오른쪽: 모스 부호 신호 재생 (점·선 시각화 + 효과음)
- 부호 해독 완료 시 알파벳 입력 → 정답 시 점수 + 보호막
- 모스 부호 속도 점진적 상승 (WPM 단위)
- 장애물 패턴: 낮은 벽(슬라이드), 구멍(점프), 이중 장애물(타이밍)
- 정답 연속 시 시간 슬로우 보너스 (러너 파트 쉬워짐)
- 모스 부호 학습 모드: 알파벳/숫자 26+10자 단계별 해금
- 스토리 모드: 수신된 모스 메시지가 2차대전 스파이 통신 내러티브

**기술**: 모스 부호 데이터 테이블 (국제 모스 코드 표준), 오디오 비프음 생성 (`System.Media.SoundPlayer`), 사이드스크롤 러너 (WPF Canvas 스프라이트 이동), 인지 부하 난이도 곡선

**매력**: 두 가지를 동시에 하는 멀티태스킹의 압박감이 독특한 재미. 모스 부호를 자연스럽게 익히는 교육적 효과. 해독 성공이 게임플레이에 직접 도움이 되는 보상 연결.

**차별화**: Rhythm.Type(가사를 박자에 맞춰 타이핑) ≠ Signal.Cast(시각+청각 시퀀스 재현) ≠ Glyph.Rush(글자 타이핑 슈터) — Morse.Run은 **모스 부호 해독 + 러너 동시 이중 조작** 전용 메커닉.

### 9. `Quill.Quest` — 붓 획 순서 캘리그래피 추적 기술 게임

**장르**: Skill / Educational / Artistic
**핵심 메커닉**: 고대 문자(한글 고체·한자·아랍 문자·데바나가리·갑골문)의 올바른 획 순서를 마우스/터치로 정확히 따라 그리는 기술 게임. 획 시작점·방향·속도·압력(마우스 속도)까지 평가해 정밀 점수를 산출한다.

**게임플레이**:
- 반투명 가이드 글자가 표시되고, 올바른 획 순서로 따라 그리기
- 각 획 시작점 오차 범위: ±5px (초급) ~ ±1px (고급)
- 방향 정확도: 획 경로가 가이드와 겹치는 비율로 채점 (0~100점)
- 속도 리듬: 빠른 획 / 느린 강세 / 연속 흘림체 각각 리듬 판정
- 연속 콤보 시 획이 황금빛으로 변하는 시각 피드백
- 5단계 난이도: 기본 한글 자모 → 복잡한 갑골문·아랍 이어쓰기
- 챌린지 모드: 화면 회전·미러 반전 상태로 획 그리기
- 작품 저장: 완성된 캘리그래피를 PNG 내보내기 (배경 한지 스킨 선택)

**기술**: 획 경로 유사도 계산 (DTW - Dynamic Time Warping), 베지어 곡선 근사, 마우스 속도→압력 변환, 문자 데이터 (SVG 경로 기반 획 정의)

**매력**: 디지털로 캘리그래피 감각을 익히는 독특한 경험. 완벽한 획이 완성됐을 때의 시각·청각 피드백이 주는 장인 만족감. 획 순서라는 숨겨진 규칙을 발견하는 탐구 재미.

**차별화**: Sym.Draw(4배 거울 대칭 드로잉 만다라) ≠ Shutter.Run(이동 궤적 장노출 드로잉) ≠ Pixel.Forge 앱(픽셀아트) — Quill.Quest는 **역사적 문자 획 순서 정밀 추적** 기술 게임.

### 10. `Bloom.Maze` — 살아서 자라나는 미로 생존 아케이드

**장르**: Arcade / Survival / Bio
**핵심 메커닉**: L-시스템으로 구현된 식물 덩굴이 실시간으로 자라면서 미로를 채워가는 가운데, 플레이어는 계속 좁아지는 통로를 이용해 목표 지점에 도달해야 한다. 정적인 미로가 아니라 매 초 변하는 생동감 있는 공간 속 생존 게임이다.

**게임플레이**:
- 50×50 격자 미로에서 시작, 사방에서 덩굴이 자라기 시작
- 덩굴: L-시스템 규칙에 따라 분기하며 격자를 채움 (1초마다 1~3칸 확장)
- 플레이어: WASD 이동, 덩굴이 있는 타일로는 진입 불가
- 목표: 맵 중앙의 씨앗을 수집 후 출구로 탈출 (시간이 갈수록 경로 소멸)
- 도구: 제초제 폭탄(반경 3×3 덩굴 제거), 성장 지연제(5초 일시 정지)
- 레벨별 식물 종류: 빠른 덩굴(단방향 선형), 나무(방사형 성장), 버섯(BFS 확산)
- 점수: 출구 도달 시 남은 공간 비율 × 수집한 씨앗 수
- 생존 기록 저장 (최대 생존 시간 히스토리)

**기술**: L-시스템 문자열 재작성 규칙 + 격자 적용, BFS 식물 확산 시뮬레이션, 절차 생성 시작 미로 (Hunt-and-Kill 알고리즘), WPF 스프라이트 애니메이션

**매력**: 미로가 실시간으로 변하는 긴박감이 끊임없이 유지됨. 안전한 경로가 사라지기 전에 이동해야 하는 타임 프레셔. 덩굴이 자라는 패턴을 예측해 선제적으로 이동하는 전략 재미.

**차별화**: Maze.Dread(정적 미로 + 시야 제한 생존) ≠ Maze.Craft(역발상 미로 설계 vs AI) ≠ Leaf.Grow ✅(L-시스템 성장 관찰 시뮬레이션) — Bloom.Maze는 **실시간 L-시스템 덩굴 성장이 미로를 채우는 동적 생존** 아케이드.
