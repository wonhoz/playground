### 1. `Folder.Watch` — 폴더 실시간 변경 감시 트레이 앱

**카테고리**: System / Tray / Developer
**핵심 기능**: FileSystemWatcher 기반으로 지정한 폴더의 파일 추가·수정·삭제를 실시간 감시하고 Windows Toast 알림으로 즉시 보고한다. Stay.Awake처럼 트레이에서 조용히 항상 구동되며, 개발·빌드·배포 폴더 감시에 최적화되어 있다.

**상세 기능**:
- FileSystemWatcher로 다중 폴더 동시 감시 (최대 20개 경로)
- 확장자 필터 (`.exe`, `.log`, `.json` 등 특정 확장자만)
- 이벤트 종류별 Toast 알림 (생성·수정·삭제·이름 변경)
- 이벤트 로그 뷰어 (타임스탬프·경로·이벤트 유형 기록)
- 정적 기간 무시 (N초 내 연속 이벤트 디바운스)
- 감시 프리셋 저장/불러오기 (프로젝트별 설정)
- 트레이 더블클릭 → 변경 로그 팝업

**기술 스택**:
- `System.IO.FileSystemWatcher` (변경 감시)
- `Microsoft.Toolkit.Uwp.Notifications` (Toast 알림)
- WinForms TrayIcon + ContextMenuStrip
- SQLite (이벤트 로그 영구 저장)
- JSON (감시 프리셋 설정 파일)

**수익 모델**: 무료 오픈소스. 확장 기능(Webhook 연동, 이메일 알림) 프리미엄
**차별화**: Diff.Snap(스냅샷 타임머신)은 사후 비교 도구. Dupe.Guard(중복 감시)는 파일 내용 해시 비교. Folder.Watch는 변경 즉시 알림에 집중한 트레이 상주 개발자 도우미

### 4. `Reg.Watch` — 레지스트리 변경 실시간 감시 트레이

**카테고리**: System / Security / Tray
**핵심 기능**: Windows 레지스트리 특정 키·하이브를 실시간 감시하고, 소프트웨어 설치·삭제·설정 변경을 Toast 알림으로 즉시 감지한다. 개발 중인 앱의 레지스트리 기록 추적이나 시스템 보안 모니터링에 필수적이다.

**상세 기능**:
- `RegNotifyChangeKeyValue` Win32 API로 키 변경 이벤트 구독
- 감시할 키 북마크 목록 관리 (HKLM·HKCU·HKCR 전체 지원)
- 소프트웨어 설치 감지 전용 모드 (`HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall` 감시)
- 변경 전후 값 비교 스냅샷 (Diff 표시)
- 이벤트 로그 (타임스탬프·키 경로·변경 유형·이전값·이후값)
- 신뢰하지 않는 프로세스가 특정 키 변경 시 경고 알림
- 트레이 우클릭 → 최근 변경 10건 즉시 확인

**기술 스택**:
- `RegNotifyChangeKeyValue` + `RegistryKey` Win32/P/Invoke (변경 감지)
- `Microsoft.Win32.Registry` (.NET 내장)
- Background thread polling + SemaphoreSlim
- WinForms TrayIcon + DataGridView (로그 뷰어)
- SQLite (변경 로그 영구 저장)

**수익 모델**: 무료. 보안 탐지 규칙 커스텀 팩 유료
**차별화**: Reg.Vault(레지스트리 브라우저·비교기)는 수동 탐색 도구. Reg.Watch는 실시간 알림에 특화된 트레이 수비수

### 7. `Pipe.View` — Windows Named Pipe IPC 실시간 인스펙터

**카테고리**: Dev / Tools / Debugging
**핵심 기능**: Windows Named Pipe와 로컬 소켓 IPC 채널을 실시간으로 감시·캡처·디코딩하는 개발자 디버깅 도구. 프로세스 간 통신을 Api.Probe처럼 시각적으로 분석할 수 있어 내부 컴포넌트 통신 디버깅에 필수적이다.

**상세 기능**:
- 시스템의 모든 Named Pipe 목록 자동 탐색 (`\\.\pipe\*`)
- 선택한 파이프의 메시지 실시간 캡처 (미러 모드)
- 바이트·HEX·텍스트·JSON·XML 자동 디코딩 뷰
- 파이프 메타데이터 (연결된 프로세스 PID·이름, 접근 권한, 최대 인스턴스 수)
- 메시지 타임라인 뷰 (요청·응답 페어 매칭)
- 캡처 데이터 PCAP 스타일 내보내기
- 테스트 메시지 주입 (파이프에 직접 데이터 전송)
- Named Pipe 서버 생성 (간단한 에코 서버 내장 테스터)

**기술 스택**:
- `System.IO.Pipes.NamedPipeClientStream` / `NamedPipeServerStream`
- `CreateFileW` + `DeviceIoControl` Win32 P/Invoke (파이프 열거)
- `NtQuerySystemInformation` (파이프 연결 프로세스 조회)
- WPF + HEX 뷰어 커스텀 컨트롤
- Newtonsoft.Json / System.Xml (메시지 자동 디코딩)

**수익 모델**: 무료 오픈소스. 팀 라이선스(원격 파이프 감시) 유료
**차별화**: Api.Probe(HTTP REST API 테스터), Serial.Cast(UART 시리얼), Hex.Peek(파일 HEX 뷰어). Pipe.View는 프로세스 간 Named Pipe IPC에 특화된 유일한 도구

**상세 기능**:

- 변환 목표 설정: 탭→스페이스(N칸) 또는 스페이스→탭
- 파일 형식별 규칙 (.py는 스페이스 4, .go는 탭 등)
- `.editorconfig` 파일 자동 읽기 및 생성
- 미리보기 Diff 창 (변환 전후 비교)
- 일괄 변환 + 원본 백업
- 혼용 파일 우선 처리 모드 (정리가 급한 파일 강조)
- Git 친화적 모드: 공백 변경을 단일 커밋으로 제안

**기술 스택**:
- 커스텀 텍스트 파싱 (행별 선행 공백 분석)
- `.editorconfig` 파서 (EditorConfig NuGet)
- WPF DataGrid + DiffPlex (Before/After 뷰)
- `System.IO.FileSystemWatcher` (편집 중 실시간 감지)
- Glob 패턴 파일 필터 (`.gitignore` 호환)

**수익 모델**: 무료 오픈소스 CLI + GUI
**차별화**: Str.Forge(Find&Replace 정규식 치환), Deep.Diff(범용 파일 비교). Tab.Space는 인덴테이션이라는 단 하나의 문제를 전문적으로 해결하는 특화 도구

### 9. `Git.Quick` — Git 일상 명령어 트레이 팝업

**카테고리**: Dev / Productivity / Tray

### 7. `Cue.Blitz` — 당구 물리 블록깨기 융합

**장르**: Arcade / Physics / Classic
**핵심 메커닉**: 당구 큐로 공을 쳐서 블록을 부수는 블록깨기 변형. 쿠션(벽) 반사각과 회전(사이드스핀·탑스핀·백스핀)을 활용해 비직선 경로로 숨어있는 블록을 공략하는 것이 핵심이다. 한 번의 샷으로 쿠션을 활용한 3쿠션 연속 블록 파괴가 최고 쾌감이다.

**게임플레이**:
- 조준선 표시 (직사·예상 반사 경로 점선)
- 큐의 힘·각도·스핀 3요소 설정 후 발사
- 스핀 효과: 탑스핀(전진 가속), 백스핀(후진), 사이드스핀(쿠션 후 꺾임)
- 블록 종류: 일반·강화(여러 번 타격)·반사(공을 엉뚱한 방향으로 반사)·분열(작은 공 2개로 나뉨)
- 공 개수: 1~4개 동시 (체인 히트 콤보)
- 파워 샷: 충전 후 발사 → 공이 블록 관통
- 3쿠션 이상 연속 반사 후 블록 파괴 시 보너스 점수

**기술**: 2D 강체 충돌(Pool 물리, 마찰·탄성계수), Magnus 효과 스핀(각속도→횡력), 반사 벡터, B2D Sharp 또는 Farseer Physics
**매력**: 당구처럼 각도를 계산하는 쾌감 + Brick.Blitz의 블록 파괴 쾌감. 3쿠션 성공 시 극적인 카타르시스
**차별화**: Brick.Blitz(패들로 공 받기), Pool.Break(순수 당구), Bounce.House(탄성 반사각 퍼즐). Cue.Blitz만의 조준-스핀-쿠션 삼각 조합

### 8. `Volt.Surge` — 회로 기판 위 생존 슈터

**장르**: Shooter / Arcade / Survival
**핵심 메커닉**: 거대한 회로 기판이 게임 필드. 플레이어는 전류 파형(웨이브)이며, 회로 기판 위의 커패시터·저항·트랜지스터 소자들이 이동을 방해하거나 도와준다. 접지선을 밟으면 즉사, 배터리를 먹으면 속도 부스트, 코일을 통과하면 이동 방향이 꺾인다.

**게임플레이**:
- 회로 기판 격자 맵: 배선(이동 경로), 접지(즉사), 저항(이동 속도 감소), 코일(방향 꺾임)
- 적 캐릭터: 전하 입자들이 배선을 따라 이동하며 플레이어 추격
- 커패시터 충전: 적을 N마리 처치 시 충전 → 방출 시 EMP (화면 전체 적 스턴)
- 배터리 아이템: 섭취 시 2초간 무적+속도 3배
- 트랜지스터 스위치: 클릭으로 ON/OFF → 연결된 배선 구역 개폐 (도망 경로 조작)
- 웨이브 진행: 회로 기판 복잡도가 증가하고 배선 경로가 좁아짐
- 보스: 대형 CPU 칩 (주기적 전압 폭풍으로 화면 일부 무효화)

**기술**: 그래프 기반 맵(배선=간선, 소자=노드), BFS 경로 적 AI, 파티클 전기 방전 이펙트, WPF Canvas
**매력**: 전기 회로를 배경으로 한 독특한 시각. 소자별 특성을 이해하고 활용하는 전략적 생존
**차별화**: Volt.Chain(회로 설계 퍼즐), Volt.Race(전기 회로 트랙 레이싱), Chip.Logic(논리 게이트 교육). Volt.Surge는 회로 기판을 액션 슈터 필드로 활용

### 9. `Fire.Craft` — 화재 연소 물리 샌드박스

**장르**: Sandbox / Simulation / Creative
**핵심 메커닉**: 재질마다 연소 속도·온도·연기·잔불이 완전히 다른 화재 물리 샌드박스. 종이는 순식간에 타고, 나무는 천천히 숯이 되며, 얼음은 녹아 불을 끈다. 바람 방향을 조절해 불길을 퍼뜨리거나 소화기로 끄는 자유로운 실험이 가능하다.

**게임플레이**:
- 재질 8종 배치: 종이(빠른 연소), 나무(느린 연소+숯), 금속(전도열), 얼음(소화+수증기), 풀(빠른 번짐), 폭발물(폭발), 방화재(난연), 기름(연소 촉진)
- 불씨 도구로 화재 시작 (점화 위치·크기 조절)
- 바람 방향 슬라이더 (화염 편향·산소 공급량)
- 소화 도구: 물(넓은 범위), CO₂ 소화기(정밀), 모래(지속 차단)
- 온도 시각화 오버레이 (빨강=고온, 파랑=저온)
- 연기 유체 시뮬레이션 (바람 방향에 따라 흐름)
- 타이머 챌린지: N초 안에 특정 오브젝트만 소화
- 자유 실험 모드: 무제한 크기 맵, 모든 재질·도구

**기술**: 셀룰러 오토마타(연소 전파), 열 확산 유한차분법(온도 분포), Jos Stam 연기 시뮬(벡터 필드), WPF WriteableBitmap
**매력**: 재질마다 다른 연소 패턴을 관찰하는 호기심. Sand.Fall의 낙하 물리처럼 끝없이 실험하고 싶은 매력
**차별화**: Sand.Fall(입자 낙하 샌드박스), Crystal.Grow(결정화 성장), Magma.Flow(용암 유체 제어). Fire.Craft는 화재 연소 특화, 소화 인터랙션 추가

### 10. `Glitch.Hop` — 디지털 글리치 세계 플랫포머

**장르**: Platformer / Action / Surreal
**핵심 메커닉**: 픽셀이 무작위로 뭉개지고 색상이 왜곡된 글리치 디지털 세계에서 숨어있는 진짜 발판을 찾아 점프하는 플랫포머. 배경·적·함정이 모두 글리치 아트로 위장해 시각적 판별이 도전의 핵심이다. 플레이어 이동 시 글리치 강도가 변화해 진짜 지형이 순간 노출된다.

**게임플레이**:
- 배경 전체가 RGB 채널 분리, 스캔라인, 픽셀 쉬프트 등 글리치 효과로 왜곡
- 착지 가능한 발판은 짧은 주기로 원래 픽셀이 노출 (깜빡임)
- 점프 시 임팩트 프레임에 글리치 강도 감소 → 지형 일시 노출 이용
- 적 캐릭터: 글리치 아트로 위장해 발판처럼 보이나 닿으면 데미지
- 글리치 폭탄 아이템: 화면 전체 글리치 잠시 해제 → 진짜 지형 확인 시간
- 비밀 경로: 완전히 보이지 않는 투명 발판 (발판 표면 파티클로 힌트)
- 수집 요소: 숨겨진 코루프트 데이터 픽셀 수집 → 글리치 스킨 언락

**기술**: CRT 글리치 쉐이더(HLSL/GLSL → WPF ShaderEffect), RGB 채널 오프셋, 스캔라인 오버레이, 노이즈 함수(Perlin/Simplex), 플랫포머 물리(Verlet)
**매력**: 시각적 혼돈 속 패턴 인식 쾌감. 발판이 언제 나타날지 모르는 긴장감과 성공 시 쾌감. 강렬한 레트로 미래주의 미학
**차별화**: Dream.Shift(중력 8방향 회전 플랫포머), Phase.Gate(두 차원 발판 전환), Bolt.Hop(전기 플랫포머). Glitch.Hop은 시각적 왜곡 속 지형 판별이 게임플레이의 전부
