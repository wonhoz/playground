### 1. `Manga.View` — 오프라인 만화·망가 전용 리더

**카테고리**: Media / Files
**핵심 기능**: CBZ/CBR/ZIP 형식의 만화 파일을 추출 없이 탐색·열람하는 경량 WPF 리더. Epub.Cast가 EPUB만 지원하는 것과 달리 만화 특화 포맷(CBZ/CBR)을 지원하며, 이중 페이지·세로 스크롤·망가 RTL 읽기 방향을 지원한다.

**상세 기능**:
- CBZ(ZIP)/CBR(RAR)/7Z 만화 아카이브 추출 없이 스트리밍 열람
- 단일 페이지 / 이중 페이지 / 세로 스크롤 3종 뷰 모드
- 망가 모드(우→좌) / 코믹 모드(좌→우) 방향 전환
- 책장 라이브러리: 썸네일 그리드, 마지막 읽은 페이지 북마크
- 키보드/마우스휠 빠른 페이지 전환, 확대/축소(Pinch/Ctrl+휠)
- 시리즈 연속 읽기 (같은 폴더 내 다음 권 자동 이어가기)
- 배경색 커스터마이즈 (흰/검/크림), 다크 테마

**기술 스택**:
- .NET 10 / WPF
- SharpCompress (CBZ/CBR/7Z 스트리밍 추출)
- WPF WriteableBitmap + VirtualizingPanel (대용량 아카이브 지연 로딩)
- SQLite (읽은 페이지, 북마크 저장)

**수익 모델**: 무료 오픈소스
**차별화**: Epub.Cast는 EPUB 전용. Manga.View는 CBZ/CBR 만화 특화 포맷 + 이중 페이지 + RTL 지원으로 완전히 다른 사용자 대상.

### 2. `Wave.Gen` — 오디오 테스트 신호 생성기

**카테고리**: Audio / Dev·Tools
**핵심 기능**: 스피커·헤드폰·오디오 인터페이스 테스트를 위한 정밀 신호 생성기. 사인파·구형파·삼각파·핑크노이즈·화이트노이즈를 즉시 생성하고, 주파수 스위프(sweep)와 채널별 독립 제어로 스테레오 분리 테스트까지 지원한다.

**상세 기능**:
- 신호 유형: 사인파/구형파/삼각파/톱니파/화이트노이즈/핑크노이즈/침묵
- 주파수 정밀 입력 (1Hz~22kHz), 진폭(dBFS), 상승/하강 페이드
- 주파수 스위프: 선형/로그 스위프 (20Hz→20kHz, 사용자 설정 구간)
- 스테레오 채널 독립 제어 (L/R 각각 다른 신호, 채널 반전)
- 실시간 스펙트럼 분석기 표시 (FFT 512~8192 포인트)
- WAV 내보내기 (16/24/32bit, 44.1/48/96kHz)
- 저주파 20Hz/40Hz 우퍼 테스트 프리셋

**기술 스택**:
- .NET 10 / WPF
- NAudio (WaveOutEvent, SignalGenerator)
- MathNet.Numerics (FFT 스펙트럼 계산)
- WPF WriteableBitmap (실시간 스펙트럼 렌더링)

**수익 모델**: 무료
**차별화**: Flow.Beat(앰비언트 바이노럴), Audio.Cut(편집기)와 달리 Wave.Gen은 측정·테스트 목적의 정밀 신호 발생기. 오디오파일 개발자 특화.

### 3. `Timer.Chain` — 순차 연결 타이머 체인

**카테고리**: Productivity / Health
**핵심 기능**: 운동 루틴·요리 프로토콜·시험 시간 분배 등을 위한 순차 타이머 체인. Split.Ring(동시 다중 타이머)과 달리 각 단계가 자동으로 연속 실행되며, 이름·색상·알림 사운드를 단계별로 설정할 수 있다.

**상세 기능**:
- 단계 목록 생성: 이름/시간/색상/알림음 개별 설정
- 자동 순차 실행: 현재 단계 완료 → 자동으로 다음 단계로
- 남은 시간 대형 디스플레이 + 전체 진행률 프로그레스바
- 남은 시간 n초 전 경고음 커스터마이즈
- 루프 모드 (전체 체인 반복)
- 체인 프리셋 저장/불러오기 (HIIT/포모도로/요리/발표 등)
- 최소화 시 항상 위 떠있는 미니 오버레이 모드
- TTS 단계 이름 읽어주기 (SAPI)

**기술 스택**:
- .NET 10 / WPF
- NAudio (알림음 재생)
- System.Speech (TTS 단계 이름 읽기)
- JSON (프리셋 저장)

**수익 모델**: 무료
**차별화**: Split.Ring(동시 카운트다운)과 완전히 다른 순차 실행 메커닉. Slide.Timer(발표 전용)보다 범용적이며 루프·TTS 지원.

- shields.io URL 가져오기 (URL 붙여넣기 → 설정 역파싱)

**기술 스택**:
- .NET 10 / WPF
- WPF SVG 렌더링 (SharpVectors 또는 직접 SVG 템플릿)
- Simple Icons JSON 데이터 번들
- SkiaSharp (PNG 내보내기)

**수익 모델**: 무료
**차별화**: 기존 어떤 앱과도 겹치지 않는 완전 새로운 영역. shields.io 웹 의존성 없이 완전 오프라인 작동.

### 5. `ANSI.Forge` — ANSI/ASCII 아트 에디터 & 터미널 이스케이프 뷰어

**카테고리**: Dev · Tools / Creative
**핵심 기능**: 터미널 색상 이스케이프 코드(ANSI)와 ASCII/ANSI 아트를 편집·미리보기하는 WPF 도구. 개발자가 CLI 도구의 컬러 출력 디자인을 시각적으로 설계하고, 레트로 BBS 스타일 아트를 제작할 수 있다.

**상세 기능**:
- ANSI 이스케이프 시퀀스 실시간 렌더링 미리보기 (VT100/VT220/xterm-256)
- 16색/256색/트루컬러 전경·배경 색상 팔레트 GUI
- 문자 셀 그리기 도구: 펜/선/직사각형/채우기, CP437 블록 문자 팔레트
- 레이어 지원 (여러 레이어 합성 → 최종 ANSI 생성)
- .ans/.asc/.nfo 파일 열기/저장
- 이스케이프 코드 원시 텍스트 + 렌더링 결과 분할 뷰
- 내보내기: PNG 스냅샷, 원시 ANSI 텍스트, HTML `<pre>` 블록

**기술 스택**:
- .NET 10 / WPF
- 커스텀 ANSI 파서 (VT100/256color/truecolor)
- Lucide/CP437 완전 문자 팔레트
- WPF Canvas 셀 그리드 렌더링

**수익 모델**: 무료
**차별화**: Char.Art(ASCII 아트 생성)와 달리 ANSI.Forge는 터미널 색상 코드 편집 + .ANS/.NFO 파일 포맷 + CLI 컬러 출력 디자인 도구로 개발자 특화.

### 6. `SQL.Lens` — SQL 쿼리 실행 계획 시각화 & 최적화 도우미

**카테고리**: Dev · Tools
**핵심 기능**: SQLite/로컬 데이터베이스의 쿼리 EXPLAIN QUERY PLAN 결과를 시각적 트리 다이어그램으로 표시하고, 인덱스 미사용·전체 테이블 스캔 등 비효율 패턴을 자동 감지해 최적화 힌트를 제공한다.

**상세 기능**:
- SQLite DB 파일 드래그&드롭 연결
- SQL 에디터 (구문 강조, 자동완성) + 실행 결과 그리드
- EXPLAIN QUERY PLAN → 시각적 노드 트리 다이어그램
- 느린 패턴 자동 감지: 전체 테이블 스캔(SCAN TABLE), 미인덱스 정렬(USE TEMP B-TREE)
- 최적화 힌트 패널: "이 컬럼에 인덱스 추가 시 예상 성능 향상"
- 인덱스 현황 탭: 테이블별 인덱스 목록, 미사용 인덱스 경고
- 쿼리 히스토리 (실행시간, 반환 행수, 계획 저장)
- 쿼리 포맷터 (SQL Beautifier)

**기술 스택**:
- .NET 10 / WPF
- Microsoft.Data.Sqlite (EXPLAIN QUERY PLAN 실행)
- Graphviz 또는 자체 WPF Canvas 트리 렌더링
- AvalonEdit (SQL 에디터)

**수익 모델**: 무료
**차별화**: DB.Peek(SQLite 뷰어+SQL 에디터)와 달리 SQL.Lens는 실행 계획 시각화와 성능 최적화 힌트에 특화. DBA·백엔드 개발자 페인포인트 해결.

### 7. `Img.Compare` — 이미지 픽셀 비교기 & 품질 측정기

**카테고리**: Dev · Tools / Media
**핵심 기능**: 두 이미지를 픽셀 수준에서 비교하고 SSIM·PSNR·MSE 품질 지표를 계산하는 전문 도구. Deep.Diff가 텍스트/폴더/HEX 비교에 초점을 두는 것과 달리, Img.Compare는 이미지 품질 측정·압축 아티팩트 시각화·편집 전후 비교에 특화된다.

**상세 기능**:
- 두 이미지 좌우 분할 뷰 (싱크 줌/패닝, 독립 줌도 지원)
- 차이 레이어 오버레이: 픽셀 차이 절댓값 히트맵, 증폭 강도 조절
- 지표 패널: SSIM / PSNR(dB) / MSE / MAE / 다른 픽셀 비율
- 영역 선택 비교: 드래그로 특정 영역만 지표 계산
- 슬라이더 와이프: 수직 분할선을 좌우로 드래그해 비교
- GIF/PNG 반복 애니메이션 프레임별 비교
- 배치 비교: 폴더 A vs 폴더 B의 같은 이름 파일 자동 매핑
- CSV로 지표 일괄 내보내기

**기술 스택**:
- .NET 10 / WPF
- SkiaSharp (픽셀 연산, 히트맵 생성)
- ImageSharp (SSIM/PSNR 계산)
- WPF WriteableBitmap (실시간 diff 오버레이)

**수익 모델**: 무료
**차별화**: Deep.Diff는 텍스트 중심 비교기. Img.Compare는 SSIM/PSNR 측정, 와이프 뷰, 배치 비교로 이미지 품질 전문 도구.

### 8. `Hotkey.Map` — 전역 단축키 충돌 감지기 & 시각화

**카테고리**: System / Dev · Tools
**핵심 기능**: Windows에 등록된 모든 전역 단축키(Ctrl+Shift+..., Win+... 등)를 스캔해 현재 점유 앱과 함께 시각화하고, 충돌·중복을 경고한다. Key.Map(키보드 다이어그램 PDF)과 달리 실시간 동적 스캔이 핵심.

**상세 기능**:
- 시스템 전역 핫키 스캔: RegisterHotKey 시험 + 프로세스 핸들 매핑
- 키보드 다이어그램: 점유된 키 조합 색상 표시 (앱별 컬러)
- 충돌 경고: 동일 조합을 여러 앱이 사용하려 할 때 알림
- 앱별 필터: 특정 앱의 단축키만 강조
- 미사용 단축키 찾기: 비어있는 Ctrl/Alt/Win 조합 추천
- 커스텀 단축키 테스트: 직접 입력해 충돌 여부 즉시 확인
- AutoHotkey 스크립트 내보내기 (특정 조합 리매핑)

**기술 스택**:
- .NET 10 / WPF
- Win32 RegisterHotKey / UnregisterHotKey (테스트 방식 스캔)
- WMI / EnumProcesses (프로세스-단축키 매핑)
- WPF Canvas (키보드 다이어그램)

**수익 모델**: 무료
**차별화**: Key.Map은 앱별 단축키 다이어그램 시각화. Hotkey.Map은 실시간 시스템 스캔으로 충돌 감지 + 빈 조합 발굴. 개발자가 새 앱 단축키 설계 시 즉시 활용.

- 난이도: 주니어(명백한 버그) → 시니어(미묘한 엣지 케이스)
- 커뮤니티 버그 제출 시스템 (로컬 JSON 파일 공유)

**기술**: 내장 코드 스니펫 DB (JSON 200+ 문제), WPF RichTextBox 구문 강조(AvalonEdit), SQLite 고득점, 타이머 DispatcherTimer
**매력**: 개발자들의 직업적 자존심 자극 ("나는 이 버그 금방 찾겠지?"). 틀렸을 때의 "아!" 학습 효과. 신입 개발자 교육 도구로도 활용 가능.
**차별화**: Circuit.Break(전기 회로 디버깅), Asm.Quest(어셈블리 퍼즐)와 달리 실제 프로그래밍 언어의 고전적 버그 패턴을 빠르게 식별하는 게임.

### 9. `Chord.Build` — 화성학 코드 진행 선택 퍼즐

**장르**: Puzzle / Music / Educational
**핵심 메커닉**: 주어진 "분위기"(슬픔, 긴장, 희망, 신비로움 등)를 표현하기 위한 코드 진행을 화성학 규칙에 따라 선택하는 퍼즐. Ear.Train(청음 인식)과 달리 코드 진행의 구조적 설계와 화성학 이론을 배우는 게임이다.

**게임플레이**:
- 분위기 설명 + 장조/단조 키 제시
- 4마디 구성: 각 마디에 사용할 코드를 선택지 중 고르기
- 코드 선택 기준: 조성 내 다이어토닉 코드, 대리화음, 이차 도미넌트
- 선택할 때마다 해당 코드가 즉시 재생(NAudio 사인파)
- 목표 분위기와의 "매칭도" 점수: 화성학 규칙 준수 + 감성 적합도
- 해설 모드: 선택한 코드 진행의 기능화성 분석 (T/D/SD)
- 작곡 모드: 제약 없이 자유 코드 진행 만들기 + 재생
- 악보 표시 (코드 네임 + 음표)

**기술**: 화성학 규칙 엔진(다이어토닉 코드 테이블, 진행 평가 알고리즘), NAudio 화음 합성, WPF 악보 렌더링(Canvas)
**매력**: "왜 이 코드 진행이 슬프게 들릴까?"의 비밀을 직접 탐구. 음악 이론을 딱딱한 교과서가 아닌 실시간 청각 피드백으로 체험. Sound.Grid(멜로디 시퀀서), Wave.Form(사인파 합성)과 다른 화성학 포커스.
**차별화**: Ear.Train(청음 훈련), Beat.Drop/Chord.Strike(퍼포먼스 리듬)와 달리 코드 진행의 설계와 화성학 이론 학습에 집중.

### 10. `Path.Grid` — N×N 격자 최단경로 vs AI 최장경로 대결

**장르**: Strategy / Puzzle / Educational
**핵심 메커닉**: 플레이어는 A*로 최단 경로를 찾고, AI는 동시에 최장 경로(해밀턴 경로 탐색)를 방해하는 실시간 경쟁 퍼즐. 알고리즘 시각화 교육 도구이자 두뇌 싸움 게임이다.

**게임플레이**:
- N×N 격자 (시작=초록, 도착=빨강), 장애물 타일 배치
- 플레이어: 마우스로 경로 드래그 → 최단 경로(최소 타일) 그리기
- AI: 동시에 장애물을 하나씩 추가해 플레이어 경로를 길게 방해
- 턴 기반 모드: 교대로 경로 타일 vs 장애물 배치
- 미션: "장애물 추가 후에도 최단 경로 길이가 X 이하"
- 알고리즘 오버레이: BFS/A* 탐색 과정 애니메이션(교육 모드)
- 맵 에디터: 직접 장애물과 시작/끝 지점 배치
- 통계: 선택한 경로 vs 최적 경로 비교 표시

**기술**: A* 알고리즘(플레이어 경로 검증), BFS 최단거리 계산, 탐욕적 장애물 AI(최대 경로 증가 위치 선택), WPF Canvas 격자 렌더링
**매력**: "최단 경로 찾기"의 직관적 목표 + AI의 방해라는 긴장감. 알고리즘 학습 효과: 자연스럽게 A*, BFS, 그래프 이론 체득. Trace.Run(한붓 그리기)과 다른 경쟁적 최적화 메커닉.
**차별화**: Trace.Run(해밀턴 경로 홀로 풀기), Ice.Slide(슬라이딩 블록)와 다름. AI와의 실시간 경로 최적화 대결은 기존 목록에 없는 고유 개념.
