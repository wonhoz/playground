# 🧩 이미지 포맷 변환 툴 개발 지침서 (C#)

## 1. 📌 목표

SVG, PNG, ICO 등 다양한 이미지 포맷 간 변환을 지원하는 **경량 데스크탑 애플리케이션**을 개발한다.

특히 다음을 반드시 만족해야 한다:

- 원본 품질 유지 (무손실 변환)
- 투명도(Alpha Channel) 완전 보존
- ICO 생성 시 **멀티 사이즈 포함 필수**
- 대량 파일 처리 지원
- 직관적인 최소 UI

---

## 2. 🎯 핵심 기능 요구사항

### 2.1 포맷 변환

지원 입력:
- SVG
- PNG
- JPG / JPEG
- BMP

지원 출력:
- PNG
- JPG
- BMP
- ICO (중요)

---

### 2.2 ICO 변환 요구사항 (핵심)

ICO 생성 시 반드시 다음 포함:

- Multi-size 구성:
  - 16x16
  - 24x24
  - 32x32
  - 48x48
  - 64x64
  - 128x128
  - 256x256

- PNG 기반 ICO 지원 (Vista 이상 호환)
- 투명도 유지 (Alpha Channel 포함)
- 원본 비율 유지 (Aspect Ratio 유지)

---

### 2.3 품질 유지 (매우 중요)

- 리사이즈 시:
  - 고품질 알고리즘 사용 (Lanczos3 등)
- JPG 변환 시:
  - Quality 옵션 (기본 100)
- PNG:
  - Lossless 유지
- SVG → Raster:
  - 충분한 DPI 기반 렌더링

---

### 2.4 Drag & Drop

- 여러 파일 동시 드롭 지원
- 폴더 드롭 시 내부 파일 자동 처리

---

### 2.5 저장 정책

- 기본: **원본 파일 위치에 저장**
- 파일명: example.png → example.ico
- 덮어쓰기 옵션:
- 체크박스:
  - [ ] 기존 파일 덮어쓰기

- 미체크 시: example.png → example_converted.ico

---

### 2.6 Progress UI

- ProgressBar
- 현재 처리 파일 표시
- 전체 진행률 (%)

---

### 2.7 결과 다이얼로그

작업 완료 후:

- 성공 파일 수
- 실패 파일 수
- 실패 파일 목록
- 에러 메시지 요약

## 4. ⚙️ 내부 처리 흐름

### 전체 프로세스

1. 파일 드롭
2. 파일 필터링 (지원 포맷만)
3. 출력 포맷 기준 분기
4. 이미지 로드
5. 변환 처리
6. 저장
7. 결과 수집
8. 완료 다이얼로그 표시

---

## 5. 🧠 처리 로직 상세

### 5.1 SVG 처리

- SVG → Bitmap 변환 시:
  - DPI: 최소 300 이상
  - Background: Transparent

---

### 5.2 리사이즈 정책

- 비율 유지 (Aspect Ratio 유지)
- 여백 필요 시:
  - 투명 padding 추가

---

### 5.3 ICO 생성 로직

- 각 사이즈별 이미지 생성
- 하나의 ICO 파일에 모두 포함
- PNG 포맷으로 내부 저장

---

### 5.4 예외 처리

- 파일 손상
- 포맷 불일치
- 권한 문제
- 이미 사용 중인 파일

→ 개별 실패 처리 (전체 중단 금지)

---

## 6. ⚠️ 주의사항

- ❗ 품질 저하 절대 금지
- ❗ 투명도 손실 금지
- ❗ ICO 멀티사이즈 필수
- ❗ UI 프리즈 방지 (비동기 처리)

---

## 7. 🧪 테스트 케이스

- SVG → ICO (투명 유지)
- PNG → ICO (멀티사이즈 확인)
- PNG → JPG (품질 유지)
- 대량 파일 (100개 이상)
- 동일 파일 덮어쓰기 테스트
- 실패 파일 포함 케이스

---

## 8. 🎯 완료 기준 (Definition of Done)

- 모든 포맷 변환 정상 동작
- ICO 멀티사이즈 정상 포함
- 투명도 유지 확인
- 대량 처리 안정성 확보
- UI 끊김 없음
- 에러 처리 완료
