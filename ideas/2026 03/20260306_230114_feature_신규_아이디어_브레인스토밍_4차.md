### 1. `Context.Cast` — AI 프로젝트 컨텍스트 패커

| 항목 | 내용 |
|------|------|
| **카테고리** | Tools / AI |
| **한 줄 요약** | 코드베이스를 Claude/GPT에게 붙여넣기 위한 컨텍스트 파일 자동 패킹 도구 |
| **차별화** | AI.Clip(Claude API 처리기)과 달리, **코드베이스 → AI 입력 준비**에 특화 |

#### 핵심 기능
- 프로젝트 폴더 드래그&드롭 → **파일 트리 + 선택 파일 내용**을 하나의 텍스트 블록으로 병합
- 포함/제외 패턴 설정 (`.gitignore` 자동 인식, `*.min.js` `bin/` 등 글로브 패턴)
- 실시간 **토큰 카운터** (GPT-4o: ~128k / Claude Sonnet: ~200k 한도 표시)
- 출력 형식 선택: Markdown 코드블록 / XML 태그 / 플레인 텍스트
- 자주 쓰는 프로파일 저장 (프로젝트별 포함 파일 세트 기억)
- 클립보드 원클릭 복사 / 파일 저장

#### 왜 유용한가
개발자가 AI 코딩 어시스턴트에게 버그 수정이나 리팩토링을 요청할 때, 관련 파일을 수작업으로 복사해 붙여넣는 반복 작업을 해소함. AI 도구 사용이 일상화된 현재 **매일 쓸 도구**.

#### 수익 모델
- 무료 기본 (단일 폴더, 10파일 이하)
- Pro: 무제한 파일, 팀 프로파일 공유, VSCode 확장 연동, git blame 포함 모드
- **수익 잠재력: ★★★★★** (AI 도구 수요 폭발적 성장)

#### 기술 포인트
- `System.IO` 재귀 파일 탐색, GitIgnore 파서 (libgit2sharp 또는 자체 구현)
- 토큰 카운팅: BPE 근사 (Tiktoken .NET 포팅 or 자체 wordpiece 근사)
- WPF 가상화 트리뷰 (파일 수천 개 체크박스)

### 2. `Mosaic.Forge` — 포토모자이크 생성기

| 항목 | 내용 |
|------|------|
| **카테고리** | Applications / Media |
| **한 줄 요약** | 수백~수천 개의 작은 사진을 타일로 사용해 목표 이미지를 재현하는 포토모자이크 생성기 |
| **차별화** | Img.Forge(일괄 변환·압축)나 Photo.Video.Organizer(날짜 정리)와 완전히 다른 **창작 합성 도구** |

#### 핵심 기능
- **목표 이미지** + **소스 이미지 폴더** 선택 → 포토모자이크 자동 합성
- 타일 크기 조정 (16×16 ~ 128×128px), 겹침 허용 여부, 중복 타일 사용 횟수 제한
- 색상 매칭 알고리즘 (평균 Lab 색차, 히스토그램 매칭 선택)
- 실시간 미리보기 (저해상도 빠른 프리뷰 → 고해상도 렌더링)
- 최대 1억 픽셀 출력 (A1 인쇄 품질), PNG/JPG 내보내기
- 소스 이미지 자동 썸네일 캐시 (재사용 시 초고속)

#### 왜 유용한가
부모님 사진, 여행 사진, 팬아트 등 **추억이 담긴 사진**들로 대형 포스터를 만들 수 있는 감성 도구. Word.Cloud나 Char.Art처럼 "완성물이 인상적인" 카테고리.

#### 수익 모델
- 무료: 소스 이미지 100장 이하, 최대 4K 출력
- Pro: 무제한, 8K 출력, 프린트 주문 연동 (파트너 인쇄소 API)
- **수익 잠재력: ★★★★☆** (특별한 날 선물용 수요)

#### 기술 포인트
- k-d Tree 색상 공간 인덱싱 (CIE Lab) → 타일당 O(log n) 최근접 색상 검색
- 병렬 타일 매칭 (`Parallel.For`), 진행률 ProgressBar
- 고해상도 비트맵 청크 렌더링 (메모리 초과 방지)

### 3. `Volume.Cast` — 앱별 볼륨 믹서 트레이

| 항목 | 내용 |
|------|------|
| **카테고리** | Tools / Tray |
| **한 줄 요약** | Windows 볼륨 믹서보다 훨씬 빠르고 예쁜 앱별 볼륨 독립 제어 트레이 앱 |
| **차별화** | Sound.Cast(출력 **장치** 전환)와 완전히 다름 — **앱마다 볼륨 수치·뮤트**를 빠르게 제어 |

#### 핵심 기능
- 실행 중 오디오 앱 자동 감지 + 앱 아이콘 + 볼륨 슬라이더 + 뮤트 버튼 한눈에
- **볼륨 프로파일** 저장·전환 (게임 모드: 게임 100% 음악 30%; 회의 모드: Discord 100% 나머지 뮤트)
- 전역 단축키: 특정 앱 볼륨 즉시 +10/-10, 특정 앱 뮤트 토글
- 트레이 아이콘 휠 스크롤 → 마스터 볼륨 조절
- 앱별 볼륨 변경 히스토리 (어떤 앱이 갑자기 소리를 바꿨는지 추적)

#### 왜 유용한가
YouTube 보다가 Discord 알림이 터지거나, 게임 중 음악을 줄이고 싶을 때 Windows 믹서 열기가 너무 번거로움. Mouse.Flick처럼 **마우스/단축키 즉각 반응** 계열의 쾌적한 UX.

#### 수익 모델
- 무료 기본 (프로파일 3개 이하)
- Pro: 무제한 프로파일, 앱 자동 규칙 (특정 앱 실행 시 자동 전환), EQ 플러그인
- **수익 잠재력: ★★★★☆**

#### 기술 포인트
- Windows Core Audio API (IAudioSessionManager2, ISimpleAudioVolume)
- WinForms NotifyIcon + 커스텀 팝업 폼 (슬라이더 컴포넌트)

#### 기술 포인트

### 7. `Noti.Hub` — Windows 알림 히스토리 관리자

| 항목 | 내용 |
|------|------|
| **카테고리** | Tools / Tray |
| **한 줄 요약** | 사라진 Windows 알림을 다시 볼 수 있는 알림 히스토리 허브 트레이 앱 |
| **차별화** | Stay.Awake(화면 꺼짐 방지)나 Toast.Cast(직접 알림 발송)와 달리 **기존 알림 수신·보관** |

#### 핵심 기능
- 모든 앱의 Toast 알림을 백그라운드에서 가로채 히스토리 DB에 저장
- 트레이 팝업으로 최근 N개 알림 목록 (앱 아이콘, 제목, 내용, 시간)
- 앱별 필터, 키워드 검색, 중요 알림 별표 표시
- **재발송**: 특정 알림을 다시 Toast로 보내기
- **무음 앱 지정**: 특정 앱 알림을 소리/팝업 없이 히스토리에만 저장
- 오늘/이번 주/전체 기간 통계 (어떤 앱이 알림을 가장 많이 보냈나)

#### 왜 유용한가
"분명 뭔가 알림이 왔는데 놓쳤다", "5분 전에 온 Slack 알림 내용이 뭐였지" — Windows에는 macOS 알림 센터 같은 히스토리 기능이 사실상 없음. **Stay.Awake처럼 트레이 상주 + 실용성** 카테고리.

#### 수익 모델
- 무료: 최근 100개 알림 저장
- Pro: 무제한 + 알림 규칙 자동화 (특정 키워드 → 이메일 전달) + 검색
- **수익 잠재력: ★★★★☆**

#### 기술 포인트
- Windows.UI.Notifications / WinRT 알림 API (`ToastNotificationManager.History`)
- 또는 접근성 API + UIAutomation으로 알림 인터셉트
- SQLite 로컬 저장, WinForms NotifyIcon 팝업 UI

### 8. `Tray.Stats` — 통합 시스템 성능 트레이 모니터

| 항목 | 내용 |
|------|------|
| **카테고리** | Tools / Tray / System |
| **한 줄 요약** | CPU·RAM·GPU·디스크·네트워크를 하나의 트레이 아이콘과 팝업에서 모두 보는 통합 성능 모니터 |
| **차별화** | Thermal.View(온도만) · Net.Speed(네트워크만)와 달리 **모든 지표를 한 앱에서** |

#### 핵심 기능
- 트레이 아이콘 자체가 실시간 미니 그래프 (CPU 사용률 색상 막대)
- 클릭 시 팝업: CPU/RAM/GPU/디스크/네트워크 6개 그래프 (60초 슬라이딩 윈도우)
- 임계치 알림: "CPU 95% 지속 30초 → Toast 경고"
- **상위 프로세스 오버레이**: 어떤 프로세스가 CPU/RAM을 가장 많이 쓰는지 3위까지 표시
- 온도 표시 옵션 (OpenHardwareMonitor 라이브러리 연동)
- 최소 메모리 사용 (<5MB) 상시 실행 최적화

#### 왜 유용한가
Disk.Lens·Sys.Clean을 좋아하는 시스템 관심 유저에게 딱 맞는 도구. "지금 뭐가 느린지" 항상 트레이에서 확인 가능.

#### 수익 모델
- 무료 + 기부 모델 (가볍고 유용한 도구)
- Pro: 히스토리 CSV 내보내기, 원격 모니터링 (다른 PC 지표 수신)
- **수익 잠재력: ★★★☆☆** (경쟁 앱 있지만 WPF 다크 테마 차별화)

#### 기술 포인트
- `System.Diagnostics.PerformanceCounter` (CPU/RAM/Disk)
- WMI `Win32_VideoController` GPU 사용률
- `System.Net.NetworkInformation` 네트워크 통계
- WinForms NotifyIcon + 커스텀 OwnerDraw 트레이 아이콘 (그래프 직접 그리기)
