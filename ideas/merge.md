  ---
  Log.Lens vs Log.Tail

  두 앱 모두 FileSystemWatcher + 멀티탭 + 레벨 파싱 + 정규식 필터 + 자동 스크롤이 중복 구현되어 있습니다.

  ┌────────────────────────────────────┬──────────┬──────────┐
  │                기능                │ Log.Lens │ Log.Tail │
  ├────────────────────────────────────┼──────────┼──────────┤
  │ 실시간 tail 감시                   │    O     │    O     │
  ├────────────────────────────────────┼──────────┼──────────┤
  │ 멀티탭 + 드래그앤드롭              │    O     │    O     │
  ├────────────────────────────────────┼──────────┼──────────┤
  │ 레벨 파싱 (FATAL~TRACE)            │    O     │    O     │
  ├────────────────────────────────────┼──────────┼──────────┤
  │ 정규식 필터                        │    O     │    O     │
  ├────────────────────────────────────┼──────────┼──────────┤
  │ 키워드 하이라이트 (노란 배경 강조) │    O     │    X     │
  ├────────────────────────────────────┼──────────┼──────────┤
  │ 레벨 드롭다운 필터                 │    O     │    X     │
  ├────────────────────────────────────┼──────────┼──────────┤
  │ 폴더 열기 (최대 20개 일괄 탭)      │    O     │    X     │
  ├────────────────────────────────────┼──────────┼──────────┤
  │ UI 가상화 (VirtualizingStackPanel) │    X     │    O     │
  ├────────────────────────────────────┼──────────┼──────────┤
  │ 최대 줄 수 제한 (10K/50K/100K)     │    X     │    O     │
  ├────────────────────────────────────┼──────────┼──────────┤
  │ 제외(Exclude) 필터                 │    X     │    O     │
  ├────────────────────────────────────┼──────────┼──────────┤
  │ 대소문자 구분 필터                 │    X     │    O     │
  ├────────────────────────────────────┼──────────┼──────────┤
  │ 파일 로테이션 감지                 │    X     │    O     │
  └────────────────────────────────────┴──────────┴──────────┘

  결론: 거의 같은 앱. Log.Tail의 강점(성능 가상화 + 제외 필터 + 로테이션 대응)과 Log.Lens의 강점(하이라이트 + 레벨 필터
  + 폴더 열기)을 합치면 훨씬 강력한 단일 앱이 됩니다.

  ---
  Mock.Desk vs Mock.Server

  두 앱 모두 ASP.NET Core 임베디드 서버 + 와일드카드 매칭 + 응답 지연 + 요청 로그가 중복 구현되어 있습니다.

  ┌──────────────────────────────────────────┬───────────┬─────────────┐
  │                   기능                   │ Mock.Desk │ Mock.Server │
  ├──────────────────────────────────────────┼───────────┼─────────────┤
  │ ASP.NET Core 서버 엔진                   │     O     │      O      │
  ├──────────────────────────────────────────┼───────────┼─────────────┤
  │ 와일드카드(*) 경로 매칭                  │     O     │      O      │
  ├──────────────────────────────────────────┼───────────┼─────────────┤
  │ 응답 지연 시뮬레이션                     │     O     │      O      │
  ├──────────────────────────────────────────┼───────────┼─────────────┤
  │ 요청 로그 패널                           │     O     │      O      │
  ├──────────────────────────────────────────┼───────────┼─────────────┤
  │ GUI 클릭 편집 (마우스로 엔드포인트 구성) │     O     │      X      │
  ├──────────────────────────────────────────┼───────────┼─────────────┤
  │ 엔드포인트 활성화 토글                   │     O     │      X      │
  ├──────────────────────────────────────────┼───────────┼─────────────┤
  │ YAML/JSON 코드 에디터                    │     X     │      O      │
  ├──────────────────────────────────────────┼───────────┼─────────────┤
  │ CORS 자동 처리                           │     X     │      O      │
  ├──────────────────────────────────────────┼───────────┼─────────────┤
  │ 파일 hot-reload                          │     X     │      O      │
  ├──────────────────────────────────────────┼───────────┼─────────────┤
  │ 실시간 유효성 검사                       │     X     │      O      │
  └──────────────────────────────────────────┴───────────┴─────────────┘

  결론: 접근 방식이 달라서 상호 보완적입니다 — "마우스로 빠르게 만들 때" vs "파일로 정밀하게 관리할 때". 하지만 서버
  엔진 자체가 사실상 동일 코드라서 합치면 더 강력합니다.
