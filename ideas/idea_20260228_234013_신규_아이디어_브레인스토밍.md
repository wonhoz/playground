### 🖼️ GIF.Cast — 화면 녹화 GIF 변환기

| 항목 | 내용 |
|------|------|
| **카테고리** | Screen / Dev Tools |
| **수익 모델** | 무료 + Pro (고품질 팔레트, 편집, 1080p) |
| **수익 비전** | GitHub README·Slack 공유용으로 개발자 수요 폭발적 — 개인 라이선스 판매 가능 |

**핵심 기능**
- 화면 영역 드래그 선택 → GIF 녹화 시작
- 실시간 FPS·품질·크기 제한 설정
- 마우스 하이라이트 + 클릭 이펙트 옵션
- 팔레트 최적화 (256색 디더링) + 압축
- 녹화 후 트림·속도 조절 → 클립보드 or 저장

**차별점 vs Screen.Recorder**: Screen.Recorder는 MP4 비디오 중심, GIF.Cast는 GIF 특화 — GitHub PR, 이슈, Notion 임베드 용도로 완전히 다른 유즈케이스

**기술 스택**: `SharpDX` 화면 캡처 + `AnimatedGif` NuGet, WPF

- 특수 타일: 이중 파이프(두 색 교차 가능), 잠금 셀(변경 불가), 워프(순간이동)
- 시간 모드 / 무한 시도 모드 (Hue.Flow처럼 느긋하게)

**비주얼 방향**: 액체 흐르는 네온 파이프 → 완성 시 파이프에 흐름 애니메이션

### ⚽ Fuse.Ball — 듀얼 볼 동시 조작 퍼즐

| 항목 | 내용 |
|------|------|
| **장르** | Puzzle / Action |
| **참고 최애** | Hue.Flow의 논리 + Dodge.Blitz의 반응 속도 혼합 |
| **수익 비전** | 독특한 메커닉 → 바이럴 잠재력 (유튜브 "이게 뭔 게임이야" 반응) |

**핵심 메커닉**
- 두 공(A·B)이 화면에 동시 존재
- 하나의 입력(WASD or 마우스)으로 둘을 동시에 반대 방향으로 조종
  - ex) A → A는 오른쪽, B는 왼쪽 동시 이동
- 각 공이 자신의 목표 지점에 동시 도달해야 클리어
- 장애물은 공마다 다름 (A에게만 벽, B에게만 빈 공간)
- 고급 레벨: 3개 공, 타이밍 기반 스위치 장애물
- 스피드런 모드: 최단 시간 클리어 리더보드

**비주얼 방향**: 두 공의 색상 대비 (핑크 vs 시안), 궤적 트레일, 도착 시 폭발 이펙트
