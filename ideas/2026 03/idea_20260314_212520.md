### 1. `Str.Forge` — 대용량 멀티파일 Find & Replace 엔진

**카테고리**: Dev/Tools / Desktop
**핵심 기능**: 수십~수백 개 파일을 한 번에 열어 정규식·리터럴 검색·치환을 수행하고, 패치 미리보기 후 선택 적용하는 일괄 텍스트 치환 도구다. `Text.Forge`(해시·인코딩 변환)나 `Find.Fast`(파일명 검색)과 달리 *파일 내용*의 정밀 다중 치환에 집중한다.

**상세 기능**:
- 폴더·glob 패턴 파일 대상 지정 (**.cs, **.md, **.json 등)
- 정규식 치환: 캡처 그룹 참조($1, $2), Named Group (이름·날짜 재배치)
- 멀티스텝 파이프라인: 치환 규칙 여러 개를 순서대로 적용
- 패치 미리보기: diff 하이라이트(삭제=적, 추가=녹)로 적용 전 확인
- 파일별 선택 적용 / 전체 적용 / 되돌리기 (`.bak` 자동 생성)
- 인코딩 자동 감지 (UTF-8/UTF-16/CP949) + BOM 보존 옵션
- 치환 규칙 세트를 프리셋으로 저장·불러오기 (JSON)
- 드래그앤드롭 폴더 지정, 결과 CSV 내보내기

**기술 스택**:
- `System.Text.RegularExpressions.Regex` (캡처 그룹, Named Group)
- `DiffPlex` NuGet — 줄 단위 diff 생성
- WPF `FlowDocument` 구문 강조 (기존 JSON.Fmt 패턴 재사용)
- `System.IO.FileSystemWatcher` (치환 후 파일 변경 감시 옵션)

**수익 모델**: 무료 (오픈소스). 개발자·기술 문서 작성자 타겟.
**차별화**: `Text.Forge`는 해시·인코딩 변환 도구. `Find.Fast`는 파일명 검색+grep 뷰어. `Str.Forge`는 *멀티파일 내용 일괄 치환 + 패치 미리보기*에 특화된 유일한 도구.

### 2. `Csv.Diff` — CSV/스프레드시트 두 버전 셀 단위 Diff

**카테고리**: Dev/Tools / Data / Desktop
**핵심 기능**: 두 CSV 파일을 열어 행·셀 단위로 추가/삭제/변경을 시각화하는 데이터 전문 diff 도구다. 키 컬럼을 지정해 행 재정렬에도 올바른 매칭을 수행하고, 결과를 컬러 테이블·Excel·HTML로 내보낸다.

**상세 기능**:
- 키 컬럼(Primary Key) 지정 → 행 순서가 바뀌어도 정확한 매칭
- 셀 변경 강조: 추가(녹), 삭제(적), 변경(황) 3색 표시
- 필터: 변경된 행만 / 추가 행만 / 삭제 행만
- 컬럼 단위 통계: 변경률 (%), 변경 값 분포
- Excel(.xlsx) / HTML / CSV(마커 컬럼 포함) 내보내기
- 대용량 지원: 100만 행 이상 가상화 DataGrid
- 드래그앤드롭, 클립보드 직접 붙여넣기 (TSV 포함)
- 인코딩 자동 감지, 구분자 자동 감지(콤마/탭/세미콜론)

**기술 스택**:
- `CsvHelper` NuGet — CSV 파싱
- `ClosedXML` NuGet — Excel 내보내기
- WPF `DataGrid` + `VirtualizingPanel` — 대용량 가상화
- Myers diff 알고리즘 (LCS 기반) — 행 매칭

**수익 모델**: 무료. DB 마이그레이션 검증, 데이터 파이프라인 QA 타겟.
**차별화**: `Deep.Diff`는 텍스트/이미지/HEX 범용 비교기로 CSV 구조를 모름. `Schema.Diff`는 스키마 구조 diff. `Table.Craft`는 단일 파일 분석. `Csv.Diff`는 *두 CSV의 데이터 내용을 키 매칭으로 정밀 비교*하는 유일한 도구.

### 3. `Link.Check` — HTML/Markdown/소스코드 깨진 링크 감사기

**카테고리**: Dev/Tools / QA / Desktop
**핵심 기능**: 폴더·파일 내 HTML, Markdown, 소스코드에서 모든 링크(URL, 앵커, 로컬 파일 경로)를 추출하고 HTTP 상태 확인 + 로컬 파일 존재 여부를 검증하는 링크 감사 도구다. 보고서를 Markdown·HTML·JSON으로 내보낸다.

**상세 기능**:
- 지원 파일: `.html`, `.md`, `.rst`, `.cs`, `.js`, `.ts`, `.py`
- URL 추출: `href`, `src`, `url()`, `[text](url)`, 코드 내 문자열 리터럴
- HTTP 검증: GET/HEAD 상태 코드 (200·301·302·404·5xx), 타임아웃 설정
- 로컬 파일 링크: 상대 경로 해석, 대소문자 불일치 감지
- 앵커 검증: `#section-id` 존재 여부 확인
- 병렬 검증 (SemaphoreSlim, 동시 연결 수 제한)
- 결과 필터: 깨진 링크만 / 경고만 / 전체
- Markdown·HTML·JSON·CSV 보고서 내보내기

**기술 스택**:
- `HtmlAgilityPack` NuGet — HTML 파싱
- `System.Net.Http.HttpClient` — 병렬 HTTP 검증
- `System.Text.RegularExpressions` — Markdown/코드 링크 추출
- WPF DataGrid, TreeView (파일별 그룹)

**수익 모델**: 무료. 문서 관리자·웹 개발자·테크니컬 라이터 타겟.
**차별화**: `Api.Probe`는 수동 API 테스터. `Uptime.Eye`는 지속 폴링 모니터. `Link.Check`는 *파일 코드베이스 전체의 링크 무결성을 일괄 감사*하는 도구.

### 4. `Hotspot.Cast` — Windows 모바일 핫스팟 원클릭 트레이 관리자

**카테고리**: Network / Tray / Desktop
**핵심 기능**: Windows 기본 모바일 핫스팟 기능을 트레이에서 원클릭 ON/OFF하고, SSID·비밀번호를 자빠르게 설정·공유하며, 접속 기기 목록과 실시간 데이터 사용량을 모니터링하는 트레이 앱이다.

**상세 기능**:
- Windows API (`INetSharingManager`, `MobileHotspot` COM API) 원클릭 ON/OFF
- SSID·비밀번호 설정 (트레이 메뉴에서 직접 편집)
- QR 코드 자동 생성 (QR.Forge 로직 재사용) → 폰 카메라로 즉시 연결
- 접속 기기 목록 (MAC·IP·호스트명, 연결 시각)
- 실시간 데이터 사용량 (기기별 업·다운 KB/s 그래프)
- 자동 시작 설정, 일정 시간 후 자동 종료 타이머
- 풍선 알림 (기기 접속/해제 알림)
- 최대 접속 기기 수 제한 설정

**기술 스택**:
- `Windows.Networking.NetworkOperators.NetworkOperatorTetheringManager` (WinRT via CsWinRT)
- `System.Net.NetworkInformation` — 기기 목록·트래픽
- WinForms `NotifyIcon` + `ContextMenuStrip` (`DarkMenuRenderer`)
- QR 생성: `QRCoder` NuGet

**수익 모델**: 무료 트레이 앱.
**차별화**: `WiFi.Cast`는 Wi-Fi 채널·신호 분석기. `VPN.Cast`는 WireGuard GUI. `DNS.Flip`은 DNS 전환. `Hotspot.Cast`는 *Windows 핫스팟 생성·관리*에 특화.

### 5. `Cue.Cast` — 발표자용 텔레프롬프터

**카테고리**: Productivity / Presentation / Desktop
**핵심 기능**: 원고 텍스트를 자동 스크롤하는 텔레프롬프터 앱으로, 발표자·유튜버·방송인이 카메라를 바라보며 원고를 읽을 수 있도록 돕는다. 2번 모니터 전용 표시, 가로 반전(미러), 속도 실시간 조절을 지원한다.

**상세 기능**:
- 원고 텍스트 편집 (마크다운 볼드·헤딩으로 강조 표시)
- 자동 스크롤: 속도 슬라이더 (WPM 기반, 1~200 WPM)
- 가로 반전(미러) 모드 — 하프미러 프롬프터 장비 지원
- 2번 모니터 전용 전체화면 (기본 모니터는 편집 뷰)
- 원격 제어: 같은 PC의 키보드/게임패드·블루투스 미디어 키로 일시정지/재개/속도 조절
- 현재 위치 커서 강조 (읽고 있는 줄 배경 하이라이트)
- 글꼴 크기·색상·배경색 자유 설정
- .txt / .docx / .md 파일 가져오기

**기술 스택**:
- WPF `DispatcherTimer` 기반 ScrollViewer 애니메이션
- `DocumentFormat.OpenXml` NuGet — DOCX 가져오기
- `WpfScreenHelper` NuGet — 멀티 모니터 인식
- WPF `FlowDocument` — 마크다운 강조 렌더링

**수익 모델**: 무료. 유튜버·강사·발표자 타겟.
**차별화**: `Slide.Cast`는 마크다운 슬라이드 프레젠테이션. `Slide.Timer`는 발표 타이머. `Mark.View`는 마크다운 뷰어. `Cue.Cast`는 *텔레프롬프터 스크롤·미러·WPM 제어*에 특화.

### 6. `Git.Guard` — Git 훅 GUI 설정기

**카테고리**: Dev/Tools / Git / Desktop
**핵심 기능**: Git의 pre-commit, commit-msg, pre-push 훅을 GUI로 설정하는 도구다. 린터·포매터·테스트 실행 조건을 드래그앤드롭으로 편집하고 쉘 스크립트를 자동 생성하여 `.git/hooks/`에 설치한다.

**상세 기능**:
- 훅 종류 선택: pre-commit, commit-msg, pre-push, post-checkout
- 조건 블록 빌더: "파일 패턴이 *.cs면 → `dotnet format` 실행" 등 if-then 조건 편집
- 기본 제공 레시피: dotnet format, eslint, prettier, pytest, 커밋 메시지 패턴 검사
- 쉘 스크립트 자동 생성 (bash/PowerShell 선택) + 즉시 `.git/hooks/` 설치
- 저장소별 훅 프로파일 저장·내보내기·팀 공유 (JSON)
- 훅 실행 시뮬레이션 (테스트 파일 세트에 적용해 통과/실패 미리보기)
- 훅 비활성화(skip) 토글, 실행 로그 뷰어
- 여러 저장소에 동일 훅 일괄 배포

**기술 스택**:
- `LibGit2Sharp` NuGet — 저장소 탐지·훅 경로 접근
- WPF 드래그앤드롭 블록 에디터 (Canvas + 조건 Panel)
- `System.Diagnostics.Process` — 훅 시뮬레이션 실행
- Handlebars/T4 템플릿 — 쉘 스크립트 생성

**수익 모델**: 무료. 개발팀·CI/CD 담당자 타겟.
**차별화**: `Git.Stats`(통계 분석), `Git.Reel`(그래프 탐색), `Git.Blame`(blame 히트맵)과 완전히 다른 영역. `Git.Guard`는 *훅 자동화 스크립트를 GUI로 설계*하는 유일한 도구.

### 7. `Win.Scope` — 실행 창 Z-order·투명도 실시간 조작기

**카테고리**: Dev/Tools / System / Desktop
**핵심 기능**: 실행 중인 모든 Window를 실시간으로 나열하고, Z-order(항상 위/아래), 투명도(Alpha), 크기·위치를 GUI에서 직접 수정하는 창 인스펙터·조작 도구다. `Win.Spy`(UIAutomation 계층 탐색)와 달리 시각적 레이아웃 제어에 집중한다.

**상세 기능**:
- 실행 창 목록 (PID, 타이틀, 클래스명, 프로세스명)
- Z-order 조작: HWND_TOPMOST / HWND_NOTOPMOST / HWND_BOTTOM
- 투명도 설정: `SetLayeredWindowAttributes` (0~255 슬라이더)
- 크기·위치 수동 입력 (`SetWindowPos`)
- 창 숨김/표시 토글 (ShowWindow SW_HIDE / SW_SHOW)
- 특정 창의 마우스 클릭 통과(Click-through) 토글
- 창 스냅샷 저장 (타이틀·Z-order·투명도 세트) → 복원
- 드래그로 창 위치 가상 조작 (썸네일 미리보기)

**기술 스택**:
- `User32.dll` P/Invoke: `EnumWindows`, `SetWindowPos`, `SetLayeredWindowAttributes`, `ShowWindow`
- WPF `DispatcherTimer` — 창 목록 실시간 갱신
- `DwmGetWindowAttribute` — 썸네일 캡처
- `Accessibility Insights` 없이 순수 Win32 API 기반

**수익 모델**: 무료. 멀티모니터 파워유저·스트리머·UI 테스터 타겟.
**차별화**: `Win.Spy`는 UIAutomation 기반 컨트롤 계층 탐색. `Win.Tamer`는 CPU 친화도·우선순위. `Win.Scope`는 *투명도·Z-order·클릭통과 등 창 시각·인터랙션 속성 실시간 제어*에 특화.

### 8. `Diff.Snap` — 폴더 스냅샷 + 시점 간 변경 히스토리 타임라인

**카테고리**: Dev/Tools / System / Desktop
**핵심 기능**: 폴더 상태를 스냅샷으로 찍어 두고, 나중에 두 스냅샷을 비교해 파일 추가·삭제·수정을 타임라인으로 시각화하는 변경 추적 도구다. Git이 없는 환경에서 설정 폴더·게임 세이브·프로젝트 빌드 산출물 변화를 추적하는 데 유용하다.

**상세 기능**:
- 폴더 지정 → 파일 해시(SHA-256) + 메타데이터 스냅샷 저장
- 스냅샷 이름·태그·메모 부여 (버전처럼 관리)
- 두 스냅샷 비교: 추가(녹)·삭제(적)·수정(황) 파일 목록
- 파일 내용 diff: 텍스트 파일은 줄 단위 diff 표시
- 타임라인 뷰: X축=시간, Y축=변경 파일 수 스파크라인
- 변경된 파일 복원: 이전 스냅샷 버전으로 되돌리기
- 스냅샷 내보내기(ZIP), 외부 저장소 경로 이동
- 자동 스케줄 스냅샷 (매 N분/시간, `Task Scheduler` 연동)

**기술 스택**:
- `System.Security.Cryptography.SHA256` — 파일 해시
- `DiffPlex` NuGet — 텍스트 diff
- `SQLite` (`Microsoft.Data.Sqlite`) — 스냅샷 메타데이터 저장
- WPF `Canvas` 스파크라인, `DataGrid` 파일 목록

**수익 모델**: 무료. 개발자·게이머·시스템 관리자 타겟.
**차별화**: `Deep.Diff`는 두 파일/폴더 즉시 비교. `Folder.Sync`는 동기화 도구. `Dupe.Guard`는 중복 파일 실시간 감시. `Diff.Snap`은 *특정 시점의 상태를 아카이빙하고 나중에 비교*하는 타임머신 접근.

### 9. `Img.Slice` — UI 목업·스프라이트시트 자동 슬라이서

**카테고리**: Dev/Tools / Media / Desktop
**핵심 기능**: 큰 이미지(UI 목업, 스프라이트시트)를 격자·수동 선택·여백 감지로 슬라이싱하여 개별 PNG/SVG 파일로 내보내는 도구다. `Sprite.Forge`(개별 이미지→시트로 *조립*)와 반대 방향 — 시트→개별 이미지로 *분해*.

### 1. `Knit.Cast` — 실 교차 패턴 완성 퍼즐

**장르**: Puzzle / Logic / Creative
**핵심 메커닉**: N×M 격자에 가로·세로 실을 배치하고 교차점에서 "위/아래" 순서를 지정하여 목표 매듭 패턴을 완성하는 퍼즐이다. 단순 색 맞추기가 아니라 교차 *순서*가 구조적 유효성을 결정하므로, 매듭이론의 시각적 직관을 게임화한다.

**게임플레이**:
- 격자에 가로·세로 실 색상을 드래그로 배치
- 교차점을 클릭하면 위/아래 순서가 토글됨 (Over/Under)
- 목표 이미지(교차 패턴)와 일치하면 스테이지 클리어
- 잘못 배치된 실은 되감기 가능 (Undo 횟수 제한 있는 하드 모드)
- 레벨 에디터: 나만의 매듭 패턴 만들어 공유
- 스코어: 최소 교차 수 달성 시 Perfect (최적 매듭 구조)
- 챌린지 모드: 제한 시간 내 최대 패턴 완성
- 난이도 곡선: 3×3 → 최대 8×8 격자, 실 수 증가

**기술**: WPF Canvas Bezier 곡선으로 실 렌더링, 교차점 Z-order 관리, BFS로 연결 구조 검증, 매듭 위상 동치 검사(Reidemeister move 단순화)
**매력**: "위/아래만 바꾸면 되는데..." 하면서 계속 꼬이는 쾌감. 완성된 패턴의 시각적 아름다움이 수집 욕구를 자극. 자수·매크라메 취미와 연결되는 감성.
**차별화**: `Knot.Craft`는 이미 *만들어진* 매듭을 드래그로 *풀기*. `Knit.Cast`는 빈 격자에서 실을 *짜서* 목표 패턴을 *만드는* 건설 방향. `Slice.Chain`(절단), `Net.Weave`(그래프 평면화)와 완전히 다른 메커닉.

### 2. `Fog.Build` — 안개 전쟁 기억력 턴제 전략

- 방문 기억: 칸을 방문하면 그 순간의 정보가 반투명 표시 (이미 변했을 수 있음)

- AI 적: 다른 탐색 패턴(공격적/방어적/탐색적) 3종
- 목표: 안개 속 적 거점 파괴 (위치는 처음에 힌트 1회만 공개)
- 잘못된 기억에 기반한 오공격이 역전의 묘미
- 로그라이트 모드: 랜덤 맵, 스테이지 클리어 시 유닛 업그레이드

**기술**: WPF Canvas 안개 오버레이 (Per-cell 투명도 애니메이션), A* 병합 탐색 AI, 방문 기록 `Dictionary<Point, TileState>`, 시야 계산 Bresenham's Line
**매력**: "저 칸에 뭐가 있었는지 기억나는데..." 라는 긴장감. 정보 우위의 쾌감과 잘못된 기억으로 인한 역전. 짧은 판 (5~10분) 반복.
**차별화**: `Hex.Storm`(마이크로 RTS, 정보 공개), `Tactics.Rush`(엄폐·플랭킹 전술), `Rogue.Tile`(로그라이크 던전)과 다름. `Fog.Build`는 *기억력 기반 정보 전쟁*이 핵심.

### 3. `Glass.Drop` — 유리 낙하 각도 제어 파괴 퍼즐

**핵심 메커닉**: 다양한 두께·경도의 유리판을 지정 높이에서 낙하시켜 정확한 각도·힘으로 깨뜨리는 물리 퍼즐이다. 유리 내부 응력 분포와 충격 방향에 따라 균열 패턴이 달라지며, 아래에 갇힌 물체를 해방하거나 타겟 구역에 유리 파편을 정확히 배치해야 한다.

**게임플레이**:
- 유리판 종류: 두꺼운 유리(천천히 균열), 강화유리(한 번에 산산조각), 색유리(목표 구역에 파편 배치)
- 낙하 높이·각도·회전 속도를 조절해 충격 지점 및 균열 방향 제어

- 유리 두께 시각화: 두꺼울수록 불투명, 얇을수록 반투명
- 연속 낙하: 깨진 유리 파편이 쌓이면 다음 낙하에 영향
- 파편 청소 모드: 파편 없이 물체만 통과시키기
- 점수: 파편 수 최소화 Perfect / 지정 구역 적중 Precision

**매력**: 유리 깨지는 쾌감 + 파편 경로 예측의 정밀함. ASMR 요소. "한 번만 더"를 부르는 타격감.
**차별화**: `Fracture.Fall`(Voronoi 파단 지형 *달리기* 게임), `Jenga.Pull`(블록 *제거*), `Crush.Box`(압착 샌드박스). `Glass.Drop`은 *유리 낙하 각도로 파단 패턴 정밀 제어*가 독자 메커닉.

### 4. `Thermal.Run` — 열전도 시뮬레이션 퍼즐

- 배치 수 제한 (최소 재료로 목표 달성할수록 높은 점수)

### 5. `Stitch.Grid` — 자수 격자 X자 스티치 매치 퍼즐

**장르**: Puzzle / Casual / Creative
**핵심 메커닉**: N×N 격자에 X자(크로스 스티치) 형태의 컬러 스티치를 배치하고, 가로·세로·대각선 방향으로 같은 색 스티치가 연속 N개 이상 이어지면 폭발·제거되는 매치 퍼즐이다. 테트리스처럼 위에서 스티치가 쌓이며, X자의 양쪽 대각선을 서로 다른 색으로 칠할 수 있어 한 칸으로 두 방향 매칭이 가능하다.

- 특수 조각: 무지개(모든 색 매칭), 폭발(주변 3×3 제거), 고정(이동 불가)
- 스코어: 연쇄(Chain) 폭발 콤보 × 배율
- 생존 모드 (속도 증가), 퍼즐 모드 (주어진 조각 배치로 목표 클리어)
- 배경이 실제 자수 직물 텍스처로 완성 시 아트워크 공개

**기술**: WPF Canvas 대각선 Path 그리기, 4방향(↘↗→↓) 연속 매칭 BFS, 조각 낙하 물리(중력 가속 + 인접 지지 검사), 연쇄 폭발 애니메이션
**매력**: X자 조각의 2색 동시 매칭 전략이 퍼즐 깊이를 높임. 자수 아트워크 완성 수집 요소. 눈이 편한 부드러운 비주얼.
**차별화**: `Hue.Flow`(파이프 색 연결), `Flip.Grid`(타일 뒤집기), `Chain.Blast`(색 구슬 발사). `Stitch.Grid`는 *X자 2색 조각의 대각선 매칭*이 독자 메커닉.

### 6. `Orbit.Raid` — 관성만으로 적 궤도에 진입하는 중력 침투 게임

**장르**: Puzzle / Physics / Precision
**핵심 메커닉**: 플레이어 자신이 발사체가 되어 행성·위성의 중력만으로 가속·방향 전환하여 적 위성의 궤도에 정확히 진입하거나 충돌하는 물리 퍼즐이다. 연료 없이 중력 어시스트(스윙바이) 기동만으로 목표에 도달해야 한다.

**게임플레이**:
- 발사 각도·초기 속도 지정 → 이후 중력만으로 궤도 결정
- 궤도 예측선 표시 (실시간 RK4 수치 적분 궤도 시뮬)
- 목표: 적 위성과 동일 궤도(±허용오차) 진입, 또는 정확한 충돌
- 스윙바이 포인트: 행성 인근 통과 시 가속/방향 변경 효과 활용
- 중력 지뢰: 특정 구역에 숨겨진 소행성 중력 → 예측 방해
- 발사 횟수 제한 (최소 시도로 완수할수록 높은 점수)
- 샌드박스 모드: 행성 배치 자유 설정, 궤도 탐구

**기술**: RK4 수치 적분 (다체 중력, n-body simulation 단순화), WPF Canvas 궤도 예측 Bezier 근사, 충돌 검출(원형 SOI 진입)
**매력**: "이 각도로 쏘면 저 행성을 돌아서..." 계획 단계가 이미 재밌음. 계획대로 이루어지는 쾌감. 케플러 궤도의 아름다운 시각화.
**차별화**: `Orbit.Craft`(궤도 설계 퍼즐, 위성 배치), `Zero.G`(스러스터 Δv 제어), `Warp.Drift`(중력 렌즈 탄환). `Orbit.Raid`는 *플레이어 자신이 발사체, 중력 어시스트만으로 침투*하는 고유 메커닉.

### 7. `Sand.Storm` — 바람 방향 예측 모래폭풍 생존 퍼즐

**장르**: Puzzle / Strategy / Survival
**핵심 메커닉**: 매 턴 바람 방향이 바뀌며 모래가 격자를 덮어오는 환경에서, 모래 방벽을 건설하고 이동 경로를 확보하며 목표 지점에 도달하는 생존 퍼즐이다. 바람 방향 예보(다음 2턴)를 보고 최적 방벽 위치를 선점하는 전략이 핵심이다.

**게임플레이**:
- N×N 격자, 매 턴 바람 방향(북/남/동/서) 변화 + 다음 2턴 예보 표시
- 바람이 부는 방향에서 모래가 날아와 빈 칸을 채움
- 모래 쌓인 칸은 이동 불가 → 출구가 막히기 전에 탈출
- 행동: 이동(1칸) / 방벽 건설(현재 칸 주변, 내구도 있음) / 모래 치우기(1턴 소모)
- 방벽은 모래를 막지만 3~5회 후 무너짐
- 특수 칸: 오아시스(모래 제거), 폭풍 핵(주변 5×5 즉시 모래)
- 목표 다양화: 탈출 / 특정 물체 수집 / NPC 구출

**기술**: 파티클 시스템 (바람 모래 애니메이션), 셀룰러 오토마타 (모래 확산 규칙), WPF Canvas 그리드 렌더링, BFS 경로 검증 (탈출 가능성 체크)
**매력**: 바람 예보를 보며 "여기에 방벽을 쌓아야겠다..." 계획의 쾌감. 모래에 점점 잠겨가는 긴장감. 턴 당 선택의 무게.
**차별화**: `Sand.Fall`(낙하 샌드박스), `Seed.Fall`(씨앗 분산 물리), `Magma.Flow`(용암 점성). `Sand.Storm`은 *동적 바람 예보 기반 방벽 건설 생존*이 고유 메커닉.

### 8. `Ink.Drop` — 기울기 제어 물방울 합류 색 혼합 퍼즐

**장르**: Puzzle / Physics / Creative
**핵심 메커닉**: 격자 기울기를 상하좌우로 조절해 여러 색 물방울을 굴리고, 동색 방울은 합류(크기 증가)·이색 방울은 새 색 혼합(RGB 가산 혼합)하여 목표 색상의 방울을 만드는 물리 퍼즐이다.

**게임플레이**:
- 화면 기울기 조작(WASD): 모든 방울이 동시에 같은 방향으로 굴러감
- 동색 충돌 → 합류 (반지름 ∝ 면적의 합)
- 이색 충돌 → 혼합 (빨+파=보라, 빨+노=주황 등 5×5 혼합 테이블)
- 목표: 특정 색상+크기의 방울을 목표 구역에 도달
- 장애물: 못(방울 고정), 분기점(두 방울로 분열), 흡수재(방울 소멸)
- 제한 기울기 횟수 (최소 기울기로 목표 달성)
- 특수 방울: 색 고정(합류해도 색 변화 없음), 폭발(충돌 시 주변 분산)
- 아트 모드: 자유롭게 혼합하며 그라데이션 만들기

**기술**: Verlet 원형 입자 충돌 (Impulse Resolution), 합류 시 질량 보존, 색 혼합 테이블 룩업, WPF Canvas 반투명 원 렌더링 (Alpha blend)
**매력**: 방울이 굴러다니는 통통한 시각적 재미. 색 혼합 퍼즐이 화학·미술 직관과 연결. 목표 색 만들기의 퍼즐 깊이.
**차별화**: `Ink.Spread`(BFS 잉크 전파 전략), `Fluid.Paint`(자유 유체 페인팅), `Hue.Flow`(파이프 색 연결). `Ink.Drop`은 *기울기 제어 + 방울 물리 + 색 혼합*의 삼중 메커닉.

### 9. `Volt.Race` — 전기 회로 트랙 레이싱

**장르**: Racing / Puzzle / Arcade
**핵심 메커닉**: 전기 회로 도면이 레이싱 트랙이다. 직렬·병렬 분기, 저항(속도 감소 구간), 콘덴서(에너지 충전·부스트), 인덕터(가속 관성) 등 전기 소자들이 트랙 기믹으로 작동하며, 경로 선택 전략과 에너지 관리가 승부를 결정한다.

**게임플레이**:
- 회로 도면 스타일 트랙 (도선=도로, 소자=특수 구간)
- 저항: 속도 감소 구간 (저항값 ∝ 감속 비율)
- 병렬 분기: 두 경로 중 선택 (저항 낮은 경로=빠르지만 짧음 / 높은 경로=느리지만 지름길)
- 콘덴서: 에너지 충전 구간 → 이후 부스트 방전 활성화
- 인덕터: 진입 느림·진출 빠름 (관성 가속)
- 단락 구간: 매우 빠르지만 과열 위험 (N회 연속 사용 시 탈락)
- 3랩 구성, 랩마다 소자 상태 변화 (콘덴서 방전/재충전)
- AI 레이서: 전기공학 최적 경로 알고리즘 기반

**기술**: 회로 그래프 파싱 (노드-간선 저항 모델), 레이싱 물리 (속도=전압, 가속=전류 유사 모델), WPF Canvas 회로 도면 렌더링 (격자선 + 소자 기호)
**매력**: "이 경로가 더 빠른 게 맞는데 왜 졌지?" 전기 법칙 역설적 결과. 회로 도면의 친숙한 비주얼에 레이싱 긴장감 조합. 전기공학 학습 요소.
**차별화**: `Volt.Chain`(회로 퍼즐 스위치 설계), `Circuit.Break`(회로 디버깅), `Chip.Logic`(논리 게이트 빌더). `Volt.Race`는 *회로 소자를 레이싱 트랙 기믹*으로 전환한 독자 장르.

### 10. `Root.War` — 지하 뿌리 성장 영토 대전

**장르**: Strategy / Simulation / 2P or vs AI
**핵심 메커닉**: 두 플레이어(또는 AI)가 지하 단면 격자에서 뿌리를 성장시켜 영토를 점령하는 실시간 전략 게임이다. 뿌리는 깊이·방향·성장 속도를 조절할 수 있으며, 상대 뿌리와 충돌 시 영양분 농도 비교로 승패가 결정된다.

**게임플레이**:
- 지하 단면 2D 격자, 각 플레이어 지상 한쪽에서 시작
- 뿌리 성장 방향 지정 (상하좌우 대각선), 한 턴에 N칸 성장
- 토양 영양분 분포: 영양분 높은 칸은 성장 속도 증가, 수분 칸은 일시 가속
- 충돌 해결: 충돌 칸의 영양분 농도 합산 → 높은 쪽 흡수 (낮은 쪽 뿌리 절단)
- 특수 성장: 굵은 뿌리(범위↑, 영양분↑, 성장↓), 얇은 뿌리(빠른 탐색), 독 뿌리(인접 상대 약화)
- 지진 이벤트: 랜덤 구역 영양분 재배치
- 승리 조건: 상대 뿌리 전체 절단 / 지정 깊이 도달 / 영토 70% 점령

**기술**: 격자 기반 성장 자동화 (매 턴 성장 큐 처리), Dijkstra 영양분 최적 경로 AI, WPF Canvas 뿌리 bezier 렌더링 (유기적 곡선), 파티클 영양분 흡수 애니메이션
**매력**: 보이지 않는 지하에서 조용히 뻗어가는 뿌리 전략. 충돌 순간의 긴장감. 2P 로컬 대전의 흥분.
**차별화**: `Root.Spread`(솔로 플레이 뿌리 방향 지정 시뮬), `Spore.Net`(슬라임 몰드 단일 군집), `Bloom.Craft`(세포 확산 vs AI). `Root.War`는 *대전 상대(로컬 2P/AI)와 뿌리 충돌·흡수 전략*이 핵심.
