### 1. `Contact.Vault` — 완전 오프라인 개인 CRM / 연락처 관리자

**카테고리**: Productivity / Personal
**핵심 기능**: 인터넷 연결 없이 동작하는 로컬 개인 연락처 관리자. vCard/CSV 가져오기·내보내기, 생일/기념일 Toast 알림, 관계 메모 및 첨부 파일 지원. Outlook이나 클라우드 동기화 없이 인맥 정보를 체계적으로 관리한다.

**상세 기능**:
- vCard (.vcf) / CSV 가져오기·내보내기, 중복 감지 병합
- 이름·회사·역할·이메일·전화·SNS·메모 필드 + 사진 첨부
- 생일·기념일 자동 Toast 알림 (7일 전 / 당일)
- 태그·그룹 분류 + FTS5 전문 검색 (이름, 회사, 메모 통합)
- 관계 연결 맵 (A↔B 관계 시각화, 소속 그룹 그래프)
- SQLite 단일 파일 저장, AES-256-GCM 암호화 잠금 옵션
- 최근 연락일 자동 기록 + 연락 주기 리마인더 설정

**기술 스택**:
- WinForms / WPF (.NET 8)
- SQLite + FTS5 (Microsoft.Data.Sqlite)
- AES-256-GCM (System.Security.Cryptography)
- vCard 파서 (MixERP.Net.VCards)
- SkiaSharp 관계 그래프 렌더링

**수익 모델**: 무료 오픈소스
**차별화**: Family.Pin(LAN 공유 메모보드), Habit.Chain(습관 추적)과 달리 개인 인맥 데이터 관리에 특화. Outlook 없이 완전 오프라인 vCard 생태계.

### 3. `Deploy.Diff` — 배포 환경 설정 파일 차이 비교기

**카테고리**: Dev / Tools
**핵심 기능**: dev / staging / production 간 appsettings.json, .env, YAML 설정 파일을 나란히 펼쳐 차이를 한눈에 파악한다. 민감 값 자동 마스킹, 환경별 변경 이력 타임라인, 스냅샷 저장으로 배포 전 환경 설정 검증에 특화.

**상세 기능**:
- JSON / YAML / TOML / .env / .ini / XML 설정 파일 지원
- 최대 5개 환경 병렬 diff 뷰 (dev / staging / prod / local / test)
- 키 단위 차이 강조 + 추가·삭제·변경 색상 구분
- 민감 키 패턴(password, secret, key, token) 자동 마스킹 + 화이트리스트
- 환경별 스냅샷 저장 + 날짜별 히스토리 타임라인
- 설정 키 의존성 트리 (A 키가 B 키를 참조 시 그래프 표시)
- 누락 키 감지 (dev에 있고 prod에 없는 키 경고)
- Markdown 리포트 내보내기 (배포 체크리스트용)

**기술 스택**:
- WPF (.NET 8), DiffPlex 라이브러리
- YamlDotNet, Tomlyn, DotNetEnv
- SkiaSharp 키 의존성 그래프

**수익 모델**: 무료
**차별화**: Deep.Diff(일반 텍스트·이미지·HEX·폴더 비교), Json.Craft(포맷팅+변환)과 달리 배포 환경 설정 파일 전용. 병렬 5열 diff + 누락 키 감지 + 민감 값 마스킹 특화.

**수익 모델**: 무료

### 8. `Diff.Merge` — 3-way 파일 병합 도구

**카테고리**: Dev / Tools
**핵심 기능**: 베이스·로컬·원격 세 파일을 나란히 비교하고 충돌 행을 인터랙티브하게 해결하는 3-way merge 전문 도구. Deep.Diff는 2-way 비교기이지만 Diff.Merge는 Git 병합 충돌 해결을 위한 3-column 레이아웃.

**상세 기능**:
- 베이스 / 로컬(내 변경) / 원격(상대 변경) 3열 병렬 뷰
- 충돌 행 자동 감지 + 인터랙티브 해결 (로컬 선택 / 원격 선택 / 양쪽 병합)
- 줄 단위 cherry-pick: 원격 변경 중 일부만 선택 적용
- Non-conflict 변경 자동 머지, conflict 만 수동 해결
- 결과 파일 미리보기 + 저장
- git merge-tool 외부 도구로 등록 (git config mergetool.difmerge)
- 폴더 diff: 디렉토리 레벨 3-way 비교 (추가·삭제·충돌 파일 목록)
- 인코딩 자동 감지 (UTF-8/UTF-16/CP949)

**기술 스택**:
- WPF (.NET 8), DiffPlex (Myers diff 알고리즘)
- Git merge-tool 인터페이스 (`git mergetool`)

**수익 모델**: 무료
**차별화**: Deep.Diff(2-way 텍스트·이미지·HEX·폴더 비교기)와 달리 3-way merge 특화. git mergetool 통합으로 실제 Git 충돌 해결 워크플로우 지원.

### 8. `Archery.Cast` — 궁술 정밀 조준 + 물리 게임

**장르**: Sports / Skill / Puzzle
**핵심 메커닉**: 활과 화살의 탄도를 계산해 목표물을 정확히 맞히는 궁술 게임. 풍향·중력·거리에 따라 조준선을 보정하는 물리 기반 사격 퍼즐. Arc.Blast(구조물 파괴 포물선)과 달리 단일 타깃 정밀 조준이 핵심.

**게임플레이**:
- 마우스 드래그 방향·힘 조준 → 릴리즈로 발사
- 중력(거리 비례 낙차), 풍향·풍속(화살 편류), 거리 감쇠
- 정조준 모드 (숨 참기: 화면 안정화 3초 → 점진 흔들림 증가)
- 과녁 모드: X링/10링/9링/8링 점수 + 화살 군집 분포 통계
- 퍼즐 모드: 밧줄 절단·풍선 연쇄 사격·거울 반사 타깃 맞히기
- 특수 화살: 불화살(연쇄 점화), 폭발 화살, 분열 화살(3갈래)
- 스테이지별 조건: 최소 화살 수·제한 시간·이동 타깃
- 바람 시각화 오버레이 (화살표 + 풍속 수치)

**기술**: Verlet Integration 포물선 탄도, Magnus 효과(회전에 의한 편류), 바람 벡터장, 폴리곤 충돌감지(SAT)
**매력**: "딱 맞겠다" 싶은 조준 후 발사하는 짜릿함, 바람을 읽고 보정한 화살이 정중앙 X링을 꿰뚫을 때의 쾌감. 특수 화살 연쇄 반응의 장관.
**차별화**: Arc.Blast(포물선+구조물 파괴), Throw.Stars(무기별 비행 물리), Crack.Shot(턴 기반 저격 퍼즐)과 달리 Archery.Cast는 활·화살 특유의 낙차+풍향+숨참기 조합 + 과녁 점수 스포츠.

### 9. `Hex.Board` — 추상 전략 보드 게임 컬렉션

**장르**: Strategy / Abstract / Classic Board Game
**핵심 메커닉**: 오셀로(리버시), Hex, Y-게임, Connect6 등 수학적 추상 전략 보드 게임 컬렉션. MCTS AI 상대, 기보 저장·재생, 온라인 대전 없이 완전 오프라인으로 클래식 보드 게임의 깊이를 즐긴다.

**게임플레이**:
- **오셀로/리버시**: 8×8, 뒤집기 전략, MCTS/MinMax AI
- **Hex**: 11×11 육각 격자, 자기 색 돌로 좌→우 연결 (AI 강도 5단계)
- **Connect6**: 19×19, 6연속 승리, MCTS AI
- **Gomoku(오목)**: 15×15, 금수 규칙 (3-3·4-4·장목)
- **Y-Game**: 삼각형 격자, 세 모서리 연결 전략
- 공통: 2P 로컬 대전, 컴퓨터 AI 5단계, 기보 저장(SGF/JSON), 기보 재생(배속 조절)
- 분석 모드: 착점 별 AI 평가 점수 표시, 최선 수 제안
- 사용자 정의 보드 크기 (Hex 9~19, 오셀로 6·8·10)

**기술**: Monte Carlo Tree Search (UCB1 선택, 랜덤 시뮬레이션, 역전파), Alpha-Beta Pruning (오셀로/오목), SGF 기보 파일 파서
**매력**: 단순한 규칙에서 수 백 가지 전략이 파생되는 깊이. Hex의 Bridging 전략, 오셀로의 모서리 싸움. AI와 대결하며 점점 강해지는 성취감. 기보를 복기하며 실수를 발견하는 재미.
**차별화**: Card.Solo(솔리테어 패시엔스), Neon.Card(로그라이크 덱빌딩), Hex.Storm(마이크로 RTS)과 달리 Hex.Board는 수학적 추상 전략 보드 게임. 운 요소 0%, 순수 전략 대결.

### 10. `Maze.Iso` — 아이소메트릭 3D 큐브 회전 미로 탐색

**장르**: Puzzle / 3D / Spatial
**핵심 메커닉**: 아이소메트릭 뷰의 3D 큐브 미로를 탐험한다. 큐브를 회전시키면 중력 방향이 바뀌어 기존에 벽이었던 면이 바닥이 되고, 새로운 경로가 생성된다. 3D 공간 감각과 큐브 회전 계획이 핵심.

**게임플레이**:
- 4×4×4 ~ 8×8×8 큐브 미로, 아이소메트릭 2.5D 렌더링
- WASD 이동 + Q/E로 큐브 90° 회전 (중력 방향 전환)
- 회전 시 현재 위치한 면의 아이템·키·문이 중력에 의해 재배치
- 잠긴 문은 같은 면의 열쇠로만 해제 (큐브 면별 색상 코딩)
- 출구는 큐브의 특정 면에 위치 → 그 면이 "바닥"이 되어야 도달 가능
- 투명 X-Ray 모드: 큐브 내부 구조를 일정 시간 확인
- 절차적 큐브 미로 생성 (DFS 스패닝 트리, 면별 연결성 보장)
- 50+ 큐레이션 레벨 + 무한 랜덤 미로, 이동 수 최소화 도전

**기술**: 아이소메트릭 투영 변환 (2D 렌더링), 3D 격자 회전 행렬(Quaternion), DFS 3D 미로 생성, 중력 벡터 기반 이동 방향 계산
**매력**: 큐브를 돌리면 3D 공간이 완전히 재구성되는 경이로움. "이 방향으로 돌리면 이 통로가 열리겠다"는 공간 계획의 쾌감. 처음엔 방향 감각을 잃다가 점점 3D를 머릿속에 그리는 능력이 생기는 성장감.
**차별화**: Maze.Dread(2D 시야 제한 추격 미로), Echo.Maze(소나 항법 미로), Frame.Jump(만화 패널 메타 플랫포머)과 달리 Maze.Iso는 큐브 회전으로 중력 방향이 바뀌는 3D 아이소메트릭 공간 퍼즐. 유일한 3D 공간 회전 메커닉.
