### 1. `AI.Recap` — 회의 트랜스크립트 → AI 요약기
> **카테고리**: AI / Productivity
> **수익 잠재력**: ★★★★★ (B2B SaaS, 팀 구독 모델)

**핵심 기능**
- Zoom / Teams / Google Meet / Naver Works 등 자동 생성 트랜스크립트(TXT/DOCX/VTT) 붙여넣기 또는 드래그&드롭 로드
- Claude API 호출 → 세 가지 출력 동시 생성

  - **Action Items** (담당자·기한 자동 추출, 표 형태)
  - **Key Decisions** (결정 사항 bullet list)

**기술 스택**
- WPF, net8.0-windows, Claude API (Anthropic SDK), RichTextBox

**왜 수익이 나는가**

### 3. `Quick.Calc` — 개발자 전용 프로그래머 계산기
> **카테고리**: Dev / Tools
> **수익 잠재력**: ★★★☆☆ (스토어 판매, 구독 없는 one-time)

**핵심 기능**
- 10진수 / 16진수 / 8진수 / 2진수 **실시간 동시 표시** (하나 수정 → 나머지 자동 갱신)
- 비트 연산: AND / OR / XOR / NOT / SHL / SHR — 버튼 클릭 또는 연산자 입력
- **IEEE 754** 단정밀도(32bit) / 배정밀도(64bit) float 구조 시각화
  - 부호 1bit / 지수 8bit / 가수 23bit 색상 구분 표시
  - 비트 직접 클릭으로 값 조작
- **2의 보수** / **부호 확장** 자동 계산
- 비트 단위 레이아웃: 64비트를 가로 토글 버튼으로 시각화
- 계산 히스토리 (hex/dec/bin 표시, 클릭으로 재사용)
- 전역 단축키 팝업 (Win+K 기본)

**기술 스택**
- WPF, net8.0-windows

**왜 차별화되는가**
- Windows 내장 계산기의 프로그래머 모드 대비: IEEE 754 시각화, float 비트 편집, 전역 팝업 없음
- 임베디드 / 드라이버 / 저수준 개발자가 일상적으로 필요로 하는 기능 집중

### 4. `Color.Blind` — 실시간 화면 색맹 시뮬레이션 오버레이
> **카테고리**: Dev / Tools / Accessibility
> **수익 잠재력**: ★★★★☆ (디자인 팀 B2B, 접근성 컴플라이언스)

**핵심 기능**
- 화면 전체 또는 선택 영역에 색맹 필터 실시간 오버레이 (DirectX/DXGI 캡처)
- 지원 색맹 유형:
  - 제1 색맹(Protanopia), 제2 색맹(Deuteranopia), 제3 색맹(Tritanopia)
  - 전색맹(Achromatopsia), 청황색맹(Tritanomaly), 단색맹
- **분할 뷰**: 화면을 좌우로 나눠 원본 ↔ 색맹 시뮬 동시 비교
- **비교 스크린샷**: 현재 시뮬 결과를 PNG로 저장 (보고서 첨부용)
- 필터 강도 슬라이더 (0~100%, 부분적 색각 이상 시뮬)
- 트레이 상주, 전역 단축키로 오버레이 토글

**기술 스택**
- WPF, net8.0-windows, Windows.Graphics.Capture API (DXGI), SharpDX 또는 D3D11 interop

**왜 수익이 나는가**
- 웹/앱 WCAG 2.1 접근성 준수 법제화 확대 (EU, 한국)
- 디자이너·QA 팀 일상 도구로 채택 가능
- Access.Check(색상 대비율 계산기)와 **다름**: 라이브 화면 전체 시뮬이 핵심

### 5. `Palette.Gen` — 이미지 → 색상 팔레트 추출 + CSS/디자인 시스템 출력기
> **카테고리**: Dev / Tools / Creative
> **수익 잠재력**: ★★★★☆ (Figma 플러그인 확장, 디자이너 워크플로우)

**핵심 기능**
- 이미지(JPEG/PNG/WebP) 드래그&드롭 → 지배적 색상 자동 추출
  - k-Means 군집화 (k=5~12 사용자 설정)
  - Lab 색공간에서 계산 → 지각적으로 고른 팔레트
- 팔레트 편집: 색상 추가·삭제·순서 변경, 수동 피커 보정
- **여러 출력 형식** 원클릭:
  - CSS 변수 (`--color-primary: #1A2B3C`)
  - Tailwind config 오브젝트
  - SCSS `$` 변수
  - 16진수 목록 / RGB / HSL
  - iOS Swift `UIColor` 코드
  - Android XML colors
- 팔레트 라이브러리 저장 (로컬 JSON)
- 두 이미지 간 팔레트 블렌딩 기능

- WPF, net8.0-windows

**왜 차별화되는가**

- 디자인 → 개발 연결 고리 (Tailwind/Swift/Android 동시 출력)

### 7. `Burn.Rate` — 노트북 배터리 건강도·방전 분석기
> **카테고리**: System / Tray
> **수익 잠재력**: ★★★☆☆ (노트북 사용자, 기업 IT 자산관리)

**핵심 기능**
- 배터리 설계 용량 vs 실제 풀충전 용량 (wh) 실시간 표시
- **충전 사이클 횟수** + 예상 수명 잔여 계산
- 방전 곡선 그래프 (100% → 0% 까지 실시간 플롯)
- **배터리 교체 권장**: 설계 용량 대비 현재 용량이 80% 미만 시 Toast 알림
- 세션별 방전량 로그 (날짜·앱별 소비량 분석)
- AC 어댑터 연결/분리 시 자동 트리거 알림
- 트레이 아이콘에 % + 예상 잔여 시간 표시 (OS 기본 대비 더 정확한 알고리즘)
- 충전 최적화 팁 (20%~80% 유지 권장 알림)

**기술 스택**
- WPF, net8.0-windows, WMI (`Win32_Battery`, `BatteryStatus`), Windows.Devices.Power

**왜 차별화되는가**
- Tray.Stats(CPU/RAM/GPU 통합 성능)와 다름 — 배터리 특화 심층 분석 + 교체 판단 도우미
- BatteryInfoView(무료 도구) 대비: 실시간 방전 곡선, 교체 권장 알림, 모던 UI

### 9. `Snap.Cast` — 화면 캡처 → 구조화된 마크다운 변환기
> **카테고리**: Productivity / Capture / AI
> **수익 잠재력**: ★★★★☆ (기술 문서 작성자, 개발팀, B2B 문서화 도구)

- 배치 생성: CSV 파일로 링크 목록 일괄 생성

**기술 스택**
- WPF, net8.0-windows, P/Invoke (CreateSymbolicLink, CreateHardLink), NtQueryInformationFile

**왜 차별화되는가**
- Windows에서 심링크 생성은 `mklink` 명령어 + 관리자 터미널 필요 → 진입장벽 높음
- 개발 환경 설정, 모드 파일 링크, 프로젝트 공유 폴더 등에 필수
- Batch.Rename(이름 변경), Disk.Lens(크기 시각화)와 완전히 다른 영역
