### 1. `Regex.Lab` — 정규식 실시간 테스터 & NFA 상태 다이어그램

**카테고리**: Applications / Development / Inspector
**핵심 기능**: 정규식 패턴을 입력하면 테스트 문자열에 실시간 하이라이팅을 적용하고, 캡처 그룹 트리와 NFA/DFA 상태 전이 다이어그램을 시각화한다. 개발자가 복잡한 패턴을 디버깅할 때 "왜 이 문자열이 매치되는지" 를 즉각 파악할 수 있게 해준다.

**상세 기능**:
- 패턴 입력 → 테스트 문자열 실시간 매치 하이라이팅 (Named/Unnamed 그룹 색상 구분)
- 캡처 그룹 트리뷰: 그룹 번호·이름·매치 범위 인라인 표시
- NFA 상태 전이 다이어그램 Canvas 렌더링 (WPF `StreamGeometry`)
- C# / Python / JavaScript / Go 코드 스니펫 자동 생성 (클립보드 복사)
- 정규식 라이브러리 저장 (자주 쓰는 이메일·URL·날짜 패턴 즐겨찾기)
- 플래그 토글 UI (IgnoreCase, Multiline, Singleline, Compiled 등)
- 성능 측정: 매치 소요 ms, Backtracking count 표시

**기술 스택**:
- `System.Text.RegularExpressions.Regex` (C# 빌트인)
- WPF RichTextBox custom highlighting (`TextRange` + `Foreground`)
- NFA 그래프: 직접 파싱 후 WPF Canvas `StreamGeometry` 렌더링
- SQLite (`Microsoft.Data.Sqlite`) 패턴 라이브러리 영속화

**수익 모델**: 무료
**차별화**: Log.Lens·SQL.Lens 등 기존 분석기와 달리 정규식 특화. 상태 다이어그램 시각화가 핵심 차별점.

### 2. `JSON.Tree` — JSON / YAML / TOML 구조 탐색기 & JSONPath 쿼리

**카테고리**: Applications / Development / Inspector
**핵심 기능**: 대용량 JSON·YAML·TOML 파일을 빠른 트리뷰로 탐색하고, JSONPath / jq 스타일 쿼리로 원하는 값만 추출한다. 두 파일의 구조적 차이를 나란히 Diff 해주어 API 응답 비교나 설정 변경 추적에 활용한다.

**상세 기능**:
- 파일 열기 / 드래그 앤 드롭 / 클립보드 붙여넣기 지원
- WPF `TreeView` 가상화(VirtualizingStackPanel) 로 수백 MB JSON 처리
- JSONPath 쿼리 바 — 실시간 필터링 + 결과 수 표시
- 두 파일 Side-by-Side 구조 Diff (추가/삭제/변경 색상 구분)
- 값 유형 아이콘 (string/number/boolean/null/array/object)
- JSON → YAML → TOML 상호 변환 내보내기
- 노드 경로 클립보드 복사 (예: `$.users[0].email`)

**기술 스택**:
- `System.Text.Json` (JSON 파싱)
- `YamlDotNet` (YAML 파싱/직렬화)
- `Tomlyn` (TOML 파싱)
- WPF `TreeView` + 가상화

**수익 모델**: 무료
**차별화**: DB.Peek(스키마 탐색)·Table.Craft(CSV 편집)·SQL.Lens(쿼리 계획)과 달리 JSON 계층 구조 전문. Diff 기능 내장.

### 3. `Color.Token` — 디자인 시스템 색상 토큰 에디터 & WCAG 대비 감사

**카테고리**: Applications / Tools.Utility
**핵심 기능**: CSS Custom Properties·SCSS 변수·JSON 토큰 파일을 열어 라이트/다크 팔레트를 시각적으로 편집하고, WCAG 2.1 텍스트 대비(명도비 4.5:1 / 3:1) 통과 여부를 자동 검사한다. 접근성을 고려한 디자인 시스템 구축의 핵심 도구.

**상세 기능**:
- CSS / SCSS / JSON (Figma Token 포맷) 파일 로드 & 저장
- 색상 팔레트 그리드 뷰 — 클릭하면 컬러피커 팝업
- 라이트/다크 모드 토글 → 토큰 별칭 매핑 시각화
- WCAG 2.1 AA/AAA 명도 대비 전체 조합 표 (통과/실패 배지)
- 색상 변경 시 의존 토큰 연쇄 미리보기
- HEX / RGB / HSL / OKLCH 형식 상호 변환
- 변경 내역 비교(Diff) 후 파일 저장

**기술 스택**:
- WCAG 명도 계산: `(0.2126R + 0.7152G + 0.0722B)` sRGB 공식 직접 구현
- `System.Text.RegularExpressions` CSS 변수 파싱
- `System.Text.Json` JSON 토큰 파싱
- WPF `Popup` 컬러피커 컨트롤

**수익 모델**: 무료
**차별화**: Section 6의 "색상 피커/팔레트" 그룹(Color.Drop, Palette.Cast)은 화면 추출·팔레트 생성 도구. Color.Token은 CSS 파일 직접 편집 + 접근성 감사 특화.

### 4. `Git.Blame` — Git Blame 열 히트맵 & 기여자 통계 드릴다운

**카테고리**: Applications / Development / Analyzer
**핵심 기능**: 소스 파일의 각 줄에 Git blame 정보(커밋·날짜·작성자)를 열 형태로 표시하고, 날짜 기준 열 히트맵으로 "오래된 코드 vs 최신 코드" 를 한눈에 보여준다. 기여자별 드릴다운으로 특정 개발자가 담당한 코드 영역을 탐색할 수 있다.

**상세 기능**:
- 파일 열기 → `git blame` CLI 결과 파싱 & 인라인 어노테이션 표시
- 날짜 기반 열 히트맵 (오래될수록 어두운 파란색, 최신일수록 밝은 노란색)
- 기여자 필터 → 해당 기여자 줄만 하이라이트
- 커밋 SHA 클릭 → `git show` 커밋 상세 팝업
- 디렉터리 단위 blame 요약 테이블 (파일별 최근 수정자·날짜)
- 시간대별 애니메이션 (슬라이더로 날짜 이동 → 블레임 변화 시각화)

**기술 스택**:
- `System.Diagnostics.Process` — `git blame --line-porcelain` 실행
- WPF `FlowDocumentScrollViewer` 또는 `ItemsControl` 줄별 렌더링
- 히트맵 색상 보간 (`Color.FromRgb` HSL 계산)

**수익 모델**: 무료
**차별화**: Git.Stats(구현완료, 커밋 통계/차트)와 달리 파일·줄 단위 blame 특화. 시간 애니메이션이 핵심 차별점.

### 5. `MIME.Peek` — 파일 MIME 타입 & 매직 바이트 검사기

**카테고리**: Applications / Files / Inspector
**핵심 기능**: 파일을 드래그하면 확장자가 아닌 매직 바이트(파일 시그니처)로 실제 MIME 타입을 판별한다. 위장된 악성 파일 탐지, 손상된 파일 진단, 미디어 파일 메타데이터 확인 등에 활용한다.

**상세 기능**:
- 드래그 앤 드롭 → 매직 바이트 즉시 분석 (200+ 시그니처 DB)
- 헥스 헤더 뷰 (첫 256 바이트) — 시그니처 바이트 하이라이팅
- 확장자 vs 실제 MIME 불일치 시 경고 배지
- EXIF / ID3 / ICC 프로파일 등 주요 메타데이터 표시
- 배치 처리: 폴더 통째로 분석 → CSV 리포트 내보내기
- 압축 파일 내부 파일 목록도 재귀 검사

**기술 스택**:
- 매직 바이트 DB: 직접 구현 (Dictionary<byte[], string>)
- `MetadataExtractor` NuGet (EXIF/IPTC/XMP)
- `System.IO.Compression` (ZIP 내부 탐색)

**수익 모델**: 무료
**차별화**: Hex.Peek(헥스 에디터)는 바이트 편집 도구. MIME.Peek는 타입 판별·보안 검사 특화. Hash.Check(해시 검증)과도 다름.

### 6. `Proc.Tree` — 프로세스 부모-자식 트리 & 로드된 DLL 의존성 뷰

**카테고리**: Applications / System / Monitor
**핵심 기능**: 현재 실행 중인 모든 프로세스를 부모-자식 관계 트리로 시각화하고, 선택한 프로세스가 로드한 DLL 목록과 열린 핸들(파일·소켓)을 보여준다. 악성코드 탐지, 비정상 자식 프로세스 탐지, 의존 DLL 확인에 활용한다.

**상세 기능**:
- 프로세스 부모-자식 트리 (PID, 이름, CPU%, 메모리 열 포함)
- 선택 프로세스 → 로드된 DLL 목록 (경로·버전·서명 여부)
- 열린 파일 핸들 / TCP-UDP 소켓 목록
- 프로세스 종료 버튼 (관리자 권한 요청)
- 새 프로세스 생성/종료 실시간 하이라이팅 (FlashTimer)
- 트리 스냅샷 저장 → 나중에 비교

**기술 스택**:
- `System.Management` WMI `Win32_Process` (부모 PID)
- `System.Diagnostics.Process.Modules` (DLL 목록)
- P/Invoke `NtQueryInformationProcess`, `GetTcpTable2`
- WPF `TreeView` 가상화

**수익 모델**: 무료
**차별화**: Proc.Pilot(Section 6, 프로세스 관리)과 달리 트리 시각화 + DLL 의존성 분석 특화. App.Temp(임시 파일)와 다름.

### 7. `Icon.Maker` — SVG → ICO 멀티 사이즈 변환 + 픽셀 그리드 아이콘 에디터

**카테고리**: Applications / Tools.Utility

- 픽셀 그리드 에디터: 16×16 / 32×32 확대 편집, 연필·지우개·채우기 도구
- 각 사이즈별 미리보기 패널 (다크/라이트 배경 전환)
- Windows 앱 요구 사이즈 체크리스트 (MSIX / Win32)
- ICO → PNG 개별 사이즈 추출

- `SkiaSharp` SVG 래스터라이징 + 픽셀 조작

- WPF `WriteableBitmap` 픽셀 그리드 렌더링

**수익 모델**: 무료
**차별화**: SVG.Forge(SVG 편집·내보내기)는 벡터 도구. Icon.Hunt(아이콘 검색)는 검색 도구. Icon.Maker는 변환+직접 편집 특화.

- 과거 벤치마크 결과 히스토리 저장 & 비교

**기술 스택**:
- `Microsoft.CodeAnalysis.CSharp.Scripting` (Roslyn)
- `BenchmarkDotNet` (별도 프로세스 실행)
- `SkiaSharp` 차트 렌더링
- `System.Text.Json` 결과 영속화

**수익 모델**: 무료
**차별화**: Git.Stats·SQL.Lens 등 기존 분석기는 결과물 분석. Bench.Run은 코드 성능 측정 특화. Visual Studio 없이 독립 실행 가능.

### 10. `Clip.Snap` — 영역 선택 화면 캡처 → 클립보드 즉시 복사 (트레이 앱)

**카테고리**: Applications / Tools.Utility
**핵심 기능**: 전역 단축키(기본 Win+Shift+S 대체)를 누르면 화면이 반투명 오버레이로 덮이고, 드래그로 영역을 선택하면 즉시 클립보드에 PNG로 복사한다. 트레이에 상주하며 최근 캡처 히스토리를 팝업으로 관리한다.

**상세 기능**:
- 전역 단축키 등록 (사용자 정의 가능)
- 오버레이 드래그 영역 선택 → 클립보드 복사
- 캡처 후 즉시 파일 저장 옵션 (날짜·시각 자동 파일명)
- 최근 캡처 20개 트레이 팝업 히스토리 (섬네일 그리드)
- 기본 이미지 편집: 화살표·텍스트·박스 어노테이션 추가 후 저장
- 다중 모니터 지원

**기술 스택**:
- WinForms 전체화면 오버레이 (`Form` `TopMost + TransparencyKey`)
- `user32.dll` `RegisterHotKey` 전역 단축키
- `System.Windows.Forms.Clipboard` PNG 복사
- `System.Drawing.Bitmap` 화면 캡처
- `NotifyIcon` + `DarkMenuRenderer` 트레이 아이콘

**수익 모델**: 무료
**차별화**: Screen.Recorder(전체 녹화), Web.Shot(웹 스크린샷), OCR.Live(OCR 추출)와 달리 빠른 영역 캡처+클립보드 특화. Snap.Cast(Section 6)는 OCR 1회 스냅샷 도구로 차이 있음.

### 1. `Prism.Split` — 프리즘 색 분산 & 색 수용체 퍼즐

**장르**: Puzzle / Optics
**핵심 메커닉**: 흰 레이저 빔이 프리즘을 통과하면 RGB 세 가지 색으로 분리된다. 거울·프리즘·컬러 필터를 격자 위에 배치해 목표 색상 수용체에 정확한 색을 도달시키는 광학 퍼즐 게임.

**게임플레이**:
- 격자 위 광원(흰 빔 발사)이 고정되고, 빔 경로에 아이템 배치
- 프리즘: 입사각에 따라 R·G·B 분리 각도 계산
- 45° 거울: 빔 반사, 합산 거울: 두 색 빔을 혼합(R+G=노랑 등)
- 색상 필터: 특정 채널만 통과 (R 필터 → G,B 차단)
- 목표: 모든 컬러 수용체에 올바른 색 도달 → 스테이지 클리어
- 50+ 스테이지, 스테이지 에디터 (커스텀 퍼즐 제작)

**기술**: WPF Canvas 광선 추적 (벡터 선분 교차 알고리즘), 색 혼합 덧셈 광학 모델, `StreamGeometry` 빔 렌더링
**매력**: "빛의 색 과학"을 직접 조작하는 쾌감. 예상치 못한 색 혼합 결과가 "아하!" 순간을 만든다.
**차별화**: Light.Cast(빛 세기 집중, 단색 빔)는 물리 퍼즐. Prism.Split는 색 분산·혼합이 핵심 메커닉.

### 2. `Echo.Self` — 과거 행동 분신 동시 실행 플랫포머 퍼즐

**장르**: Puzzle / Platformer / Time-manipulation
**핵심 메커닉**: 플레이어가 한 번 플레이한 행동이 "분신"으로 녹화되어 다음 라운드에 동시에 재생된다. 최대 5개 분신이 동시에 스위치를 밟고 문을 열어야 하는 협력형 자기 퍼즐.

**게임플레이**:
- 라운드 시작 → 제한 시간 안에 이동·점프·스위치 조작
- 라운드 종료 → 내 행동이 분신 #N으로 저장, 다음 라운드에 재생
- 분신 N개가 동시에 움직이는 동안 현재 플레이어가 N+1번째 동작 수행
- 스위치 A·B·C를 동시에 눌러야 열리는 문, 분신 타이밍 조율이 핵심
- 잘못된 분신 행동 리셋 기능 (최근 1개 취소)
- 60+ 스테이지, 빠른 반복 리셋 설계

**기술**: WPF `DispatcherTimer` 프레임 단위 행동 기록 (Queue<InputFrame>), 분신 재생 프레임 동기화, AABB 충돌 감지
**매력**: "내 과거 자아와 협력"하는 독특한 시간 퍼즐. 타이밍 정확히 맞출 때의 성취감.
**차별화**: Shadow.Run(Section 6, 그림자 2중 이동)과 달리 행동 녹화·다중 분신 누적이 핵심. Phantom.Step(발자국 함정)과도 다름.

### 3. `Sonar.Run` — 음파 탐지 암흑 동굴 어드벤처

**장르**: Puzzle / Adventure / Stealth
**핵심 메커닉**: 화면이 완전한 암흑 속에서 플레이어가 소리 펄스를 발사하면 벽·적·아이템에서 반향이 돌아온다. 반향 강도와 지연 시간으로만 지형을 파악해 이동하는 음파 탐지 어드벤처.

**게임플레이**:
- 스페이스바 → 소나 펄스 발사 → 반경 확장 원형 웨이브 애니메이션
- 웨이브가 벽에 닿은 곳만 일시적으로 가시화 (0.5초 후 암흑으로 복귀)
- 적은 소나 펄스 소리를 듣고 위치 감지 → 순찰 경로 변경
- 발자국 소리로도 적 위치 파악 (적의 소나도 플레이어에게 보임)
- 아이템: 조용한 이동 신발, 소나 재사용 대기 단축, 연기 폭탄
- 30+ 레벨, 타임어택·스텔스 두 모드

**기술**: WPF Canvas 원형 웨이브 `EllipseGeometry` 애니메이션, 교차 감지 (`Rect.IntersectsWith`), 소리 거리 감쇠 모델링
**매력**: "보이지 않는 공간"에서 소리만으로 이동하는 극도의 긴장감. 조용히 해야 살아남는 딜레마.
**차별화**: 기존 구현 게임들과 완전히 다른 감각 사용 메커닉. 어둠+음파 조합이 신선.

### 4. `Cryo.Jump` — 시간 동결 플랫포머

**장르**: Platformer / Puzzle
**핵심 메커닉**: 점프 중 언제든 시간을 얼릴 수 있다. 공중에 멈춘 채 주변 물체(발사체·플랫폼·적)도 함께 정지하며, 이 상태에서 다음 행동을 계획한다. 시간을 재개하면 얼린 순간부터 물리가 이어진다.

**게임플레이**:
- 점프 후 Shift → 시간 동결 (아이스 파티클 이펙트)
- 얼린 동안: 이동 방향 조이스틱으로 다음 점프 준비, 발사체 방향 확인
- 발사체 옆에서 동결 후 살짝 옆으로 이동 → 재개 시 발사체 통과
- 시간 동결 에너지 게이지 (연속 동결 제한)
- 환경 퍼즐: 회전하는 발판, 발사체 흐름, 움직이는 적
- 50스테이지 + 도전 모드 (에너지 무한 금지)

**기술**: WPF `DispatcherTimer` Pause/Resume, 모든 물체 `Velocity` 벡터 스냅샷 저장, 파티클 Canvas 렌더링
**매력**: "멈춰서 생각하는 시간"이 실제 게임 메커닉. 동결 해제 직후 완벽한 착지의 짜릿함.
**차별화**: 시간 되감기(Time.Farm 등)와 달리 동결만 가능, 전진만 있음. Gravity.Flip(구현완료, 중력 전환)과도 다름.

### 5. `Layer.Stack` — 투명 레이어 N개 겹치기 이미지 완성 퍼즐

**장르**: Puzzle / Creative
**핵심 메커닉**: 반투명 레이어 카드 여러 장이 주어진다. 각 레이어에는 색상 도형 패턴이 있고, 이것들을 특정 순서로 겹치면 목표 이미지가 완성된다. 순서와 방향이 핵심인 시각적 퍼즐.

**게임플레이**:
- 레이어 3~6장 (점점 증가), 각각 다른 색·패턴의 반투명 도형들
- 드래그로 레이어 순서 변경, 각 레이어 X/Y 이동 조정
- 목표 이미지 슬롯: 레이어 겹친 결과와 픽셀 유사도 실시간 % 표시
- 정확도 90% 이상 → 클리어 (1~3성 평가)
- 색상 덧셈 혼합 (반투명 겹치면 색이 섞임)
- 레이어 회전(90°/자유 각도) 기능 추가 스테이지
- 스테이지 에디터: 커스텀 레이어 패턴 제작

**기술**: WPF `WriteableBitmap` 레이어 합성 (알파 블렌딩 수동 계산), 유사도 MSE 계산, `RenderTargetBitmap` 스냅샷
**매력**: 정답을 모르는 상태에서 조합을 시도하다가 목표와 일치하는 순간의 발견 쾌감.
**차별화**: 기존 어떤 퍼즐 게임과도 다른 레이어 합성 메커닉. 시각적으로 아름다운 색 혼합.

### 6. `Seed.Fall` — 씨앗 분산 물리 퍼즐

**장르**: Simulation / Puzzle
**핵심 메커닉**: 나무에서 씨앗이 떨어진다. 바람·물 흐름·동물 이동·중력 등 환경 요소를 배치해 씨앗이 목표 지점(흙 구역)에 착지하도록 유도하는 물리 퍼즐.

**게임플레이**:
- 씨앗 종류별 특성: 민들레(바람), 도토리(구름), 연꽃(물 위 부유)
- 환경 도구 배치: 바람 팬, 물 흐름 방향, 동물 이동 경로, 경사면
- 씨앗 복수 개 동시 발사, 모든 씨앗이 목표 도달 시 클리어
- 도구 사용 개수 제한 (별점 조건)
- 씨앗 여러 개가 부딪혀 경로 변경하는 연쇄 물리
- 계절 이벤트: 봄(강한 바람), 여름(물 흐름), 가을(낙엽 방해)

**기술**: 2D 물리 직접 구현 (Verlet 적분, 원형 충돌), WPF Canvas 파티클 씨앗 렌더링, 바람 벡터장 시각화
**매력**: 자연 현상을 퍼즐로 체험하는 감성. 씨앗이 바람 타고 날아가는 시각적 아름다움.
**차별화**: Sand.Fall(낙하 샌드박스), Leaf.Grow(L-시스템 성장)와 달리 씨앗 분산·목적지 도달이 핵심 목표. Rain.Maker(날씨 최적화)와도 다름.

### 7. `Sym.Build` — 회전/반사 대칭 제약 드로잉 퍼즐

**장르**: Puzzle / Creative
**핵심 메커닉**: 캔버스에 선을 그리면 회전 대칭·반사 대칭 제약에 따라 나머지 부분이 자동으로 그려진다. 목표 패턴을 이 대칭 제약 안에서만 완성해야 하는 창의적 퍼즐.

**게임플레이**:
- 대칭 모드: 2-fold(좌우), 4-fold(90° 회전), 6-fold(60° 꽃), 반사 등
- 한 붓으로 연속 드로잉, 대칭 미러링 실시간 렌더링
- 목표: 주어진 복잡한 패턴을 최소 획 수로 완성
- 금지 영역: 특정 구역에 선이 들어가면 실패
- 제한: 선 총 길이 또는 획 수 최소화로 별점 결정
- 자유 모드: 제약 없이 만다라·칼레이도스코프 패턴 제작

**기술**: WPF `Polyline` 실시간 대칭 변환 (행렬 회전·반사 `Matrix`), 자유 곡선 스플라인 (`PathGeometry`), 픽셀 교차 검사
**매력**: 한 획이 여러 겹쳐 그려지는 시각적 마법. 의도치 않은 아름다운 패턴이 탄생하는 즐거움.
**차별화**: Shutter.Run(이동 궤적=장노출 드로잉)은 러너 게임. Sym.Build는 정적 퍼즐+창작 도구 결합.

### 8. `Pulse.Run` — 심장 BPM 리듬에 맞춘 무한 러너

**장르**: Rhythm / Runner
**핵심 메커닉**: 배경 음악의 BPM에 정확히 맞춰 점프·슬라이드·방향 전환해야 한다. 박자를 타면 캐릭터가 빛나며 보너스 점수, 박자를 놓치면 장애물에 둔감해져 피격 확률 증가.

**게임플레이**:
- 무한 런 + 박자 인식 시스템 (BPM 분석 실시간)
- 박자에 맞춘 점프 → 높이 보너스, 박자 어긋난 점프 → 낮은 높이
- 콤보 시스템: 연속 온비트 → 배속 증가 + 장애물 예고 시간 단축
- 하트 박동 UI: 박자에 맞춰 배경 맥박 이펙트
- 기본 제공 음악 10트랙 + 사용자 MP3 로드 (BPM 자동 감지)
- 전용 글로벌 리더보드 (로컬 최고 점수 저장)

**기술**: `NAudio` MP3 재생 + BPM 온셋 탐지 (에너지 기반 비트 감지), WPF `DispatcherTimer` 박자 윈도우(±50ms), 파티클 이펙트
**매력**: 음악과 움직임이 완벽히 맞을 때의 "플로우" 상태. 자기 음악으로 즐기는 개인화.
**차별화**: Beat.Drop·Chord.Strike(구현완료, 악기 퍼포먼스), Rhythm.Type(Section 5, 타이핑)과 달리 러너 액션 + 리듬 보너스 결합.

### 9. `Magma.Flow` — 용암 점성 유체 흐름 제어 퍼즐

**장르**: Puzzle / Physics
**핵심 메커닉**: 격자 위 용암이 점성 유체처럼 흘러내린다. 경사로·벽·냉각석을 배치해 용암 흐름을 목표 구역으로 유도하거나 특정 기계를 활성화하는 물리 퍼즐.

**게임플레이**:
- 용암 셀 오토마타 시뮬레이션: 점성·온도·흐름 속도 속성
- 배치 도구: 경사면(흐름 방향 변경), 냉각석(용암 굳혀 다리 생성), 채널(좁혀 압력 증가)
- 목표: 용암을 기계 공급구에 정확량 채우거나, 특정 구역에 닿지 않게 유도
- 굳은 용암 = 새 발판 (이후 캐릭터 이동 경로 생성)
- 도구 개수 제한 → 최소 도구로 해결 시 퍼펙트 클리어
- 45스테이지 + 용암 수위 조절 물리 챌린지 스테이지

**기술**: 격자 셀 오토마타 (C# 2D 배열, 매 프레임 병렬 업데이트), 온도 확산 방정식 단순화 구현, WPF `WriteableBitmap` 픽셀 직접 렌더링
**매력**: 용암이 실제로 흘러가는 감각. 예상 못한 방향으로 흘러 계획을 수정하는 긴장감.
**차별화**: Sand.Fall(낙하 샌드박스 자유 플레이), Zipline.Rush(Verlet 케이블)과 달리 점성 유체 목표 유도 퍼즐 특화.

### 10. `Grid.Hack` — 격자 위 로봇 제한 명령어 프로그래밍 퍼즐

**장르**: Puzzle / Educational
**핵심 메커닉**: 격자 위 로봇에게 제한된 명령어(이동/회전/반복/조건)를 프로그래밍해 목표에 도달시킨다. 단, 명령어 슬롯 수가 엄격히 제한되어 재사용 가능한 함수와 루프를 잘 설계해야만 해결된다.

**게임플레이**:
- 명령어: 전진, 좌/우 회전, 함수 호출 F1/F2, 조건 분기 (타일 색상 기반)
- 주 프로그램 슬롯 5개, 함수 F1/F2 각각 5개 슬롯 → 총 15개로 전체 경로 커버
- 타일 색상 조건 분기: 빨간 타일에서 F1 실행, 파란 타일에서 F2 실행
- 별 수집 + 로봇 귀환 동시 달성 시 퍼펙트
- 80+ 스테이지 (Logo 터틀, 미로, 프렉탈 패턴까지 확장)
- 실행 시각화: 로봇이 한 명령씩 이동, 현재 실행 명령어 하이라이팅

**기술**: 명령어 인터프리터 재귀 실행 (Stack 기반), WPF `UniformGrid` 격자 렌더링, 로봇 애니메이션 `TranslateTransform`
**매력**: "최소 코드로 최대 경로를" 달성하는 알고리즘적 쾌감. 프로그래밍 사고력 자연스럽게 훈련.
**차별화**: Asm.Quest(어셈블리 레지스터 목표값)는 레지스터/CPU 시뮬레이션. Grid.Hack는 Logo 터틀 스타일 격자 로봇 프로그래밍. Path.Grid(경로 최적화 vs AI)와도 다름.
