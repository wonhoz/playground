### 1. `Gym.Log` — 근력 운동 루틴·세트·중량 로컬 트래커

**카테고리**: 헬스/라이프스타일 / Windows 트레이 앱
**핵심 기능**: 헬스장에서 매 세트마다 중량·반복수를 빠르게 입력하고, 근육 그룹별 볼륨 누적과 1RM(1회 최대 중량) 계산을 자동화합니다. Health.Log(건강 지표 수치 기록)와 달리 **근력 운동 특화** 루틴 관리에 집중합니다.

**상세 기능**:
- 루틴 빌더: 운동 종목 선택 → 세트 수·목표 반복수·목표 중량 사전 설정
- 세션 중 입력 패널: 세트별 실제 반복수/중량 빠른 입력 (키보드 단축키 지원)
- 1RM 자동 계산: Epley / Brzycki 공식 선택, 진행도 그래프
- 근육 그룹 히트맵: 주간/월간 부위별 볼륨(세트×반복×중량) 시각화
- 운동 기록 히스토리 + CSV 내보내기
- 개인 기록(PR) 자동 감지 → 트레이 풍선 알림

**기술 스택**:
- .NET 8 WinForms 트레이 앱
- SQLite (LiteDB 대안) 로컬 저장
- LiveCharts2 (근력 진행 그래프)
- Epley/Brzycki 1RM 계산 공식 내장

**수익 모델**: 무료 (로컬 단독 앱, 오픈소스 고려)
**차별화**: Health.Log는 체중·혈압 등 건강 수치 중심. Gym.Log는 세트/반복/중량·루틴 관리·1RM 특화.

### 3. `Slide.Forge` — Markdown → 슬라이드 덱 변환기

**상세 기능**:
- 좌측: Markdown 에디터 / 우측: 슬라이드 실시간 미리보기
- `---` 구분자로 슬라이드 분할, `##` → 섹션 제목 자동 인식
- Reveal.js 내장 테마 10종 + 커스텀 CSS 지원
- 발표자 노트 (`Note:` 블록) 지원
- HTML 단일 파일 내보내기 (오프라인 발표 가능)
- PDF 내보내기 (Playwright headless 렌더링)
- 코드 블록 Syntax Highlighting (Prism.js)

**기술 스택**:
- .NET 8 WPF + WebView2 (Reveal.js 렌더링)
- Markdig (Markdown 파서)
- Playwright (PDF 내보내기)
- Reveal.js 번들 내장

**수익 모델**: 무료 오픈소스
**차별화**: Mark.View는 범용 Markdown 뷰어. Slide.Forge는 발표용 슬라이드 변환에만 집중.

### 6. `WiFi.Cast` — Wi-Fi 채널·신호 세기 분석기

**카테고리**: 네트워크/유틸리티 / Windows 트레이 앱
**핵심 기능**: 주변 Wi-Fi 네트워크의 채널·신호 세기·대역(2.4GHz/5GHz)을 실시간 스캔하여 간섭 현황을 분석합니다. Net.Scan(ARP 기반 LAN 장치 탐색)과 달리 **무선 채널 간섭·최적 채널 추천** 특화.

**상세 기능**:
- 주변 AP 스캔: SSID, BSSID, 채널, RSSI, 대역, 보안 방식
- 채널 간섭 히트맵: 2.4GHz(1~14채널) / 5GHz(36~165채널) 시각화
- 최적 채널 자동 추천 (현재 간섭 최소 채널)
- 신호 세기 실시간 그래프 (특정 AP 모니터링)
- 연결된 Wi-Fi 통계 (속도·지연·패킷 손실)
- 스캔 결과 CSV 내보내기

**기술 스택**:
- .NET 8 WinForms 트레이 앱
- Windows WLAN API (NativeWifi / ManagedWifi)
- LiveCharts2 (채널 히트맵·신호 그래프)

**수익 모델**: 무료
**차별화**: Net.Scan은 LAN ARP 장치 탐색. WiFi.Cast는 무선 채널 분석·간섭 감지·최적 채널 추천.

### 7. `Back.Cast` — Windows 폴더 증분 백업·버전 관리

**카테고리**: 시스템/유틸리티 / Windows 트레이 앱

**상세 기능**:
- 소스·대상 폴더 쌍 복수 설정 (로컬·외장드라이브·NAS)
- 증분 백업: 수정된 파일만 타임스탬프 폴더에 복사
- 백업 스케줄: 시간/일/이벤트(파일 변경 감지 시 즉시) 선택
- 버전 브라우저: 날짜별 스냅샷 트리뷰, 파일별 버전 비교
- 개별 파일 복원 + 전체 스냅샷 롤백
- VSS(볼륨 섀도 복사) 활용 (잠긴 파일 백업)

**기술 스택**:
- .NET 8 WinForms 트레이 앱
- FileSystemWatcher (실시간 변경 감지)
- VSS API (P/Invoke)
- SQLite (백업 메타데이터·버전 인덱스)

**수익 모델**: 무료 오픈소스
**차별화**: Folder.Sync는 단방향 동기화 (최신 상태 유지). Back.Cast는 버전 히스토리 보존 + 복원 브라우저.

### 1. `Quilt.Craft` — 기하 타일 회전·배치 퀼트 패턴 완성 퍼즐

**장르**: 퍼즐 / 패턴 매칭
**핵심 메커닉**: 전통 퀼트 패턴을 모티프로, 다양한 기하 타일(삼각형·사각형·육각형 조각)을 회전·배치하여 목표 패턴을 완성합니다. 색상 패턴 매칭과 기하 대칭이 동시에 요구되는 시각적 퍼즐.

**게임플레이**:
- 타일 팔레트에서 조각 선택 → 그리드에 배치, 90°/45° 회전
- 목표 패턴 이미지와 현재 배치 실시간 비교 (일치율 %)
- 제한 타일 수로 완벽 패턴 달성 (효율성 점수)
- 레벨 진행: 패턴 복잡도 점진적 증가 (4×4 → 12×12 그리드)
- 창작 모드: 자유 패턴 제작 후 퍼즐로 공유
- 일일 도전 퀼트 (전 세계 유저와 최단 수 경쟁)
- 달성 배지: 대칭 완성, 퍼펙트 배치, 스피드런 등

**기술**: WPF Canvas 기하 렌더링, 회전 행렬 변환, 패턴 비교 알고리즘
**매력**: 완성 시 아름다운 퀼트 패턴이 나타나는 시각적 보상, 창작욕 자극
**차별화**: avoid.md 내 타일 퍼즐 없음. 퀼트 패턴이라는 독특한 테마 + 창작 모드로 차별화.

### 2. `Deep.Swim` — 수중 부력·해류 물리 탐험

**장르**: 물리 시뮬레이션 / 탐험
**핵심 메커닉**: 수중 환경에서 부력·해류·수압을 실제 물리로 구현합니다. 캐릭터는 산소 게이지를 관리하며 다이빙하고, 해류를 이용하거나 맞서 싸우며 해저 유물을 탐험합니다.

**게임플레이**:
- 2D 사이드뷰 수중 탐험 (절차 생성 해저 지형)
- 부력 계산: 깊이에 따른 수압, 공기통 잔량에 따른 부력 변화
- 해류 시스템: 방향·세기 다양한 조류, 와류 지점
- 산소 게이지 관리: 공기 보충 포인트 탐색
- 해양 생물 생태계: 포식자 회피·공생 생물 활용
- 유물·보물 발굴 → 업그레이드 (잠수복·장비)
- 보스: 대형 해양 생물 (조류·부력 이용 회피 전략)

**기술**: Box2D 물리 (부력·유체 시뮬레이션), MonoGame/WPF 렌더링, 절차 생성
**매력**: 수중의 유동적·불안정한 물리감, 산소 관리의 긴장감, 미지의 해저 탐험
**차별화**: avoid.md 내 물리 기반 수중 게임 없음. 부력·해류 물리 시뮬레이션이 핵심 메커닉.

### 3. `Geo.Build` — 컴퍼스+자 유클리드 기하 구성 퍼즐

**장르**: 퍼즐 / 교육
**핵심 메커닉**: 컴퍼스와 자만 사용하는 유클리드 기하 작도 퍼즐입니다. 주어진 점에서 정삼각형, 황금 비율, 정오각형 등을 최소 작도 횟수로 완성하는 논리 퍼즐.

**게임플레이**:
- 도구: 자(직선 연장), 컴퍼스(원 그리기) 두 가지만
- 목표: 기하 도형 완성 (이등분선, 수직선, 정다각형 등)
- 점수: 최소 작도 횟수 도전 (파 시스템)
- 힌트 시스템: 다음 작도 단계 점선 표시
- 레벨 100+: 탈레스 정리→황금 나선→정20면체 투영
- 업적: 수학자 이름 업적 (유클리드, 피타고라스, 아르키메데스)
- 자유 작도 모드: 자신만의 기하 구성 제작

**기술**: WPF Canvas (기하 렌더링), 교차점 계산 (원-직선·원-원), 벡터 수학
**매력**: 수학적 아름다움과 퍼즐 성취감의 결합, 실제 유클리드 기하 학습 효과
**차별화**: Asm.Quest(프로그래밍 퍼즐)와 다른 영역. 작도·기하 특화 완전 새 장르.

### 4. `Text.Venture` — 절차 생성 텍스트 어드벤처

**장르**: 텍스트 어드벤처 / 로그라이크
**핵심 메커닉**: 클래식 텍스트 어드벤처를 절차 생성 세계관으로 현대화합니다. 매 플레이마다 다른 던전·NPC·아이템·퀘스트가 생성되며, 사용자가 세계관 에디터로 자신만의 텍스트 어드벤처를 제작할 수 있습니다.

**게임플레이**:
- 자연어 명령 파서: `LOOK`, `TAKE SWORD`, `GO NORTH`, `TALK TO MERCHANT`
- 절차 생성 세계: 룸·연결·아이템·NPC·퀘스트 랜덤 조합
- 롤플레이: HP·인벤토리·스킬 포인트 시스템
- 사망 시 새 세계 생성 (로그라이크 구조)
- 세계관 에디터: 방 배치·연결·이벤트 스크립트 편집
- 완성된 어드벤처 공유 (파일 내보내기)

**기술**: C# IF(Interactive Fiction) 파서 엔진, 절차 생성(그래프 기반 룸 연결), WPF 터미널 UI
**매력**: 인터랙티브 소설의 창작 + 플레이 양면, 매번 다른 경험
**차별화**: avoid.md 내 텍스트 어드벤처 없음. 절차 생성 + 에디터 제공으로 차별화.

### 5. `Mini.Golf` — 2D 탑다운 미니 골프 + 코스 에디터

**장르**: 스포츠 / 퍼즐
**핵심 메커닉**: 탑다운 시점의 미니 골프 게임입니다. 물리 기반 공 궤도, 장애물(회전 날개·워터 해저드·경사로), 코스 에디터로 자신만의 코스 제작·공유. Pool.Break(당구)와 다른 골프 특화.

**게임플레이**:
- 탑다운 뷰 골프: 조준선 각도·파워 조절 → 퍼팅
- 물리 엔진: 마찰·반발·경사 효과
- 장애물: 회전 날개, 워터 해저드, 모래 트랩, 이동 블록
- 홀당 파(Par) 도전, 언더파 업적
- 코스 에디터: 드래그&드롭으로 벽·장애물 배치, 스타트/홀 위치 설정
- 코스 공유: 파일 내보내기/불러오기
- 멀티플레이: 로컬 2인 교대 플레이

**기술**: Box2D (물리 시뮬레이션), WPF Canvas 렌더링, 코스 에디터 직렬화 (JSON)
**매력**: 직관적 단순함 + 복잡한 코스의 도전, 창작 코스 공유의 무한 컨텐츠
**차별화**: Pool.Break는 당구·포켓볼. Mini.Golf는 골프 룰·코스 에디터 특화.

### 6. `Dig.Rush` — 고고학 발굴 퍼즐

**장르**: 퍼즐 / 시뮬레이션
**핵심 메커닉**: 격자 위의 흙을 픽셀 단위로 조심스럽게 제거하며 유물을 발굴합니다. 너무 세게 파면 유물이 파손되는 압력 시스템, 소나(탐지기)로 유물 위치를 추정하는 전략적 퍼즐.

**게임플레이**:
- 그리드 위 흙 레이어: 깊이별 다른 재질(흙/모래/암반)
- 도구 선택: 브러시(섬세), 픽(중간), 망치(강력) → 유물 파손 위험 다름
- 소나 탐지기: 사용 횟수 제한, 반경 내 유물 존재 여부 신호
- 유물 파손도: 100%에서 시작, 과도한 힘에 비례 감소 → 점수 반영
- 시대별 유물: 청동기·철기·중세·고대 이집트 등 테마 스테이지
- 박물관: 발굴한 유물 전시, 도감 완성

**기술**: WPF Canvas (픽셀 그리드 렌더링), 물리적 파손 계산, 절차 생성 (유물 배치 알고리즘)
**매력**: 조심스럽게 파는 긴장감과 유물 발견의 쾌감, 박물관 컬렉션 완성욕
**차별화**: avoid.md 내 발굴 게임 없음. 픽셀 파손 시스템·소나 탐지의 독특한 조합.

### 7. `Code.Path` — 로봇 이동·물체조작 프로그래밍 퍼즐

**장르**: 퍼즐 / 교육 (프로그래밍)
**핵심 메커닉**: 로봇에게 명령어(MOVE, TURN, PICK, DROP, IF, LOOP)를 시퀀스로 작성해 목표를 달성합니다. Asm.Quest(어셈블리어 레지스터 조작)와 달리 **공간 탐색·물체 조작** 중심, 더 직관적인 블록 코딩.

**게임플레이**:
- 그리드 맵에서 로봇이 목표 지점까지 아이템 운반
- 블록 코딩 인터페이스 (드래그&드롭 명령 조립)
- 명령어: MOVE(방향), TURN(좌/우), PICK, DROP, IF(조건), LOOP(반복)
- 최소 명령 수 도전 (별점 3개 시스템)
- 다중 로봇: 병렬 실행 (동기화 문제 포함 고급 레벨)
- 레벨 에디터: 맵·아이템·장애물 배치 후 공유

**기술**: WPF (블록 코딩 UI), 그리드 시뮬레이터, 인터프리터 (명령 시퀀스 실행 엔진)
**매력**: 프로그래밍 개념을 게임으로 자연스럽게 학습, 최적화의 도전
**차별화**: Asm.Quest는 어셈블리어 레지스터 레벨. Code.Path는 공간·물체조작 + 블록 코딩 인터페이스.

### 8. `Net.Route` — IP 라우터 배치·경로 최적화 퍼즐

**장르**: 퍼즐 / 네트워크 교육
**핵심 메커닉**: 네트워크 노드 사이에 라우터를 배치하고 경로를 최적화합니다. 다익스트라 알고리즘이 실시간 시각화되며, 트래픽 폭주·링크 단절 이벤트에 대응하는 동적 라우팅 퍼즐.

**게임플레이**:
- 서버·클라이언트 노드 사이에 라우터 배치 (예산 제한)
- 라우팅 테이블 직접 편집: 목적지 → 다음 홉 설정
- 다익스트라 시각화: 최단 경로 실시간 애니메이션
- 이벤트: 링크 단절(페일오버), 트래픽 폭주(병목 해소), DDoS(필터링)
- 점수: 패킷 전달율·지연 시간·비용 종합
- 실제 네트워크 토폴로지 기반 레벨 (스타·링·메시·트리)

**기술**: WPF Canvas (그래프 시각화), 다익스트라 구현, 시뮬레이션 이벤트 엔진
**매력**: 네트워킹 개념을 게임으로 자연스럽게 학습, 장애 대응의 전략적 쾌감
**차별화**: avoid.md 내 네트워크 라우팅 퍼즐 없음. 실제 라우팅 알고리즘 시각화가 핵심.

**기술**: WPF 턴제 시뮬레이션 엔진, 확률적 날씨 시스템, 픽셀아트 렌더링

**차별화**: avoid.md 내 경작 게임 없음. 턴제·날씨 불확실성·시장 가격 연동이 차별점.

### 10. `Loot.Sort` — 인벤토리 테트리스 최적화 퍼즐

**장르**: 퍼즐 / 전략
**핵심 메커닉**: RPG 인벤토리를 모티프로 한 공간 최적화 퍼즐입니다. 불규칙한 폴리오미노 형태의 아이템을 격자 인벤토리에 최대한 효율적으로 배치하고, 아이템 세트 시너지를 노려 점수를 극대화합니다.

**게임플레이**:
- 다양한 폴리오미노 아이템 (검·방패·갑옷·물약 등 RPG 테마)
- 격자 인벤토리에 아이템 배치 (90°/180° 회전 가능)
- 세트 시너지: 특정 조합 인접 배치 시 보너스 점수
- 공간 효율 점수 + 세트 점수 합산

- 일일 도전: 정해진 아이템 세트로 최고 점수 경쟁

**기술**: WPF (드래그&드롭 폴리오미노 UI), 배치 유효성 검사 알고리즘, 점수 계산 엔진
**매력**: 퍼즐 최적화의 뇌섹 쾌감 + RPG 아이템 수집욕의 결합
**차별화**: avoid.md 내 인벤토리 퍼즐 게임 없음. 폴리오미노 + 세트 시너지 조합이 독창적.
