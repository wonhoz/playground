### 1. `Hash.Check` — 파일 무결성 검증기
**카테고리**: Files / Dev/Tools
**수익 잠재력**: ★★★★☆

#### 핵심 기능
- 파일 드래그&드롭 → MD5 / SHA-1 / SHA-256 / SHA-512 즉시 계산
- 예상 해시 붙여넣기 → 자동 비교 (일치=초록, 불일치=빨강 강조)
- `.md5` / `.sha256` / `.checksum` 파일 자동 파싱 → 일괄 검증
- 폴더 내 파일 전체 해시 계산 → CSV 내보내기 (변조 추적용)
- 파일 변경 감시 모드: 해시 스냅샷 저장 → 이후 비교

#### 차별점
- `Text.Forge`의 해시 기능은 **텍스트 입력 대상** → `Hash.Check`는 **파일 무결성 전용**
- 다운로드 파일 검증, 배포 파일 체크섬 확인 = 매일 쓰는 류
- 경쟁 도구 없음 (대부분 CLI) → GUI 독점

#### 기술 스택
- WPF / `System.Security.Cryptography` / `FileSystemWatcher`

### 5. `Patch.View` — Git Patch 파일 뷰어 & 적용기
**카테고리**: Dev/Tools
**수익 잠재력**: ★★★☆☆

#### 핵심 기능
- `.diff` / `.patch` 파일 드래그&드롭 → 구문 강조 diff 뷰어 (myers diff, hunk 단위)
- 헝크 단위 **선택 적용 / 스킵** (cherry-pick 적용)
- 로컬 폴더에 패치 적용 → 충돌 시 `.rej` 파일 생성
- **역방향 적용** (revert 패치)
- 현재 폴더 상태 vs 패치 적용 후 미리보기 (before/after)
- 이메일/GitHub 공유 패치 파싱 (git format-patch 출력 지원)

#### 차별점
- `Deep.Diff` = 두 파일 직접 비교 → `Patch.View` = **.patch 파일을 대상으로 적용/관리**
- 오픈소스 기여, 코드 리뷰 패치 배포에 필수
- 경쟁 도구 없음 (모두 CLI `git apply`) → GUI 독점

#### 기술 스택
- WPF / 자체 unified diff 파서 / `LibGit2Sharp`

### 3. `Cell.War` — 세포 분열 영역 전략 게임
**장르**: Strategy / Casual

#### 게임 플레이
- 원형 세포들이 화면 위에 배치 (플레이어: 파란색, 적: 빨간색)
- **클릭 = 선택한 세포 분열** (절반 크기 두 개로 나뉨) + 이동 방향 지정
- 세포 충돌: **큰 쪽이 흡수** (크기 합산)
- 작은 세포 = 빠르고 민첩, 큰 세포 = 강하고 느림
- **목표**: 적 세포 전멸 or 화면 영역 70% 점령
- 적 AI: 적당한 크기 유지하며 약한 세포 사냥

### 10. `Sound.Grid` — 음악 시퀀서 퍼즐 게임
**장르**: Puzzle / Music / Casual

#### 게임 플레이
- N×M 격자 (행=음정, 열=박자)
- **목표 멜로디**가 재생됨 → 격자에 노트를 배치해 일치시키기
- 재생하면 격자가 좌→우로 스캔하며 각 셀의 노트 연주
- **도전 모드**: 주어진 노트 조각(테트리스처럼 연결된 노트 블록)을 배치해 목표 달성
- **창작 모드**: 자유롭게 작곡 → 공유 코드 내보내기
- 악기 선택: 피아노, 마림바, 신스, 드럼

#### 핵심 루프
- 목표 멜로디 귀로 익히기 → 격자에 노트 배치 → 재생 → 비교 → 미세 조정 → 일치 시 멜로디 완성 쾌감

#### 왜 Hue.Flow·Sand.Fall 팬에게 최적?
- 보고 듣는 시각+청각 피드백 = Hue.Flow의 감각적 만족감 계열
- 자유 창작 + 퍼즐 = 두 가지 재미 동시

#### 기술 스택
- WPF / `NAudio` MIDI 합성 / 격자 기반 시퀀서
