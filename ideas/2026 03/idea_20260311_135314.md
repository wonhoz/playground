### 1. Win.Rules — 창 자동화 규칙 엔진

**카테고리**: System / Productivity
**핵심 기능**: "앱 X가 실행되면 → 모니터 2번 우측 절반에 배치 + 우선순위 High"처럼 이벤트 기반 규칙으로 창을 자동 관리하는 규칙 엔진

**상세 기능**:
- IF/THEN 규칙 정의 (GUI 드래그블 조건 빌더):
  - 조건: 앱 실행 / 앱 포커스 / 시간대 / 파워플랜(배터리/AC)
  - 액션: 이동할 모니터·위치·크기 / 창 우선순위 / 항상 위 / 투명도
- 복합 조건: "아침 9~18시에 Slack이 열리면 → 2번 모니터 좌측 1/3"
- 규칙 프로파일: 집/사무실 프로파일 전환 (단축키)
- 갈등 감지: 두 규칙이 같은 앱을 타깃할 때 우선순위 경고
- 실행 로그: 어떤 규칙이 언제 발동됐는지 타임라인
- Windows Shell Hook으로 실시간 창 이벤트 감지

**기술 스택**:
- `WH_SHELL` 훅 (`SetWindowsHookEx`, `HSHELL_WINDOWCREATED`)
- `SetWindowPos`, `EnumDisplayMonitors` Win32
- JSON 규칙 직렬화, WPF 규칙 에디터

**수익 모델**: 멀티모니터 파워유저 필수 도구 → 유료 프리셋 마켓플레이스
**차별화**: Screen.Split=수동 드래그 스냅, Workspace.Switcher=레이아웃 프리셋 저장·복원, **Win.Rules=이벤트 트리거 자동화 규칙 엔진** — 사람이 직접 배치할 필요 없음

### 2. Desk.Grid — 바탕화면 아이콘 레이아웃 저장·복원 관리자

**카테고리**: System / Productivity
**핵심 기능**: 해상도 변경·디스플레이 연결·Windows 업데이트 후 바탕화면 아이콘이 뒤섞이는 고질적 문제를 해결하는 레이아웃 스냅샷 관리자

**상세 기능**:
- 현재 아이콘 위치 스냅샷 저장 (이름 + 날짜)
- 원클릭 복원: 저장된 레이아웃으로 즉시 복구
- 트리거 자동 복원: 해상도 변경 감지 → 자동 스냅샷 일치 복원
- 다중 프로파일: "4K 단일 모니터", "노트북 단독", "듀얼 모니터" 각각 저장
- 아이콘 별칭 매핑: 파일 경로 변경 시에도 아이콘 이름으로 재추적
- 정렬 옵션: 가나다순·이름순·크기순 정렬 후 스냅샷으로 고정
- 트레이 상주: 해상도 변경 감지 → 즉시 Toast 알림 + 복원 원클릭 제공
- 배포: 여러 PC에 동일 바탕화면 레이아웃 배포 (JSON 내보내기)

**기술 스택**:
- `IShellFolder` / `FolderView2` COM (바탕화면 아이콘 좌표 읽기/쓰기)
- `SystemEvents.DisplaySettingsChanged` (.NET)
- JSON 스냅샷 저장

**수익 모델**: 완전 무료 유틸리티 → GitHub Stars → 기업 배포용 MSI 패키지 유료
**차별화**: 아이콘 뒤섞임은 수백만 명이 겪는 pain point인데 해결 도구가 없음

### 3. Key.Test — 키보드 전체 키 진단 도구

**카테고리**: System / Gaming
**핵심 기능**: 키보드의 모든 키를 시각적으로 테스트하고 고스팅·스티킹·채터링을 감지하는 게이머·타이피스트용 진단 도구

**상세 기능**:
- 화면에 키보드 레이아웃 렌더링 (클릭한 키 → 초록으로 점등)
- 감지 모드:
  - **스티킹 테스트**: 키를 눌렀다 떼면 정상 해제되는지 (stuck key)
  - **채터링 테스트**: 단일 입력에서 여러 키 이벤트 발생 감지 (디바운스 실패)
  - **고스팅 테스트**: 동시 여러 키 입력 시 미등록 키 감지 (N-Key Rollover 측정)
  - **레이턴시 측정**: 키 입력 → OS 이벤트 수신까지 ms 측정
- 스캔코드 & 가상키 코드 실시간 표시 (HID 개발자용)
- 리포트: 테스트 결과 HTML 파일로 저장 (키보드 모델·불량 키 목록)
- 특수 키 테스트: 미디어 키·브라우저 키·매크로 키 포함
- 연속 타이핑 테스트: 특정 키 1분 연속 입력 → 채터링 통계

**기술 스택**:
- `SetWindowsHookEx(WH_KEYBOARD_LL)` — 저수준 키 이벤트 정밀 타이밍
- WPF Canvas 키보드 레이아웃 (ANSI/ISO/75%/60% 레이아웃 선택)
- HKLM\SYSTEM\CurrentControlSet\Control\Keyboard 레지스트리 정보

**수익 모델**: 게이머·타이피스트·키보드 커뮤니티 무료 도구
**차별화**: 기존 웹 기반 키 테스터(keyboardtester.com)는 고스팅·채터링·레이턴시 측정 불가 — 로컬 앱으로 훨씬 정밀

- 투명도: 0~100% 슬라이더
- 클릭 통과 모드: 오버레이가 마우스 클릭을 아래 창으로 통과

- 잠금 위치: 특정 모니터 구석에 고정 (레이아웃 저장)

**사용 사례**:
- 코딩 중 API 문서 오버레이
- 영상 통화 중 메모 앱 미니 참조

**기술 스택**:
- `DwmRegisterThumbnail` (DWM Thumbnail API) — 다른 창을 복제 렌더링
- `DwmQueryThumbnailSourceSize`, `DwmUpdateThumbnailProperties`
- WPF 투명 Topmost 창

**수익 모델**: 프로그래머·스트리머·게이머 무료 도구 → 멀티 오버레이 무제한 프리미엄
**차별화**: Win.Cast=가상 카메라 스트리밍, Screen.Split=레이아웃 스냅, **Win.Pop=DWM 썸네일 기반 대화형 미니 창 복제**

### 5. Hotkey.Cast — 시스템 전역 단축키 관리자

**카테고리**: System / Productivity
**핵심 기능**: 원하는 전역 단축키를 자유롭게 등록하고 다양한 액션(앱 실행·텍스트 삽입·창 전환·스크립트)에 연결하는 단축키 관리 허브

**상세 기능**:
- 단축키 등록: 임의 키 조합 → 다양한 액션 연결
  - 앱 실행 (파일 경로 또는 URL)
  - 텍스트 즉시 입력 (이메일 주소, 서명, 코드 스니펫 → SendInput)
  - 창 전환 (특정 앱으로 포커스)
  - 시스템 명령 (볼륨 제어, 화면 캡처, 시스템 잠금)
  - PowerShell/CMD 스크립트 실행
- 충돌 감지: 이미 등록된 전역 단축키와 충돌 시 경고 + 현재 점유 앱 표시
- 컨텍스트 단축키: 특정 앱 포커스 시에만 발동하는 앱별 단축키
- 프로파일: 업무/게임/발표 모드 단축키 세트 전환
- 트레이 상주, 단축키 치트시트 PDF 출력

**기술 스택**:
- `RegisterHotKey` / `UnregisterHotKey` Win32 API
- `SendInput` (텍스트 삽입)
- `GetForegroundWindow` + 프로세스 이름 (컨텍스트 판단)

**차별화**:
- Mouse.Flick = 마우스 제스처 → 단축키 매핑
- Key.Tape = 매크로 시퀀스 녹화·재생
- Layout.Forge = 스캔코드 레벨 키 재배치
- **Hotkey.Cast = 새 단축키 조합 등록 + 임의 액션 연결** (새로운 단축키를 만드는 도구)

**핵심 기능**: 인터넷 없이 로컬 LAN에서 여러 사람이 동시에 그리고 메모를 붙이는 실시간 협업 화이트보드

**상세 기능**:
- 무한 캔버스 (팬+줌)

- LAN 실시간 동기화:
  - mDNS 방 자동 발견
  - WebSocket 델타 동기화 (변경분만 전송)
  - 접속자 커서 실시간 표시 (색상별)

- WPF InkCanvas (펜 입력) + WriteableBitmap (래스터 레이어)

**수익 모델**: 팀 협업 도구 → 무료(3인 이하) + 유료(무제한 참가자, 영구 보드 저장)
**차별화**: Mind.Map=트리 구조 마인드맵, Mood.Board=이미지 콜라주, Screen.Annotate=화면 위 일시적 그리기, **Ink.Board=지속적 캔버스 + 멀티유저 협업**

### 7. Ping.Grid — 멀티 호스트 ICMP 핑 모니터 그리드

**카테고리**: Network / Monitoring
**핵심 기능**: 최대 64개 호스트를 동시에 핑하고 지연시간을 색상 코딩된 그리드로 실시간 시각화

**상세 기능**:

- 셀 색상: 초록(정상) → 노랑(고지연) → 빨강(손실) → 회색(응답없음)
- 지연 기준값 설정: ms 임계치 커스텀
- 히스토리 그래프: 셀 클릭 → 해당 호스트 1시간 지연 추이 차트
- 통계: 최소/최대/평균/표준편차/손실률 컬럼
- 알림: 특정 호스트 다운 → Toast 알림 (Uptime.Eye=HTTP, Ping.Grid=ICMP)
- 자동 임포트: hosts 파일, CSV 로드
- 리포트: 현재 상태 스냅샷 HTML 내보내기
- 트레이 상주: 전체 다운 시 즉시 경고

**기술 스택**:
- `System.Net.NetworkInformation.Ping` 비동기 병렬 (`Parallel.ForEachAsync`)
- WPF DataGrid + 색상 바인딩
- SQLite 지연 히스토리 저장

**차별화**: Net.Scan=ARP 기기 탐지, Uptime.Eye=HTTP 가동 확인, **Ping.Grid=ICMP 다수 호스트 지연·손실 실시간 그리드** — 서버실·홈랩 관리자 특화

**게임플레이**:
- 화면: 기둥·고리·레버가 있는 방, 목표 물체(박스·보석 등)
- 플레이어: 체인 한쪽 끝을 잡고 드래그 → 기둥/고리에 걸기
- 체인 물리: Verlet 세그먼트 (늘어짐·쳐짐·장력 시각화)
- 퍼즐 목표:
  - 목표 물체를 특정 착지점에 내려놓기
  - 여러 체인으로 목표를 특정 높이에 매달기
  - 체인 장력으로 레버 작동 → 문 열기
- 제약: 체인 총 길이 제한, 내구도(장력 초과 시 끊어짐)
- 특수 기둥: 회전 도르래, 자석 기둥, 스프링 고리
- 40레벨 + 물리 자유 샌드박스 모드

**기술**: Verlet 제약 체인 + 기둥 충돌/걸림 판정 (AABB)
**차별화**: Slice.Chain=체인 자르기 퍼즐, Rope.Bridge=다리 위 이동, **Chain.Hook=체인을 걸고 당겨서 물체 이동** — 완전히 다른 메커닉

---

### 10. Veil.Break — 어둠 속 제한 탄약 타겟 발굴 슈터

**장르**: Puzzle / Shooter / Exploration
**핵심 메커닉**: 화면 전체를 덮은 검은 베일을 총격으로 뚫어 숨겨진 타겟을 찾고 격파하는데, 탄약이 제한되어 전략적 사격이 필요

**게임플레이**:
- 100% 검은 베일로 덮인 화면
- 마우스 클릭 = 해당 좌표에 원형 구멍 생성 (작은 원 = 소형 탄환, 큰 원 = 고탄환)
- 구멍 크기별 탄약 소모:
  - 소형 (반경 20px): 1탄
  - 중형 (반경 50px): 3탄
  - 대형 (반경 100px): 8탄
- 숨겨진 요소:
  - ✅ 타겟: 구멍이 겹치면 격파 + 점수
  - 💣 폭탄: 건드리면 탄약 감소
  - 💡 힌트 등: 구멍으로 보이면 인근 타겟 방향 화살표 표시
  - 👻 유령: 보이면 빠르게 이동 → 실시간 추격 필요
- 남은 탄약으로 더 높은 클리어 등급 달성
- 절차 생성 레벨 + 일일 시드

**기술**: 베일 텍스처 마스킹 (WriteableBitmap 알파 채널 지우기), 충돌 감지 BFS
**매력**: Echo.Hunt(소나 어두움 탐색)와 달리 능동적 사격으로 어둠 제거 — "뭐가 있을까" 기대감 + 탄약 절약 퍼즐
