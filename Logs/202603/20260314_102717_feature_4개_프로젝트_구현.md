# 4개 프로젝트 구현 — 23차 브레인스토밍 선택 아이디어

## 작업 개요

23차 브레인스토밍(idea_20260314_074913.md)에서 선택된 4개 프로젝트를 구현.

## 구현 내용

### 1. Snap.Duel (Games/Casual/)
**WPF 2P 반응속도 대전 게임**

```
Games/Casual/Snap.Duel/
├── Snap.Duel.csproj         (.NET 10, WPF)
├── App.xaml / App.xaml.cs
├── MainWindow.xaml          (3분할: P1 | 신호 | P2)
├── MainWindow.xaml.cs       (게임 로직)
└── SettingsDialog.cs        (라운드수/함정 설정)
```

**주요 기능**:
- P1=Q키, P2=P키 반응속도 측정 (Stopwatch, <1ms 정밀도)
- 조건부 신호 (단순/색상 조건/모양 조건)
- 함정 신호 25% 확률 (누르면 패배)
- 1.5~5초 랜덤 대기 후 GO 신호
- 라운드 결과 + 총 세션 승 추적
- 토너먼트 모드 전환

### 2. Ear.Train (Games/Casual/)
**청음 훈련 WPF 게임**

```
Games/Casual/Ear.Train/
├── Ear.Train.csproj         (NAudio 2.2.1, SQLite)
├── App.xaml / App.xaml.cs
├── MainWindow.xaml          (4탭: 단음/인터벌/화음/통계)
├── MainWindow.xaml.cs       (피아노 렌더링 + 퀴즈 로직)
├── Models/MusicModel.cs     (Note, Interval, ChordType, EloRecord)
└── Services/
    ├── AudioService.cs      (사인파 + ADSR Envelope)
    └── DbService.cs         (SQLite ELO 저장)
```

**주요 기능**:
- C4~B5 2옥타브 피아노 건반 UI (흰/검 건반 Canvas)
- 단음 맞추기 / 인터벌(12종) / 화음(7종) 3모드
- 화성적/선율적 인터벌 전환
- ELO 레이팅 시스템으로 약점 트래킹
- ADSR Envelope 사인파 음향 엔진

### 3. Diff.Prompt (Applications/AI/)
**WPF AI 프롬프트 A/B 비교 실험 도구**

```
Applications/AI/Diff.Prompt/
├── Diff.Prompt.csproj       (DiffPlex, SQLite)
├── App.xaml / App.xaml.cs
├── MainWindow.xaml          (좌우 분할 출력 패널)
├── MainWindow.xaml.cs       (실험 관리)
├── Dialogs.cs               (ApiKeyDialog, DiffWindow, HistoryWindow)
├── Models/ExperimentModel.cs (Experiment, ClaudeModels 비용표)
└── Services/
    ├── ClaudeService.cs     (HTTP SSE 스트리밍, 직접 구현)
    └── DbService.cs         (SQLite 실험 이력)
```

**주요 기능**:
- 두 System Prompt를 동시에 Claude API에 전송 (병렬 스트리밍)
- 실시간 스트리밍 출력 (SSE 직접 파싱)
- 토큰 수, 비용($/1M 기준), 레이턴시 표시
- DiffPlex 출력 비교 (삽입/삭제/수정 색상 표시)
- A/B/무승부 투표 기능
- SQLite 실험 이력 (불러오기/내보내기)
- 3개 모델 지원: Opus 4.6 / Sonnet 4.6 / Haiku 4.5

### 4. Circuit.Break (Games/Puzzle/)
**전기 회로 디버깅 퍼즐 WPF 게임**

```
Games/Puzzle/Circuit.Break/
├── Circuit.Break.csproj     (.NET 10, WPF)
├── App.xaml / App.xaml.cs
├── MainWindow.xaml          (Canvas 회로도 + 측정 패널)
├── MainWindow.xaml.cs       (회로 렌더링 + 상호작용)
├── Models/CircuitModel.cs   (Node, Component, PuzzleLevel)
└── Services/
    ├── CircuitSimulator.cs  (MNA + 가우스-요르단 소거법)
    └── LevelFactory.cs      (5개 내장 레벨)
```

**주요 기능**:
- 수정된 노달 분석법(MNA) 회로 시뮬레이터
- 가우스-요르단 소거법으로 연립 방정식 풀기
- 버그 유형: 단선(BrokenWire) / 단락(ShortCircuit) / 저항값 오류(WrongResistance)
- 전압계 / 저항계 / 전류계 측정 도구
- Canvas 회로도 렌더링 (소자 클릭 → 수정)
- 5개 내장 레벨: 직렬 회로, 병렬 오류, T자형, 휘트스톤 브리지, 복합 디버깅
