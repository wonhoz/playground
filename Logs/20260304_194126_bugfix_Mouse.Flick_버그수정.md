# Mouse.Flick 버그 수정 작업 로그

- **날짜**: 2026-03-04
- **태그**: bugfix
- **경로**: `Applications/Tools/Productivity/Mouse.Flick/`

---

## 수정된 버그 목록

### 1. 마우스 버튼 stuck 버그 (`daebb04`)
**증상**: 브라우저에서 제스처 사용 후 다른 윈도우·알림 영역·작업표시줄 등 마우스로 접근 불가
**원인**: WM_RBUTTONDOWN을 앱에 통과시킨 뒤 WM_RBUTTONUP을 차단 → 앱이 "우클릭 누름" 상태에서 해제 불가
**수정**: WM_RBUTTONDOWN 항상 차단 + 제스처 없을 시 SendInput으로 RBUTTONDOWN+UP 재생
**핵심 패턴**:
```csharp
case WM_RBUTTONDOWN:
    _tracking = true; _downPoint = pt; _points.Clear(); _points.Add(pt);
    return new IntPtr(1);   // 항상 차단

case WM_RBUTTONUP:
    if (_gestureStarted) { GestureCompleted?.Invoke(...); return new IntPtr(1); }
    else { ReplayRightClick(); return new IntPtr(1); }  // 우클릭 재생
```

### 2. 제스처 인식 불가 버그 (`64ecf7d`)
**증상**: RBUTTONDOWN 차단 후 제스처가 전혀 인식되지 않음
**원인**: `LLMHF_INJECTED` 플래그 전체 차단 — 터치패드·마우스 드라이버가 SendInput으로 이벤트 주입 시 해당 플래그가 설정되어 `_tracking`이 true가 되지 않음
**수정**: `dwExtraInfo = 0x4D465447`("MFTG") 고유 값으로 자체 주입 이벤트만 식별
**핵심 패턴**:
```csharp
// ❌ 금지: LLMHF_INJECTED 전체 차단 (터치패드 드라이버도 차단됨)
if ((hs.flags & LLMHF_INJECTED) != 0) return CallNextHookEx(...);

// ✅ 올바른 방법: 자체 주입 이벤트만 식별
private static readonly IntPtr OurExtraInfo = new IntPtr(unchecked((int)0x4D465447));
if (hs.dwExtraInfo == OurExtraInfo) return CallNextHookEx(...);

// ReplayRightClick에서:
inputs[0].dwExtraInfo = OurExtraInfo;
```

### 3. 제스처 활성화 메뉴 상태 표시 없음 (`bffbd3c`)
**증상**: 트레이 메뉴의 "제스처 활성화" 클릭 시 아무 시각적 변화 없음
**원인**: `DarkMenuRenderer`가 `OnRenderItemCheck` 미구현 + `ShowImageMargin = false`로 체크마크 영역 숨김
**수정**: 메뉴 텍스트에 ✓/✗ 직접 표시
```csharp
private static string EnabledText(bool enabled) =>
    enabled ? "🖱 제스처 활성화  ✓" : "🖱 제스처 활성화  ✗";
```

### 4. 첫 제스처 궤적 미표시 버그 (`bea0feb`)
**증상**: 앱 실행 후 첫 번째 제스처에서만 오버레이 궤적이 표시되지 않음
**원인**: 첫 `Show()` 시 Win32 핸들 생성 과정에서 OS가 WM_ERASEBKGND를 추가 발송 → Refresh()로 그린 궤적이 소거됨
**수정**: 앱 시작 시 핸들 미리 생성
```csharp
_overlay = new GestureOverlay();
_ = _overlay.Handle;  // 핸들 미리 생성
```

## WH_MOUSE_LL 후크 핵심 교훈

| 함정 | 해결책 |
|------|--------|
| RBUTTONDOWN 통과 + RBUTTONUP 차단 | RBUTTONDOWN도 차단, 비제스처 시 ReplayRightClick |
| LLMHF_INJECTED 전체 차단 | dwExtraInfo 고유값으로 자체 이벤트만 식별 |
| 첫 Show() 핸들 생성 타이밍 | 앱 시작 시 `_ = form.Handle` 로 미리 생성 |
