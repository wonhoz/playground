### 1. `Speak.Type` — 로컬 Whisper 음성→텍스트 받아쓰기 ⭐ **최강 추천**
**카테고리**: Productivity / AI (트레이 앱)
**수익 잠재력**: ★★★★★

#### 핵심 기능
- **전역 단축키 (Win+Space)** → 녹음 시작 → 떼면 자동 변환 → 현재 필드에 타이핑
- 완전 오프라인: `Whisper.net` (tiny/base 모델) 내장, 인터넷·클라우드 불필요
- 다국어 자동 감지 (한국어·영어·일본어 등 동시 지원)
- 클립보드 모드: 변환된 텍스트를 클립보드에 복사 → 직접 붙여넣기
- 트레이 상주 / 시작프로그램 등록 / 언어 고정 옵션

#### 차별점
- `Echo.Text`는 **TTS (텍스트→음성)** → `Speak.Type`은 **STT (음성→텍스트)** (정반대)
- `Screen.OCR`은 화면 텍스트 추출 → `Speak.Type`은 음성 입력
- OpenAI Whisper 수준의 정확도를 **완전 로컬** 오프라인으로
- 받아쓰기 불가한 필드(원격 데스크톱, 게임 내 채팅 등) 에 `Smart.Paste`와 조합

#### 기술 스택
- WinForms 트레이 / `Whisper.net` (ONNX) / NAudio / Win32 SetWindowsHookEx

### 2. `File.Guard` — 파일 시스템 변경 감사 로거
**카테고리**: System / Files (트레이 앱)
**수익 잠재력**: ★★★★☆

#### 핵심 기능
- 감시 폴더 등록 → 파일 생성·수정·삭제·이름 변경·이동 **실시간 이벤트 로그**
- **어떤 프로세스가 해당 파일을 건드렸나** 표시 (ETW/Process Monitor 방식)
- 스냅샷 모드: "지금 상태 저장" → 나중에 비교 → 변경 파일 목록 Diff 뷰

#### 차별점
- `Watch.Dog` = 프로세스 재시작 → `File.Guard` = **파일 변경 감사 로그**
- `Deep.Diff` = 두 파일 내용 비교 → `File.Guard` = **언제 무엇이 변경됐나 히스토리**

- Angry IP Scanner의 모던 다크 WPF 대체 — 가장 가까운 경쟁자가 오래된 Java 앱

### 1. `Pulse.Run` — 음악 BPM 동기화 무한 달리기 ⭐ **최강 추천**
**장르**: Arcade / Rhythm Runner

#### 게임 플레이
- 플레이어 지정 음악 파일 (MP3/WAV) → 실시간 BPM 분석
- 장애물·발판이 **정확히 비트에 맞춰** 등장·이동
- 빠른 곡 = 장애물 속도 증가, 느린 곡 = 여유롭지만 예측 어려움
- **비트 점프**: 박자에 맞게 점프하면 보너스 거리·무적
- 빌트인 곡 10곡 + 커스텀 음악 지원

#### 핵심 루프
- "이 곡이면 잘 할 수 있을 것 같은데…" → 비트 타이밍 익히기 → 완벽 클리어 → 더 빠른 곡 도전

#### 왜 Neon.Slice·Beat.Drop 팬에게 최적?
- Neon.Slice의 타이밍 아케이드 + Beat.Drop의 리듬 동기화 = 완벽한 하이브리드
- `Shadow.Run`과 다름: Shadow.Run = 고스트 레이스, `Pulse.Run` = 음악 BPM 완전 연동

#### 기술 스택
- WPF Canvas / `NAudio` BPM 실시간 분석 / 이벤트 타임라인 기반 장애물 생성

### 7. `Trace.Run` — 한붓 그리기 교차 없는 경로 퍼즐
**장르**: Puzzle / Casual

#### 게임 플레이
- 화면의 모든 점(노드)을 **단 하나의 선**으로 모두 연결
- **선은 교차 불가**, 각 점은 정확히 한 번씩 통과 (해밀턴 경로 문제)
- 격자 점 → 비격자 불규칙 배치 → 장애물 노드 → 방향 제한 화살표 노드 순으로 난이도 상승
- 힌트 시스템: 다음 이동 경로 한 단계 표시 (힌트 횟수 제한)
- 무한 생성 레벨 (절차적 퍼즐 생성)

#### 핵심 루프
- "이 방향은 막히니까 저 방향으로…" → 막다른 곳 발견 → 역추적 → "아 이쪽이었구나!"

#### 왜 Hue.Flow 팬에게 최적?
- Hue.Flow(경로 연결 퍼즐) + 한붓 그리기 제약 = 한 단계 더 깊은 퍼즐
- `Flow.Pipe`와 다름: Flow.Pipe = 색상 파이프 연결로 빈 칸 채우기, `Trace.Run` = 교차 없는 단일 경로 해밀턴

#### 기술 스택
- WPF Canvas / 절차적 해밀턴 그래프 생성 / 경로 교차 감지

### 9. `Life.Sim` — 세포 생명 게임 퍼즐
**장르**: Puzzle / Simulation / Sandbox

#### 게임 플레이
- Conway's Game of Life 기반: 살아있는 세포 → 인접 2~3개면 생존, 아니면 소멸
- **퍼즐 모드**: 제한된 세포로 초기 배치 → N세대 후 **목표 패턴** 정확히 달성
- **역산 퍼즐**: "N세대 후가 이 패턴이 되려면 초기 배치는?" (역방향 추론)
- **커스텀 규칙** 잠금 해제: HighLife(B36/S23), Day&Night, Maze 등 30+ 규칙 셋
- 샌드박스 모드: 자유 그리기 + 유명 패턴(글라이더, Gosper 총 등) 스탬프 라이브러리

#### 핵심 루프
- 초기 배치 → 시뮬 재생 → 목표와 비교 → "한 칸만 옮기면…" → 시도 → 달성 순간의 쾌감

#### 왜 Sand.Fall 팬에게 최적?
- Sand.Fall의 **관찰하는 재미 (시뮬레이션 진행)** + 퍼즐의 **목표 달성 동기** = 최강 조합

#### 기술 스택
- WPF / 64비트 비트필드 격자 (고속 시뮬) / SIMD 최적화 세대 계산

### 10. `Number.Storm` — 행/열 선택 슬라이드 숫자 융합
**장르**: Puzzle / Casual

#### 게임 플레이
- 격자에 숫자 타일 배치
- **2048과 다른 점**: 전체 슬라이드가 아니라 **선택한 행 or 열만** 이동
- 같은 숫자 두 타일이 만나면 합산 (2+2=4, 4+4=8 등)
- 목표: 지정 숫자 (2048, 4096 등) 달성 or 정해진 수 안에 최고 타일 달성
- **특수 타일**: 폭탄(인접 타일 -1), 잠금(이동 불가), 배수기(×2), 와일드(아무 수와 합산)

#### 핵심 루프
- "이 행을 밀면 저 열이 합산될 텐데…" → 계획 → 연쇄 합산 → 기대 이상의 점수 → 더 큰 수 도전

#### 왜 Hue.Flow·Brick.Blitz 팬에게 최적?
- Hue.Flow처럼 **전략적 선택** + Brick.Blitz처럼 **점수 기록 경쟁** = 두 취향 융합

#### 기술 스택
- WPF / 격자 슬라이드 시뮬레이션 / 애니메이션 이징
