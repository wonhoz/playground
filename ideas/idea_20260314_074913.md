### 1. `Slide.Cast` — Markdown 파일을 오프라인 프레젠테이션으로 변환하는 슬라이드쇼 도구

**카테고리**: Dev/Productivity / WPF
**핵심 기능**: `.md` 파일을 `---` 구분자 기준으로 슬라이드로 파싱하여 전체화면 프레젠테이션을 실행하는 Marp 스타일 오프라인 도구다. 개발자가 별도 파워포인트 없이 마크다운으로 발표 자료를 작성하고 즉시 발표할 수 있다.

**상세 기능**:
- `---` 또는 `<!-- slide -->` 구분자로 슬라이드 분리 파싱
- 실시간 미리보기 + 전체화면 발표 모드 (F5 진입)
- 발표자 뷰 — 현재 슬라이드 + 다음 슬라이드 + 경과 시간 (2번 모니터 지원)
- 코드 블록 구문 강조 (Markdig + Highlight.js 스타일 정적 파싱)
- 배경 이미지, 테마 CSS 커스터마이징 (다크/라이트 내장)
- PDF + PNG(슬라이드별) 내보내기 (PrintDocument / WPF RenderTargetBitmap)
- 파일 변경 시 자동 새로고침 (FileSystemWatcher)
- 슬라이드 축소판 목록 패널 (드래그로 순서 재배치)
- 키보드 단축키: ← → PageUp PageDown 전환, Escape 종료, B 블랙아웃

**기술 스택**:
- WPF (.NET 10), WebView2 (슬라이드 렌더링), Markdig (Markdown 파싱)
- FileSystemWatcher (자동 새로고침), System.Printing (PDF)
- RenderTargetBitmap (PNG 내보내기)

**수익 모델**: 완전 무료 오픈소스
**차별화**: Mark.View는 단순 뷰어, Render.View는 Mermaid/PlantUML 렌더링. Slide.Cast는 전체화면 발표자 모드 + 발표자 뷰 + PDF 내보내기가 핵심.

### 2. `Tape.Delay` — 화면을 지정 초 만큼 지연시켜 보여주는 타임시프트 뷰어

**카테고리**: Screen / Productivity / WPF
**핵심 기능**: 화면(또는 창) 캡처를 5~120초 링 버퍼에 저장해 2번 모니터에 지연된 화면을 실시간으로 표시한다. 스포츠 코칭(자신의 폼 복기), 댄스·악기 연습, 발표 리허설에 활용한다.

**상세 기능**:
- 딜레이 시간 슬라이더 (1~120초, 실시간 조정)
- 캡처 영역 선택: 전체 화면, 특정 모니터, 특정 창
- 링 버퍼에 프레임 저장 (GDI+ BitBlt, 설정 가능한 FPS: 1~30)
- 출력 창: 보더리스 전체화면 또는 플로팅 창으로 선택
- 일시정지/재개 (특정 순간 정지해 자세히 관찰)
- 오버레이 타임스탬프 표시
- 메모리 사용량 최소화 (FPS·해상도 조절)
- 트레이 아이콘 + 전역 단축키 (딜레이 증감, 일시정지)

**기술 스택**:
- WPF (.NET 10), GDI+ (BitBlt 화면 캡처), Win32 API (GetDesktopWindow)
- ConcurrentQueue<Bitmap> 링 버퍼, 별도 캡처 스레드, WPF WriteableBitmap 렌더링

**수익 모델**: 완전 무료
**차별화**: Screen.Recorder는 파일 저장, Time.Lapse는 타임랩스 영상 제작. Tape.Delay는 저장 없이 실시간 지연 표시가 핵심.

### 3. `Page.Watch` — 웹페이지 콘텐츠 변경 감지 트레이 알림 앱

**카테고리**: Productivity / Network / WinForms Tray
**핵심 기능**: 등록한 URL을 주기적으로 폴링하여 HTML 텍스트·DOM 구조 변경을 감지하고 Toast 알림으로 알려주는 트레이 앱. Uptime.Eye는 HTTP 상태코드(200/50x)만 확인하지만 Page.Watch는 페이지 콘텐츠 자체를 비교한다.

**상세 기능**:
- URL 목록 관리 (체크 간격: 1분~24시간 개별 설정)
- 전체 HTML 비교, 또는 CSS Selector로 특정 요소만 감시
- 변경 시 Toast 알림 + 이메일(SMTP) 옵션
- 변경 이력 (이전 vs 현재 diff 뷰)
- 동적 JS 렌더링 옵션 (WebView2 Headless 모드)
- 중요 키워드 등장/소실 감지 (예: "품절" 사라짐)
- 가격 변동 감지 모드 (숫자 추출 + 임계값 설정)
- 변경 무시 목록 (광고·시간표시 영역 제외)

**기술 스택**:
- WinForms Tray (.NET 10), HttpClient + HtmlAgilityPack (HTML 파싱)
- WebView2 (JS 렌더링 옵션), HtmlDiff 라이브러리, SmtpClient (알림)

**수익 모델**: 무료 (URL 5개 제한) / 프리미엄 무제한
**차별화**: Uptime.Eye는 단순 HTTP 상태 확인. Page.Watch는 DOM 콘텐츠 비교, CSS 선택자, 가격 추적까지 지원.

### 6. `WHOIS.Cast` — 도메인 WHOIS·DNS·IP 지오로케이션 통합 조회 GUI

**카테고리**: Network / Dev/Tools / WPF
**핵심 기능**: 도메인 또는 IP 주소 하나 입력으로 WHOIS 등록 정보, 모든 DNS 레코드(A/AAAA/MX/TXT/NS/CNAME/SOA), IP 지오로케이션을 한 화면에 보여주는 통합 도메인 정보 조회 도구.

**상세 기능**:
- WHOIS 파싱 (등록자, 만료일, 등록 일자, 네임서버)
- DNS 레코드 전체 조회 (A/AAAA/CNAME/MX/TXT/SPF/DKIM/DMARC/NS/SOA)
- IP → 지오로케이션 (ip-api.com 오프라인 캐시 또는 MaxMind GeoLite2)
- 도메인 만료 경고 (30일 이내 색상 강조)
- SPF/DMARC/DKIM 이메일 보안 설정 점수 평가
- 이력 탭 (최근 조회 100개 SQLite 저장)
- 즐겨찾기 도메인 목록 + 만료 알림 트레이

**기술 스택**:
- WPF (.NET 10), WhoisNET 또는 직접 TCP 43번 포트 WHOIS 요청
- System.Net.Dns (DNS 조회), GeoLite2 (오프라인 IP → 국가/도시)
- SQLite (이력), Toast 알림 (만료 경고)

**수익 모델**: 완전 무료
**차별화**: DNS.Flip은 프리셋 전환, DNS.Watch는 쿼리 모니터, Net.Trace는 traceroute. WHOIS.Cast는 도메인 종합 정보 센터.

### 8. `OCR.Live` — 화면 특정 영역의 텍스트를 실시간 스트리밍으로 추출하는 도구

**카테고리**: Screen / Productivity / WPF
**핵심 기능**: 화면의 지정 영역을 실시간 캡처해 연속적으로 OCR을 수행, 텍스트가 변할 때마다 추출 결과를 클립보드에 복사하거나 번역 파이프라인으로 전달한다. 자막 없는 영상, 게임 내 텍스트, 회의 화면 실시간 번역에 활용.

**상세 기능**:
- 영역 선택 (드래그로 지정) + 투명 오버레이 표시
- 초당 OCR 횟수 설정 (0.5~5 FPS)
- Tesseract 5 오프라인 (한국어/영어/일본어 tessdata)
- 변경 감지 (이전과 동일한 텍스트는 재처리 안 함)
- 추출 텍스트 → 자동 클립보드 복사 / 플로팅 결과 창
- 번역 파이프라인 연동 (Bergamot 오프라인 MT 또는 DeepL API)
- 이력 로그 (타임스탬프 포함 텍스트 변경 기록)
- 캡처 영역 레이아웃 저장/로드

**기술 스택**:
- WPF (.NET 10), Tesseract5 NuGet (오프라인 OCR), GDI+ BitBlt (캡처)
- Bergamot-translator (오프라인 번역), ConcurrentQueue (프레임 큐)

**수익 모델**: 완전 무료
**차별화**: Screen.OCR은 수동 클릭 스냅샷. Snap.Cast는 1회성 캡처→OCR. OCR.Live는 연속 스트리밍 + 자동 변경 감지 + 번역 파이프라인.

### 10. `Cert.View` — X.509 인증서 심층 파서 & 시각화 도구

**카테고리**: Security / Dev/Tools / WPF
**핵심 기능**: PEM/DER/P12/PFX 인증서 파일을 드래그&드롭하면 모든 확장 필드를 파싱해 사람이 읽기 좋은 형태로 보여주고, 인증서 체인을 트리로 시각화한다. Cert.Watch는 만료 모니터링, Cert.Forge는 생성 도구이지만 Cert.View는 인증서 내용 분석에 특화.

**상세 기능**:
- PEM/DER/P12(PKCS#12)/PFX/CRT/CER 드래그&드롭 파싱
- 발급자/주체/SAN(대체 도메인)/유효기간/알고리즘/공개키 모든 필드 표시
- 확장 필드 시각화 (Key Usage, EKU, CRL Distribution, OCSP)
- 인증서 체인 트리뷰 (Root CA → Intermediate → Leaf)
- 인증서 핀닝 SHA256 해시 자동 생성
- 자체 서명 여부, 와일드카드, CT 로그 포함 여부 표시

- 만료까지 D-Day 강조 + Windows 인증서 저장소에 설치 버튼

**기술 스택**:
- WPF (.NET 10), System.Security.Cryptography.X509Certificates (내장)
- Bouncy Castle NuGet (P12/PKCS#12 파싱 확장)

**수익 모델**: 완전 무료
**차별화**: Cert.Watch=만료 모니터, Cert.Forge=CA 생성. Cert.View는 기존 인증서 내부 구조 완전 분석·시각화.

- 사냥 모드: 방향 규칙을 지키며 도망가는 목표를 포위해 코너에 몰기
- 속도 누적 메커닉: 같은 방향 유지 시 가속, 방향 전환 시 잠시 감속
- 장애물 (벽·기둥): 이동 경로 계획에 영향
- 스테이지 편집기: 커스텀 맵 제작·공유
- 가속 아이템: 일시적으로 제약 없는 자유 이동
- 2P 로컬 대전: 서로 추격/도망 역할

**기술**: WPF Canvas 실시간 게임 루프 (CompositionTarget.Rendering), AABB 충돌 검사, 벡터 각도 수학, A* 추격 AI
**매력**: "45도씩만"이라는 단순 규칙이 만들어내는 예상치 못한 복잡한 이동 패턴. 익숙해질수록 날카로워지는 실력.
**차별화**: Ice.Slide(미끄럼 슬라이딩), Magnet.Jump(자기 발판). Pivot.Chase의 각도-제약 이동은 목록에 없는 완전히 새로운 메커닉.

### 10. `Snap.Duel` — 반응속도 로컬 2인 대전 게임

**장르**: Casual / Skill / 2P Local
**핵심 메커닉**: 화면에 신호가 나타나는 순간 자신의 버튼을 먼저 누르면 이기는 반응속도 대전 게임. Reflex.Tap이 솔로 훈련이라면 Snap.Duel은 옆에 앉은 사람과 즐기는 소셜 대전.

**게임플레이**:
- 화면 중앙 신호 대기: 랜덤 딜레이(1~4초) 후 색상/도형/숫자 표시
- 플레이어1: 키보드 왼쪽 키(예: F), 플레이어2: 오른쪽 키(예: J)
- 먼저 누른 플레이어가 점수 획득
- 함정 신호: 빨간 색 = 누르면 감점 (반응 억제력 테스트)
- 조건부 신호: "3 이상의 숫자" 나타날 때만 누르기 (판단력+반응속도)
- 미니게임 10종 (단순 반응, 짝수 반응, 색 판별, 음감 반응 등)
- 토너먼트 모드: 3~8인이 1:1 대진으로 우승자 결정
- 반응속도 히스토리 그래프 (개인별 ms 추이)

**기술**: WPF 고정밀 타이머 (Stopwatch, <1ms 정밀도), 키보드 입력 (KeyDown 이벤트 최소 레이턴시 최적화), WPF 애니메이션 (신호 등장 효과)
**매력**: 단 한 판이 2~3초, 승패가 즉각 결정되는 짜릿함. "한 판 더"를 부르는 중독성. 파티 게임으로 최적.
**차별화**: Reflex.Tap은 솔로 ms 측정. Snap.Duel은 로컬 2P+ 대전 + 조건부 반응 + 토너먼트 모드.
