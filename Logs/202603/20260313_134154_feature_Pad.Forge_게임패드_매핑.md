# Pad.Forge — 게임패드 매핑 통합 앱 구현

**날짜**: 2026-03-13
**태그**: feature
**상태**: 완료

---

## 개요

`Gamepad.Cast`와 `Pad.Forge`(아이디어) 두 프로젝트를 하나로 통합하여 **Pad.Forge** 단일 애플리케이션으로 구현.
WPF + MVVM 아키텍처, .NET 8.0-windows.

---

## 구현 내용

### 프로젝트 구조
- **위치**: `Applications/System/Manager/Pad.Forge/`
- **기술 스택**: WPF + MVVM, .NET 8.0-windows, HidSharp, Nefarius.ViGEm.Client

### 데이터 모델
- `GamepadButton.cs` — XButtons (XInput 비트플래그), GamepadInput, ControllerType 열거형
- `MappingAction.cs` — ActionType, 키보드/마우스/텍스트/ViGEm/앱 실행 동작
- `ButtonMapping.cs` — 입력↔동작 매핑 단위
- `ControllerProfile.cs` — JSON 저장 프로파일 (데드존·감도·앱 자동 전환 포함)
- `ControllerState.cs` — 실시간 컨트롤러 상태 스냅샷

### 서비스 레이어
| 서비스 | 역할 |
|--------|------|
| `XInputService` | XInput1_4.dll P/Invoke, 최대 4컨트롤러 ~100Hz 폴링, 진동 제어 |
| `HidService` | HidSharp 기반 DS4/DualSense/Switch Pro HID 감지·파싱 |
| `VirtualInputService` | user32.dll SendInput으로 키보드/마우스 가상 입력 전송 |
| `ViGEmService` | Nefarius.ViGEm.Client v1.21.x 가상 Xbox360 컨트롤러 출력 (드라이버 없을 때 graceful fallback) |
| `ControllerService` | XInput+HID 통합 폴링, 매핑 실행 조율 |
| `ProfileService` | JSON 저장/불러오기, 프로파일 관리 |
| `ProcessWatchService` | 포그라운드 프로세스 감지 → 앱별 프로파일 자동 전환 |
| `TrayService` | NotifyIcon 트레이, DarkMenuRenderer, 연결/해제 Toast |

### UI (WPF MVVM)
- **4탭 메인 창**: 컨트롤러 · 매핑 에디터 · 테스트·진동 · 앱 자동 전환
- **다크 테마**: `DarkTheme.xaml` (Button, ComboBox, TabControl, CheckBox, Slider, ScrollBar ControlTemplate)
- **DadVision 시각화**: 스틱 원형 Canvas, D-pad 커스텀 FrameworkElement, 버튼 A/B/X/Y 원 표시
- **진동 테스트**: 슬라이더로 왼쪽/오른쪽 모터 강도 제어
- 다크 타이틀바: `DwmSetWindowAttribute(handle, 20, 1)`

### 기술적 이슈 해결
- WPF + WinForms 동시 활성화로 인한 타입 모호성 (Color, Brush, Binding, Pen, Application) → GlobalUsings alias 방식으로 해결
- ViGEm API v1.21.x 변경: `Xbox360Report` 폐지 → `SetButtonsFull(ushort)` + `SetAxisValue()` + `SetSliderValue()`
- HidSharp Switch Pro 12비트 스틱 파싱, DS4/DualSense D-pad 8방향 인코딩
