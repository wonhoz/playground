## 앱 아이디어

### 1. `Audio.Scope` — 오디오 실시간 스펙트럼 분석기

| 항목 | 내용 |
|------|------|
| **카테고리** | Audio / Dev Tools |
| **플랫폼** | WPF (.NET) |

**핵심 기능**:
- 시스템 오디오 출력·마이크 입력 실시간 캡처 (WASAPI loopback)
- 3가지 뷰 탭: 오실로스코프 (PCM 파형), FFT 스펙트럼 (로그 주파수), 스펙트로그램 (시간×주파수 히트맵)
- 레벨 미터 (L/R 채널, 피크 홀드)
- 주파수 범위·스케일 (선형/로그), 윈도우 함수 (Hann/Hamming/Blackman) 선택
- PNG/SVG 스냅샷 내보내기, 오디오 클립 WAV 녹음

**차별화**: ≠ Pitch.Find (BPM·키 분석기), ≠ Noise.Guard (마이크 노이즈 필터), ≠ Volume.Cast (앱별 볼륨 제어) — 순수 시각화·측정 도구

### 4. `Run.Book` — 인터랙티브 Runbook / 체크리스트 실행기

| 항목 | 내용 |
|------|------|
| **카테고리** | Dev / Productivity |
| **플랫폼** | WPF (.NET) |

- JSON 실행 결과 내보내기

### 5. `Archive.Forge` — ZIP / 7z / TAR 아카이브 생성·관리자

| 항목 | 내용 |
|------|------|
| **카테고리** | Files / Tools |
| **플랫폼** | WPF (.NET) |

**핵심 기능**:
- 드래그&드롭으로 파일/폴더 추가, 포함 파일 트리 미리보기
- 형식 선택: ZIP / 7z / TAR.GZ / TAR.BZ2
- 압축 수준 슬라이더 (빠름 ↔ 최대)

- 제외 패턴: glob 규칙 (`.git`, `node_modules`, `*.tmp` 등)
- 진행률 표시 + 예상 완료 시간

**차별화**: ≠ Zip.Peek (읽기 전용 탐색기), ≠ Crypt.Drop (파일 직접 암호화 금고), ≠ File.Duplicates (중복 탐지)

### 6. `Dice.Shift` — 주사위 방향·면 값 소코반 퍼즐

| 항목 | 내용 |
|------|------|
| **장르** | Puzzle / Logic |
| **플랫폼** | WPF 2D |

**핵심 메커닉**:
- N×N 격자 위 주사위 큐브를 밀어 목표 칸에 도달
- 굴릴 때마다 면이 회전 → 바닥 면 값이 바뀜
- 목표 칸은 "특정 숫자가 바닥 면이어야 통과" 조건 보유
- 방향 잠금 타일, 숫자 변환 타일, 다중 주사위 협동 퍼즐
- 절차적 레벨 생성 + 유일해 검증, 레벨 에디터

**차별화**: ≠ Dungeon.Dice (덱빌딩 로그라이크 — 소코반 없음), ≠ Ice.Slide (얼음 슬라이딩 — 면 값 없음), ≠ Neon.Push (소코반+색상+레이저)

### 9. `Frame.Jump` — 만화 패널을 넘나드는 메타 플랫포머

| 항목 | 내용 |
|------|------|
| **장르** | Platformer / Puzzle / Meta |
| **플랫폼** | WPF 2D |

**핵심 메커닉**:
- 화면이 만화책 레이아웃 — 4~6개 패널이 동시에 보임

**차별화**: ≠ Rift.Jump (2점 포털 순간이동), ≠ Phase.Gate (두 차원 전환), ≠ Chrono.Drop (시간 역행) — 만화 메타 공간 첫 도입

### 10. `Chip.Logic` — 논리 게이트 회로 빌더 교육 퍼즐

| 항목 | 내용 |
|------|------|
| **장르** | Puzzle / Educational / Strategy |
| **플랫폼** | WPF 2D |

**핵심 메커닉**:
- 입력 비트 → 출력 비트 목표 달성을 위한 논리 게이트 배치
- 게이트 종류: NOT, AND, OR, NAND, NOR, XOR, XNOR
- 단계 진행: 반가산기 → 전가산기 → 디코더 → ALU → 간단한 CPU
- 진리표(Truth Table) 실시간 표시 + 각 게이트 빛 흐름 시각화
- 최소 게이트 수 도전 (NAND만으로 XOR 구현 등)
- 자유 샌드박스 모드 + 완성 회로 공유 코드

**차별화**: ≠ Volt.Chain (타이밍 기반 전류 스위치 순서 퍼즐), ≠ Laser.Net (레이저 반사 경로), ≠ 기존 모든 게임: 논리 게이트/디지털 회로 빌더 첫 도입

## TOP 3 추천

### 🥇 1위 — `Chip.Logic` (논리 게이트 회로 빌더)
**추천 이유**:
- 게임 장르 중 완전 미개척 영역 (디지털 논리 회로 교육 게임)
- 난이도 경사가 자연스럽고 무한 확장 가능 (게이트 → 가산기 → CPU)
- 교육적 가치 + 오락적 재미 동시 달성
- WPF 2D로 구현 가능 (노드-엣지 그래프 렌더링)
- 구현 난이도: 중간 (회로 시뮬레이터 로직 + UI 집중)

### 🥈 2위 — `Reg.Vault` (레지스트리 고급 브라우저)
**추천 이유**:
- Windows 개발자·시스템 관리자에게 실용적 필수 도구
- Regedit.exe는 30년째 UI 미개선 → 차별화 기회 명확
- 스냅샷 비교 기능은 앱 설치 전후 변경 추적에 매우 유용
- 구현 범위 명확: WPF + Microsoft.Win32.Registry API

### 🥉 3위 — `Run.Book` (인터랙티브 Runbook 실행기)
**추천 이유**:
- DevOps·SRE 실무에서 즉시 활용 가능
- 인시던트 대응 체크리스트를 표준화하는 수요 큼
- 경쟁 앱: 없음 (로컬 오프라인 GUI runbook runner 공백)
- YAML 파싱 + WPF + 쉘 명령 실행 = 구현 범위 현실적
