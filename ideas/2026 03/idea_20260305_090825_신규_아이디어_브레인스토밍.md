### 1. `PDF.Forge` — 오프라인 PDF 올인원 도구
**카테고리**: Files / Productivity
**수익 잠재력**: ★★★★★

#### 핵심 기능
- **병합**: 여러 PDF 드래그&드롭 → 순서 조정 → 합치기
- **분리**: 페이지 범위 지정 또는 썸네일 선택 → 개별 PDF로 추출
- **페이지 조작**: 회전, 삭제, 복제, 재배치
- **압축**: 이미지 해상도 재샘플링으로 파일 크기 최대 80% 감소
- **워터마크**: 텍스트/이미지 워터마크 반투명 삽입 (위치·각도·폰트 자유)
- **미리보기**: 페이지 썸네일 그리드 뷰

#### 차별점
- 완전 오프라인 (Smallpdf·ILovePDF 대체) — 개인정보 보호
- `PdfSharpCore` 순수 .NET → 외부 런타임 없음
- 드래그&드롭 기반 직관적 UX

#### 기술 스택
- WPF / `PdfSharpCore` (MIT) / `PDFtoImage` (썸네일)

### 3. `SSH.Vault` — SSH 설정 & 키 시각적 관리자
**카테고리**: Dev/Tools
**수익 잠재력**: ★★★★☆

#### 핵심 기능

- SSH config 직접 편집 없이 GUI로 완결
- **터널 시각화**: LocalForward/RemoteForward 포트 포워딩 다이어그램

#### 기술 스택

### 7. `Video.Trim` — 무손실 영상 트리머

**수익 잠재력**: ★★★★☆

#### 차별점
- Lossless cut이 핵심: 용량 제한된 파일 공유 전 빠른 트리밍
- 완전 오프라인, 광고 없음 — LosslessCut의 WPF 대체

### 1. `Fluid.Rush` — SPH 유체 혼합 퍼즐 ⭐ **최강 추천**
**장르**: Puzzle / Simulation
**물리 엔진**: Smoothed Particle Hydrodynamics (SPH) 근사, 격자 분류

#### 게임 플레이
- 컬러 유체들이 가득한 유리병 세트에서 시작
- 파이프·깔때기·밸브를 배치해 유체를 올바른 용기로 이동
- 서로 다른 색 혼합 시 새 색 탄생 (빨강+파랑=보라, 색 이론 기반)
- **목표**: 각 용기를 단일 색으로 가득 채우기
- 시간 제한 + 최소 밸브 수 사용 보너스

#### 물리 요소
- 유체 점성 (물 vs 꿀 vs 오일) → 흐름 속도 차이
- 부력: 밀도 낮은 유체는 위로 뜸
- 표면장력: 방울 맺힘 시각화

#### 중독성 루프
- 색 혼합 결과 예측 → 유레카 순간 → 다음 레벨 기대감
- Sand.Fall 팬이라면 100% 열광할 시각적 만족감

### 2. `Bridge.Craft` — 브리지 빌더
**장르**: Puzzle / Engineering
**물리 엔진**: 응력 시뮬레이션 (절점-부재 FEM 근사)

#### 게임 플레이
- 고정 앵커 사이에 재료를 놓아 다리를 완성
- 차량(자동차, 트럭, 기차)이 통과하면 물리 시뮬레이션 시작
- **재료 종류**: 나무(가볍고 약함) / 철 강봉(강하고 무거움) / 케이블(인장만 지지)
- 부재에 색으로 응력 시각화 (파랑=인장, 빨강=압축, 파손 임박=깜박임)
- 예산 제한으로 최소 비용 최적화 도전

#### 물리 요소
- 절점별 힘의 합산 (정적 분석 + 동적 처짐)
- 재료별 항복 강도 → 초과 시 파손 애니메이션
- 진동 공명 (특정 속도에서 다리 흔들림)

#### 중독성 루프
- 내 다리가 버틸까? → 시뮬레이션 → 쾌감 or 장엄한 붕괴 → 재설계

### 3. `Jelly.Jump` — 소프트바디 젤리 플랫포머 ⭐ **최강 추천**
**장르**: Arcade / Platformer
**물리 엔진**: Shape Matching Soft Body / Spring-Mass

#### 게임 플레이
- 플레이어 = 투명한 젤리 블롭
- 점프 시 찰랑, 착지 시 납작해졌다 복원, 벽에 부딪히면 찌그러짐
- **변형으로 퍼즐 해결**: 좁은 틈은 눌러서 통과, 가시는 피해야 함
- 뱀처럼 늘어나서 레버 조작, 공 모양으로 굴러서 속도 획득
- 각 스테이지 콜렉터블 (크리스탈) 수집 → 젤리 색상 잠금 해제

#### 물리 요소
- 점 집합 (20~40개 질량점) + 스프링 연결 → 실시간 적분
- Shape matching으로 원형 목표 형태 유지 (탄성 계수 조절)
- 충돌 시 형태 변형 후 서서히 복원

#### 중독성 루프
- 물렁물렁한 조작감 자체가 쾌감 → 탐험 욕구 + 수집 동기

### 4. `Marble.Run` — 구슬 롤러코스터 빌더
**장르**: Sandbox / Builder
**물리 엔진**: Verlet Integration + 원기둥 경로 제약

#### 게임 플레이
- 트랙 부품을 조립해 구슬이 굴러가는 경로 설계

- 구슬 출발 → 물리 시뮬레이션 관람 (카메라가 구슬 추적)

### 5. `Fracture.Fall` — Voronoi 파단 파괴 게임
**장르**: Puzzle / Destruction
**물리 엔진**: Voronoi 분열 + 강체 물리

#### 게임 플레이
- 유리, 얼음, 세라믹, 암석 재질 블록 구조물이 스테이지에 배치
- 특정 위치에 공·망치·충격파를 선택해 타격
- **목표**: 목표물(빨간 블록)만 완전히 파괴하고, 보호 대상(파란 블록)은 보존
- 구조물 연쇄 붕괴 경로 예측 → 최소 타격 수 달성

#### 물리 요소
- Voronoi 분열: 타격 지점과 재질 경도에 따라 균열 패턴 다름
- 파편 강체 물리: 무게, 마찰, 바닥 충돌
- 재질별 특성: 유리(날카로운 파편), 얼음(미끄럼), 암석(둔탁한 조각)

#### 중독성 루프
- "이 타격이면 모두 무너지겠지?" → 균열 퍼짐 → 극적 붕괴 순간

### 6. `Zero.G` — 무중력 모멘텀 퍼즐
**장르**: Puzzle / Strategy
**물리 엔진**: 뉴턴 역학, 각운동량 보존

#### 게임 플레이
- 우주 공간 (무중력, 무마찰) 에서 화물 모듈을 도킹 포트로 이동
- 플레이어 = 소형 스러스터 드론 (연료 제한)
- **밀기 = 반작용**: 화물 밀면 드론도 반대로 날아감
- 회전하는 우주 정거장, 자전 중인 소행성 등 동적 목표
- 각운동량 이용: 물체 모서리 타격 → 자전 시작 → 도킹 각도 맞추기

#### 물리 요소
- 완벽한 뉴턴 역학 (F=ma, 모멘텀 보존)
- 회전체 각운동량 (I·ω)
- 스러스터 충격량 계산 (연료=Δv)

#### 중독성 루프
- 우아한 해법 발견 → "한 번 밀기로 전부 해결" 순간의 쾌감

### 7. `Wrecking.Ball` — 추 진자 철거 게임
**장르**: Arcade / Destruction
**물리 엔진**: 구속된 진자 운동 (크레인 암 + 철구 체인)

#### 게임 플레이
- 크레인 암을 좌우 이동 + 체인 길이 조절
- 진자 운동으로 철구를 스윙 → 건물 구조물 타격
- **건물 재질**: 유리(한 번에 붕괴), 벽돌(여러 번), 콘크리트(집중 필요)
- 제한 시간 내 목표 퍼센트 이상 파괴 → 다음 스테이지
- 보너스: 도미노처럼 연쇄 붕괴 유도 → 콤보 점수

#### 물리 요소
- 비선형 진자 방정식 (RK4 수치 적분)
- 철구-건물 충돌 충격량 (질량×속도)
- 건물 구조적 내구도 (벽 지지대 제거 시 상단 붕괴)

#### 중독성 루프
- 스윙 타이밍 맞추기 → 완벽한 히트 → 장엄한 붕괴 시퀀스

### 8. `Tidal.Wave` — 파동 간섭 퍼즐
**장르**: Puzzle / Physics
**물리 엔진**: 2D 파동 방정식 (격자 기반)

#### 게임 플레이
- 수면에 진동원(부이)을 배치 → 동심원 파문 발생
- 파동 간섭: 보강간섭(마루+마루=2배) vs 상쇄간섭(마루+골=0)
- **목표**: 특정 지점(등대, 항구)에 최대 파고 도달 + 보호 구역에는 0파고
- 진동원 주파수·위상·진폭 조절
- 방파제(반사재) 배치로 파동 굴절·반사 조절

#### 물리 요소
- 2D 파동 방정식 finite-difference 풀이 (실시간)
- 반사 경계 조건 (방파제), 흡수 경계 (모래사장)
- 도플러 효과 (이동하는 진동원)

#### 중독성 루프
- 아름다운 간섭 패턴 시각화 → 해법 발견의 과학적 쾌감

### 9. `Spring.Web` — 거미줄 장력 게임
**장르**: Puzzle / Arcade
**물리 엔진**: Verlet Spring-Mass Network

- 빈 무대에서 앵커 포인트(벽·천장·기둥) 사이에 실을 쏜다
- 실은 스프링-질량 사슬 → 중력 처짐 자연 반영
- **목표**: 낙하하는 물체(과일, 보석, 폭탄)를 거미줄로 잡기
- 너무 큰 충격 → 실 끊김 → 물체 낙하 실패
- 실의 장력·굵기(내구도) 선택, 특수 실: 탄성줄, 자석줄, 점착줄

#### 물리 요소
- Verlet 적분 체인 (50~200 노드/실)
- 인장 강도 초과 시 끊김 처리
- 낙하체와 망 충돌 (연속 충돌 감지)

#### 중독성 루프
- 실 네트워크 설계 → 물체 잡기 성공 → 장력 시각화 감상
