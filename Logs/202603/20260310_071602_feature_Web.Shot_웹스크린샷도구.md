## 구현 내용

### 핵심 시스템

| 파일 | 역할 |
|------|------|
| `Services/CaptureService.cs` | WebView2 CDP `Page.captureScreenshot captureBeyondViewport:true` → 전체 페이지 PNG 캡처 |
| `Services/CaptureService.cs` | `CoreWebView2.PrintToPdfAsync()` → 네이티브 PDF 저장 |
| `Services/HistoryService.cs` | JSON 히스토리 (`%AppData%\WebShot\history.json`), 최근 20개 관리 |
| `Services/BatchService.cs` | TXT URL 목록 파일 → 순차 캡처, Progress 이벤트, 취소 지원 |
| `Models/CaptureSettings.cs` | 캡처 설정 (URL, 뷰포트 너비, 딜레이, PDF 여부, 출력 폴더) |
| `Models/HistoryEntry.cs` | 히스토리 항목 (URL, 경로, 타입, 시각, 너비) |
| `MainWindow.xaml/cs` | 전체 UI + 이벤트 처리 |

### 기능 상세

- **전체 페이지 캡처**: CDP `Emulation.setDeviceMetricsOverride`로 너비 설정 → 네비게이션 + 딜레이 대기 → `Page.captureScreenshot captureBeyondViewport:true` → base64 → PNG 파일
- **PDF 저장**: `CoreWebView2.PrintToPdfAsync(path, settings)` 네이티브 방식
- **뷰포트**: 375(모바일) / 768(태블릿) / 1280(데스크톱) / 1440(풀HD) / 커스텀
- **딜레이**: 500ms / 1초 / 1.5초 / 2초 / 3초 / 5초 선택
- **배치 모드**: TXT 파일 (한 줄 = URL, `#` 주석 지원) → 순차 캡처 + 진행 바 + 취소 버튼
- **히스토리 패널**: 최근 20개, 항목 클릭 → 파일 열기 / 개별 삭제 / 전체 삭제
- **저장 경로**: `Documents\WebShot\Captures\yyyyMMdd_HHmmss_hostname.png(pdf)`

### UI/UX

- **1280×800** 기본 창, 리사이즈 가능
- 다크 테마 (배경 `#1A1A2E`, 강조색 `#1EB8E6` 청록, `#1ED8A0` 그린)
- 상단 툴바: 앱 이름 + URL 입력 + 캡처 + 배치 + 폴더열기 + 히스토리 토글
- 옵션 바: 뷰포트 ComboBox + 딜레이 ComboBox + PDF 체크박스 + 미리보기 숨기기
- WebView2 인라인 미리보기 (토글 가능)
- 상태바: 마지막 작업 메시지 + 저장 경로 링크 (클릭 시 파일 열기)
- 다크 타이틀바: `DwmSetWindowAttribute attr=20`
