### 1. `Task.Cast` — 로컬 칸반 보드

### 2. `Bin.View` — EXE/DLL 리소스 브라우저
> **카테고리**: Tools / Dev / System
> **수익 잠재력**: ★★★★☆ (개발자 도구, Resource Hacker 현대화 대체)

**한 줄 요약**
Windows PE 파일(EXE/DLL/OCX)을 드래그하면 내부 리소스를 즉시 트리로 탐색하고 추출하는 개발자 인스펙터.

**핵심 기능**
- 리소스 타입: 아이콘(ICO 묶음) · 비트맵 · 문자열 테이블 · 버전 정보 · 매니페스트(XML) · 다이얼로그 레이아웃 · RCDATA
- 트리뷰: `ICON > Group_1 > 16px / 32px / 48px / 256px` 계층 표시
- 아이콘 미리보기: 각 크기별 렌더링 + 내보내기 (ICO/PNG/각 크기별 PNG)
- 버전 정보 테이블: FileVersion · ProductVersion · Copyright · Company · InternalName
- 매니페스트 뷰어: XML 구문 강조, requireAdministrator / supportedOS 한눈에 확인
- 문자열 테이블: 언어 ID별 정렬, 키워드 검색, CSV 내보내기
- PE 헤더 기본 정보: 32/64비트, 링크 시각, 서브시스템(GUI/Console), 체크섬
- 다중 파일 탭 지원

**기술 스택**
- WPF, net10.0-windows, System.Reflection.PortableExecutable, PEFile (자체 파서), P/Invoke FindResource/LoadResource

**차별점**
- 기존 Resource Hacker: 오래됨(2011년 마지막 주요 업데이트), UI 낡음, .NET 없음
- `Img.Forge` / `Icon.Forge` (이미지 처리·변환기) ≠ Bin.View (PE 내부 리소스 탐색)
- `Zip.Peek` (아카이브 브라우저) ≠ Bin.View (바이너리 구조 파싱)

**수익 비전**
- 개발자·보안 연구자 필수 도구 → GitHub 스타 → 스폰서십
- 고급 기능(리소스 수정·교체, DLL 패치) 프리미엄 플러그인

### 3. `Fade.Out` — 오디오 수면 페이드 타이머
> **카테고리**: Tools / Productivity / Tray
> **수익 잠재력**: ★★★☆☆ (틈새 유틸, 트레이 앱 시장)

**한 줄 요약**
"N분 후에 음악을 서서히 꺼줘" — 시스템 볼륨을 점진적으로 줄여 미디어 재생을 부드럽게 종료하는 트레이 수면 타이머.

**핵심 기능**
- 트레이 아이콘 우클릭 → 타이머 프리셋: 15분 / 30분 / 45분 / 60분 / 커스텀
- 페이드 곡선 선택: 선형 / 지수 감쇠 (자연스러운 잦아듦)
- 타이머 종료 시: ① 볼륨 0으로 → ② 활성 미디어 앱 재생 정지 (PostMessage WM_APPCOMMAND) → ③ 모니터 끄기 옵션
- 트레이 아이콘 색상으로 진행 상황 표시 (초록→주황→빨강)
- 설정 저장: 마지막 사용 타이머 기억
- 수면 중 PC 종료 옵션 (타이머 + N분 후 shutdown /s)

**기술 스택**
- WinForms (트레이), net10.0-windows, CoreAudio (IAudioEndpointVolume), P/Invoke

**차별점**
- `Toast.Cast` (건강 루틴 알림, "쉬어라" 알림) ≠ Fade.Out (실제 음악을 끄는 액션)
- `Sound.Cast` (출력 장치 전환) ≠ Fade.Out (볼륨 페이드)
- `Volume.Cast` (앱별 볼륨 제어) ≠ Fade.Out (시간 기반 자동 종료)
- Windows 기본 수면 타이머는 볼륨 페이드 없음 → 갑자기 꺼지는 불편함 해결

**수익 비전**
- 스트리밍 음악 들으며 잠드는 사용자 광범위 수요
- Mac의 "sleep timer" 앱 iOS 단골 기능 → Windows 공백 채우기
- 무료 배포 + GitHub 스폰서십, 컴팩트하고 완성도 높음

### 4. `Web.Shot` — 전체 페이지 웹스크린샷
> **카테고리**: Tools / Dev / Productivity
> **수익 잠재력**: ★★★★☆ (개발자·QA·마케터, 오프라인 도구 수요)

**한 줄 요약**
URL 입력 또는 로컬 HTML 파일 → WebView2로 완전 렌더링 → 전체 페이지 PNG/PDF 자동 캡처. 스크롤 없이 한 장 완성.

**핵심 기능**
- URL 또는 로컬 HTML 파일 입력
- WebView2 내장 렌더러 → 실제 Chrome 엔진으로 JavaScript 실행, CSS 완전 적용
- 캡처 모드: 뷰포트만 / 전체 페이지 (스크롤 높이 자동 계산)
- 해상도 설정: 뷰포트 너비 (375/768/1280/1440/커스텀) → 반응형 캡처
- 출력: PNG / WebP / PDF (다중 페이지, 인쇄 레이아웃)
- 딜레이 설정: 페이지 로드 완료 후 N초 대기 (lazy-load 이미지 기다리기)
- 배치 모드: URL 목록 파일(TXT) 입력 → 순차 캡처 → 지정 폴더에 자동 저장
- 히스토리: 최근 20개 URL + 캡처 결과 섬네일

**기술 스택**
- WPF, net10.0-windows, Microsoft.Web.WebView2, WriteableBitmap (스크롤 캡처 합성)

**차별점**
- `Screen.Recorder` (비디오 녹화) ≠ Web.Shot (정적 전체 페이지 이미지)
- `Screen.Stitch` (화면 스크롤 캡처, SIFT 특징점 매칭) ≠ Web.Shot (WebView2 직접 렌더링, 정확함)
- `Code.Snap` (코드 스크린샷) ≠ Web.Shot (웹페이지)
- 브라우저 확장 없이 순수 오프라인 도구 → 기업 환경 채택 용이

**수익 비전**
- QA 팀: 반응형 다중 해상도 스크린샷 자동화 → 배치 모드가 핵심 가치
- 마케터: 경쟁사 랜딩 페이지 저장·아카이브
- 프리랜서 개발자: 납품 전 레퍼런스 스크린샷 생성 자동화
- 팀 라이선스, CLI 버전 확장

### 5. `Perm.Audit` — Windows 앱 권한 감사기
> **카테고리**: Tools / System / Security
> **수익 잠재력**: ★★★★☆ (프라이버시 도구 수요 급증, 보안 제품 시장)

**한 줄 요약**
어떤 앱이 내 카메라·마이크·위치·연락처를 몰래 들여다보고 있는지 Windows 권한 DB를 한 화면에서 감사하고 관리.

**핵심 기능**
- 권한 유형별 탭: 카메라 / 마이크 / 위치 / 알림 / 캘린더 / 연락처 / 사진 라이브러리 / 백그라운드 앱
- 앱별 권한 상태: 허용 / 거부 / 요청 안 함 — 한 줄씩 표시
- 원클릭 허용/거부 전환 (Windows Privacy 레지스트리 직접 수정)
- 수상한 앱 플래그: 일반 유틸리티가 마이크·카메라 권한 보유 시 ⚠️ 경고
- 권한 변경 감사 로그: 언제 어떤 앱이 권한 획득/취소했는지 기록
- 스냅샷 비교: 설치 전후 권한 변화 diff
- 일괄 조치: "마이크 접근 허용 앱 전체 거부" 원클릭

**기술 스택**
- WPF, net10.0-windows, Windows.Security.Authorization.AppCapabilityAccess (WinRT), Microsoft.Win32 Registry

**차별점**
- `Spy.Guard` (실시간 클립보드·마이크·카메라 무단 접근 감지+차단) ≠ Perm.Audit (권한 DB 감사·관리)
- `App.Cage` (네트워크 방화벽 GUI) ≠ Perm.Audit (OS 권한 레이어)
- Windows 설정 앱은 앱별 드릴다운 UI라 전체 현황 파악 불가 → Perm.Audit은 권한 유형별 전체 목록

**수익 비전**
- "내 카메라를 몰래 켜는 앱이 있다?" 공포 마케팅 → 바이럴 잠재력 높음
- GDPR 컴플라이언스 감사 도구로 기업 판매
- 정기 감사 리포트 PDF 내보내기 → 프리미엄 기능

### 6. `Img.Meta` — 이미지 메타데이터 뷰어·편집기
> **카테고리**: Tools / Files / Media
> **수익 잠재력**: ★★★★☆ (사진작가·프라이버시 보호 도구, ExifTool GUI 대체)

**한 줄 요약**
사진 속에 숨겨진 GPS 위치·카메라 설정·편집 이력을 한눈에 보고, 수정하거나 일괄 제거하는 EXIF/IPTC/XMP 인스펙터.

**핵심 기능**
- 드래그&드롭 또는 폴더 로드 → 파일별 메타데이터 트리뷰 (EXIF·IPTC·XMP·ICC Profile 탭)
- 지도 연동: GPS 좌표 → OpenStreetMap 내장 뷰 (WebView2 + Leaflet, 오프라인 타일 캐시)
- 주요 필드 즉석 편집: 촬영 날짜 / 작가 / 저작권 / 설명 / 키워드 태그
- 프라이버시 모드: GPS · 카메라 모델 · 시리얼번호 · 편집 소프트웨어 선택 제거
- 일괄 처리: 폴더 내 JPG/PNG/HEIC 전체 → 특정 필드 일괄 수정 또는 제거
- 전후 비교: 수정 전 원본 값 vs 수정 후 값 diff 테이블
- 리포트: 선택 폴더 전체 메타데이터 요약 (카메라 기종별 통계 / 위치 분포 지도)

**기술 스택**
- WPF, net10.0-windows, MetadataExtractor 라이브러리, ExifLibrary.NET, WebView2

**차별점**
- `Photo.Video.Organizer` (EXIF 날짜로 파일 정리) ≠ Img.Meta (메타데이터 자체를 보고 편집)
- `Img.Forge` (리사이즈·필터·워터마크) ≠ Img.Meta (픽셀 처리 없음, 메타만)
- `Photo.Squash` (압축) ≠ Img.Meta
- ExifTool은 CLI → GUI 대체 수요 확실

**수익 비전**
- GDPR 발효 이후 "사진 공유 전 GPS 제거" 수요 폭발적
- 사진작가: 저작권 메타 일괄 삽입 / 고객사 납품 전 불필요 데이터 제거
- 소셜 미디어 운영자: 위치 노출 방지 워크플로우 필수 도구
- 법인 라이선스 판매 가능 (보안 정책 필수 도구화)

### 7. `Crypt.Drop` — 드래그&드롭 파일 암호화 금고
> **카테고리**: Tools / Security / Files
> **수익 잠재력**: ★★★★★ (프라이버시·보안 제품 시장, 기업 배포)

**한 줄 요약**
파일이나 폴더를 드래그하면 AES-256-GCM으로 암호화된 단일 `.vault` 컨테이너로 묶고, 비밀번호 입력으로 즉시 복호화하는 초간단 암호화 금고.

**핵심 기능**
- 드래그&드롭 암호화: 파일/폴더 → `.vault` 단일 암호화 파일 (내부 ZIP → AES-256-GCM)
- 키 파생: Argon2id (메모리 256MB, 3회 반복) — 브루트포스 내성
- 비밀번호 + 키 파일 이중 인증 옵션 (키 파일은 USB에 저장)
- 복호화: `.vault` 더블클릭 → 비밀번호 팝업 → 임시 폴더에 압축 해제 → 파일 열기 → 닫으면 자동 재암호화
- 마스크 미리보기: 복호화 없이 내부 파일 목록만 확인
- 보안 삭제: 원본 파일 DoD 7회 덮어쓰기 후 삭제 옵션
- 볼륨 크기 제한 없음, 폴더 구조 보존

**기술 스택**

- "내가 그린 것이 왜 이렇게 예쁘지?" → 창작 동기 강화 → 공유 유도

**왜 수익이 나는가**

- 교육: 대칭·기하 학습 도구 → 학교 B2B
- 완성 작품 프린트 서비스 연동 (마그넷·엽서·포스터)

### 7. `Tower.Fall` — 역 타워디펜스
> **카테고리**: Games / Strategy / Action
> **수익 잠재력**: ★★★★★ (역발상 게임플레이, 전략 게임 시장)

**핵심 게임플레이**
- 기존 타워디펜스와 **역할이 반전**: 플레이어가 방어자(타워)가 아닌 **침략자** 시점
- 맵에는 자동 배치된 타워·바리케이드·지뢰밭이 있음
- 플레이어는 다양한 유닛을 코스트 내에서 구매·배치:
  - 기본 보병(저비용, 약함)
  - 기갑 차량(비용 높음, 맷집 강함)
  - 자살 드론(저비용, 자폭으로 타워 파괴)
  - 연막 부대(타워 시야 차단)
  - 해커(타워 일시 무력화)
- 목표: 우측 끝 기지 점령 / 시간 내 통과 유닛 수 목표 달성
- 8맵 × 3난이도 + 무한 웨이브 생존 모드

**기술 스택**
- WPF Canvas, A* 경로 탐색 (유닛 이동), DispatcherTimer 틱 시스템, 타워 사거리 원 충돌

**차별점**
- `Tower.Guard` (타워 배치 방어 전략) ≠ Tower.Fall (침략자 시점, 완전히 다른 사고방식)
- `Dungeon.Dash` (실시간 액션 RPG) ≠ Tower.Fall (전략 배치)
- `Hex.Storm` (헥사 격자 마이크로 RTS) ≠ Tower.Fall
- "타워 디펜스를 뚫는 방법"을 배우는 독특한 교육적 재미

**왜 수익이 나는가**
- 타워 디펜스 팬층 대규모 → 뒤집힌 규칙이 화제성 보장
- 스팀 얼리 액세스 가능 아이디어
- 레벨 에디터: 방어 배치 설계 → 다른 플레이어가 공격 → 비대칭 멀티플레이

### 8. `Loop.Race` — 자기교차 트랙 레이싱
> **카테고리**: Games / Racing / Arcade
> **수익 잠재력**: ★★★★☆ (스피드런 커뮤니티, 독특한 ghost 메커닉)

**핵심 게임플레이**
- 프로시저럴 생성 레이스 트랙 (자기 자신과 교차하는 클로버·피규어8·나선 형태)
- 기본 플레이: 트랙 위 자동차를 조향해 완주 (최단 시간)
- **핵심 트위스트**: 첫 번째 랩의 이동 궤적이 **다음 랩의 물리적 장애물**로 남음
  - 1랩: 아무것도 없음
  - 2랩: 1랩 궤적이 벽이 됨
  - 3랩: 1·2랩 궤적이 모두 벽이 됨
  - → 랩을 거듭할수록 코스가 점점 복잡해져 완주 불가 가까워짐
- 스코어: 총 완주 랩 수 × 시간 보너스
- 일일 시드: 전 세계 동일 트랙, 랩 수 랭킹
- 레코드 리플레이: 최고 기록 재생

**기술 스택**
- WPF Canvas, Bezier 커브 트랙 생성, 선분-선분 충돌 감지, Verlet 차량 물리 (간략화)

**차별점**
- `Nitro.Drift` (드리프트 레이싱) ≠ Loop.Race (자기교차 트랙 + 궤적 장애물)
- `Shadow.Run` (고스트 레이스 무한 달리기) ≠ Loop.Race (궤적이 장애물이 되는 능동적 구조)
- Tron 자전거와 유사해 보이지만 레이싱 게임 컨텍스트에서 완전히 다른 활용

**왜 수익이 나는가**
- 스피드런 커뮤니티: 일일 시드 랭킹 → 매일 방문 유도
- "내가 방금 만든 장애물에 내가 막히는" 아이러니 → GIF 공유 바이럴
- 레벨 팩 (클로버 트랙 50개, 나선 트랙 50개 등) DLC
