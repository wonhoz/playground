## 개요

완전 오프라인 영어 사전 팝업 앱.
Win+Shift+D 전역 단축키로 즉시 호출, 브라우저 전환 없이 단어 뜻·유의어·반의어·예문 조회.

**경로**: `Applications/Tools/Productivity/Utility/Dict.Cast/`
**출력**: `bin/Applications/Tools/Productivity/Utility/Dict.Cast.exe`

---

## 구현 내용

### 사전 데이터 (WordNet 3.1)
- 최초 실행 시 Princeton 대학 서버에서 WordNet 3.1 데이터 다운로드 (~12MB)
- `data.noun`, `data.verb`, `data.adj`, `data.adv` 파싱 → SQLite DB 빌드 (~35MB)
- 이후 완전 오프라인 동작, 155,000+ 단어 수록

### 핵심 기능
- **전역 단축키** Win+Shift+D — 팝업 토글 (창이 활성 상태면 숨김, 아니면 표시)
- **클립보드 자동 읽기** — 단축키 호출 시 클립보드에 영단어 있으면 자동으로 검색창에 채움
- **타이핑 중 제안** — 200ms 디바운스 후 prefix 매칭 최대 8단어 드롭다운
- **결과 표시** — 품사별(n./v./adj./adv.) 색상 뱃지 + 번호 정의 + 예문 + 유의어 + 반의어
- **발음 듣기** — SAPI TTS `🔊` 버튼 원클릭
- **단어장** — `★/☆` 버튼 토글 저장, SQLite 영속
- **히스토리 바** — 하단에 최근 8단어 필 버튼 (클릭 시 즉시 검색)
- **내보내기** — CSV / Anki 덱(`.txt`) 저장
- **다크 테마** + DWM 다크 타이틀바

### 파일 구성

| 파일 | 역할 |
|------|------|
| `Services/WordNetBuilder.cs` | WordNet 3.1 다운로드·파싱·SQLite 빌드 |
| `Services/DictionaryService.cs` | dict.db 조회 (Lookup, Suggest) |
| `Services/AppDatabase.cs` | 히스토리·단어장 user.db 관리 |
| `Services/TtsService.cs` | SAPI SpeechSynthesizer 래퍼 |
| `ViewModels/MainViewModel.cs` | INPC, 검색·단어장·히스토리 로직 |
| `ViewModels/SenseViewModel.cs` | WordSense 표시용 뷰모델 (포맷팅) |
| `MainWindow.xaml` | 다크 UI — 검색바·결과·제안·히스토리바 |
| `MainWindow.xaml.cs` | 전역 단축키·WndProc·이벤트 핸들러 |

### DB 스키마 (dict.db — WordNet)
```sql
synsets: id, key(pos:offset), pos, definition, examples(JSON), synonyms(JSON), antonyms(JSON)
entries: id, word, synset_id
INDEX: entries(word)
```

### DB 스키마 (user.db — 사용자 데이터)
```sql
history:  id, word, looked_at
wordlist: id, word, added_at
```

## 트러블슈팅

### "car" 검색 결과 없음 버그 (커밋 `38b95af`)

**원인 1**: `DownloadUrl`이 Princeton 공식 URL(`https://wordnetcode.princeton.edu/3.1/WNdb-3.1.zip`) → HTTP 404 반환
**원인 2**: `ExtractDict`가 `entry.FullName.StartsWith("dict/")` 하드코딩 → NLTK zip은 `wordnet/data.noun` 구조라 매칭 실패 → 파일 0개 추출

**수정**:
- URL → NLTK GitHub 미러 (`https://github.com/nltk/nltk_data/raw/gh-pages/packages/corpora/wordnet.zip`, ~10.7MB)
- `ExtractDict`: 경로 prefix 방식 → 파일명 집합(`wanted`) 매칭 방식으로 교체
- `dictDir` 존재 여부 대신 `data.noun` 파일 존재 여부로 재추출 판단 (빈 디렉토리 재사용 방지)
- 진단 스크립트(`check-zip/state/url.ps1`) 삭제
