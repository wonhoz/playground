### 1. `DNS.Trace` — DNS 해석 경로 시각화 도구

**카테고리**: Dev/Network / Windows 데스크탑
**핵심 기능**: 도메인 이름 하나를 입력하면 Root → TLD → Authoritative DNS 서버까지 전체 해석 경로를 단계별로 추적하고 레이턴시와 응답 패킷을 시각화한다. DNS.Flip(전환), DNS.Watch(쿼리 모니터), WHOIS.Cast(도메인 조회)와는 달리 "어떻게 IP가 결정되는가"의 과정 자체를 가르쳐 주는 도구다.

**상세 기능**:
- Root NS → TLD NS → 권한 NS 단계별 UDP/TCP 쿼리 직접 전송 (recursive resolver 우회)
- 각 단계 레이턴시(ms), TTL, 응답 레코드(A/AAAA/CNAME/NS) 컬럼 표시
- Mermaid 스타일 수직 플로차트 SVG 내보내기 (발표 자료 활용)
- DNSSEC 서명 검증 상태(validated / unsigned / failed) 배지 표시
- 다중 DNS 해석기 비교 (기본 시스템, 8.8.8.8, 1.1.1.1, DoH 엔드포인트)
- 응답 패킷 HEX + 구조 파싱 패널 (섹션별 레코드 트리)
- 오프라인 캐시 뷰어 (Windows DNS 클라이언트 캐시 `ipconfig /displaydns` 파싱)

**기술 스택**:
- WPF .NET 10, DnsClient.NET (UDP/TCP 직접 쿼리)
- SkiaSharp (플로차트 렌더링), Svg.Skia (SVG 내보내기)
- System.Net.Sockets (RAW DNS 패킷 수신)

**수익 모델**: 무료 오픈소스
**차별화**: DNS.Flip은 전환 도구, WHOIS.Cast는 레코드 조회 — DNS.Trace는 해석 과정의 "여정" 시각화에 집중. 교육·디버깅 목적

### 2. `Font.Kern` — 폰트 커닝 쌍 시각화 + 편집기

**카테고리**: Dev/Typography / Windows 데스크탑
**핵심 기능**: 설치된 폰트의 커닝 쌍(kern pairs)을 추출·시각화하고, 특정 글자 조합의 간격(pair kern, class kern)을 커스텀 조정하여 CSV/OTF Feature Code로 내보낸다. Font.Scout(미리보기), Font.Probe(화면 추출), Font.Draw(픽셀 편집), Font.Ramp(스케일)와 완전히 다른 타이포그래피 정밀 도구다.

**상세 기능**:
- TrueType GPOS/kern 테이블 파싱 → 전체 커닝 쌍 목록 추출
- 글자 쌍 입력 시 실시간 간격 렌더링 (확대 배율 조절)
- 커닝 값 수동 조정 → 커스텀 오버라이드 저장 (프로젝트 파일)
- 문단 입력 시 설정된 커닝 적용 미리보기 모드
- 문제 있는 쌍 자동 탐지 (kerning 0인데 시각적으로 어색한 조합 ML 제안)
- 커닝 데이터 CSV 내보내기 / Adobe OpenType Feature Code (kern feature) 내보내기
- 영문·한글·CJK 커닝 쌍 별도 필터 탭

**기술 스택**:
- WPF .NET 10, SharpFont (FreeType 바인딩) 또는 Typography.OpenFont
- SkiaSharp (글리프 정밀 렌더링), System.Drawing (GDI+ 폰트 폴백)
- ML.NET (어색한 쌍 제안 경량 모델)

**수익 모델**: 무료 / 폰트 디자이너 대상 프리미엄 플러그인 확장
**차별화**: 기존 Font.Scout/Probe는 폰트 탐색·추출 도구. Font.Kern은 커닝 테이블 자체를 분석·수정하는 타이포그래피 정밀 도구

### 5. `SSH.Forge` — SSH 키 생성·관리·교체 전문가 도구

**카테고리**: Dev/Security / Windows 데스크탑
**핵심 기능**: Ed25519, RSA, ECDSA SSH 키 페어를 생성·관리하고 지문 확인, 공개키 복사, 키 교체 워크플로를 지원하는 GUI 도구. SSH.Vault(~/.ssh/config 편집기)와 달리 키 자체의 생성·검사·보관에 집중한다.

**상세 기능**:
- 키 알고리즘·비트 수 선택 생성 (Ed25519, RSA 2048/4096, ECDSA P-256/P-384)
- 키 파일 드래그&드롭 → SHA-256/MD5 지문 + 랜덤아트 출력
- 공개키 one-click 클립보드 복사 (OpenSSH / PEM / authorized_keys 형식 전환)
- 패스프레이즈 추가·제거·변경 (passphrase 없이 암호화)
- 키 만료 관리 (생성일·마지막 사용일·메모 태그 SQLite 저장)
- `authorized_keys` 파일 뷰어·편집기 (원격 서버 키 목록)
- 키 교체 체크리스트 알림 (N개월마다 Toast 알림)

**기술 스택**:
- WPF .NET 10, Renci.SshNet (키 생성·파싱), BouncyCastle
- SQLite + Dapper (키 메타데이터 저장)

**수익 모델**: 무료 오픈소스
**차별화**: SSH.Vault는 ~/.ssh/config 연결 관리. SSH.Forge는 키 페어 자체의 생성·검증·교체 수명주기에 집중

### 6. `Log.Tail` — 실시간 멀티파일 로그 테일러

**카테고리**: Dev/Monitor / Windows 데스크탑
**핵심 기능**: 여러 로그 파일을 동시에 `tail -f` 방식으로 모니터링하며, 각 파일에 독립적인 필터·하이라이트 규칙을 적용하고 패턴 매칭 시 Toast 알림을 보내는 개발·운영 도구. Log.Lens(분석 뷰어), Log.Merge(시간순 통합), Log.Struct(JSON 구조 로그 ★29차)와 다른 실시간 감시에 특화된 도구다.

**상세 기능**:
- 탭 별로 다른 파일 추적 (파일 순환 감지, 로그 로테이션 자동 재연결)
- 라인별 규칙: 정규식 매칭 → 배경색·볼드·아이콘 하이라이트
- 알림 트리거: 특정 패턴 감지 시 Toast + 소리 알림
- 전체 탭 통합 검색 (정규식 지원, 매칭 라인만 필터)
- 고성능 테일 (Memory-Mapped File + FileSystemWatcher, 100MB+ 파일 대응)
- 라인 타임스탬프 파싱 자동 감지 (ISO, syslog, IIS, .NET 등 주요 포맷)
- 세션 저장/복원 (탭 파일 목록 + 규칙 프리셋)

**기술 스택**:
- WPF .NET 10, AvalonEdit (고성능 텍스트 뷰어)
- FileSystemWatcher + MemoryMappedFile (효율적 테일)
- Windows.UI.Notifications (Toast), NAudio (알림음)

**수익 모델**: 무료 오픈소스
**차별화**: Log.Struct는 JSON 구조 로그 스트리머. Log.Tail은 형식 무관 범용 실시간 감시 + 멀티파일 규칙 엔진에 집중

### 7. `Run.Trace` — .NET EventPipe 런타임 콜 트레이서

**카테고리**: Dev/Profiler / Windows 데스크탑
**핵심 기능**: 실행 중인 .NET 프로세스에 EventPipe를 연결하여 메서드 호출 트리를 실시간 또는 사후 분석으로 시각화한다. 전체 디버거 없이 특정 기능이 어떤 경로로 호출되는지 추적한다. Perf.Flame(CPU 샘플링 플레임 그래프 ★30차)과 달리 콜 트리 경로 추적에 집중한다.

**상세 기능**:
- .NET DiagnosticsClient API로 실행 중 프로세스 목록 탐색·연결
- EventPipe 기반 Method Enter/Exit 이벤트 수집 (nettrace 형식)
- 호출 트리 뷰: 타임라인 + 인덴트 트리 두 가지 모드
- 네임스페이스·클래스·메서드 필터링 (불필요한 시스템 콜 제외)
- 메서드별 누적 시간·호출 횟수 통계 테이블
- nettrace 파일 저장·열기 (오프라인 분석)
- 특정 메서드 브레이크포인트: 처음 호출 시 일시 중단 및 스택 스냅샷

**기술 스택**:
- WPF .NET 10, Microsoft.Diagnostics.Runtime (ClrMD)
- Microsoft.Diagnostics.NETCore.Client (EventPipe)
- SkiaSharp (타임라인 렌더링)

**수익 모델**: 무료 오픈소스
**차별화**: Perf.Flame은 CPU 시간 기반 샘플링 프로파일러. Run.Trace는 메서드 호출 경로 자체를 추적하는 계측 트레이서 — 서로 보완적인 도구

### 8. `Num.Parse` — 다국어 숫자 형식 자동 판별기

**카테고리**: Dev/Inspector / Windows 데스크탑
**핵심 기능**: `1,234.56`, `1.234,56`, `1 234,56`, `١٢٣٤` 등 다국어 숫자 문자열을 붙여넣으면 로케일·포맷·진법을 자동 판별하고 정규화된 값을 추출한다. Locale.View(로케일별 형식 렌더링 브라우저), Quick.Calc(계산기)와 완전히 다른 "입력 판별" 도구다.

**상세 기능**:
- 1,000단위 구분자 / 소수점 기호 자동 감지 (`, . · ' 공백` 조합)
- 아라비아 / 페르시아 / 데바나가리 숫자 등 Unicode 숫자 자동 변환
- 지수 표기(1.23e4), 공학 표기(1.23M/K), 분수(3/4) 판별
- 판별 결과: "후보 로케일 목록 + 신뢰도 %" 표시
- 클립보드 실시간 감시 모드 — 숫자가 복사될 때마다 자동 판별 팝업
- 반대 방향: 원하는 로케일 선택 → 숫자를 해당 형식으로 포맷
- 트레이 팝업 모드 (전역 단축키 Win+Shift+N)

**기술 스택**:
- WPF .NET 10 (트레이 팝업), 커스텀 파서 알고리즘
- ICU4N (유니코드 숫자 변환), System.Globalization (로케일 DB)

**수익 모델**: 무료 오픈소스
**차별화**: Locale.View는 로케일별 형식을 보여주는 브라우저. Num.Parse는 미지의 숫자 문자열의 형식을 역추론하는 탐정 도구

### 9. `Exe.Sign` — Windows 실행 파일 서명·매니페스트 인스펙터

**카테고리**: Dev/Security / Windows 데스크탑
**핵심 기능**: EXE/DLL 파일을 드래그&드롭하면 Authenticode 디지털 서명 체인, 버전 정보, UAC 매니페스트 요구 권한, 가져오기 DLL 목록을 한눈에 보여주는 보안·배포 QA 도구. Bin.View(PE 리소스 브라우저), Hex.Peek(헥스 에디터)과 달리 보안·신원·권한 측면에 집중한다.

**상세 기능**:
- Authenticode 서명 검증: 서명자 체인 트리, 유효기간, 타임스탬프 서버
- 서명 없음 / 만료 / 부정 서명 즉시 경고 배지
- PE 버전 리소스 파싱 (Company, Product, FileVersion, InternalName 등 모든 필드)
- UAC requestedExecutionLevel: asInvoker / requireAdministrator / highestAvailable 감지
- 가져오기 테이블: 의존 DLL + 함수 목록 (알려진 위험 API 하이라이트)
- 파일 드래그&드롭 및 Windows 컨텍스트 메뉴 "서명 검사" 통합
- 결과 JSON/HTML 내보내기 (보안 감사 리포트용)

**기술 스택**:
- WPF .NET 10, System.Security.Cryptography.Pkcs (Authenticode)
- PeNet (PE 파일 파싱), Win32 API (VerQueryValue)

**수익 모델**: 무료 오픈소스
**차별화**: Bin.View는 리소스(아이콘·문자열·다이얼로그) 브라우저. Exe.Sign은 신원(서명)·권한(UAC)·의존성 보안 분석 전문 도구

### 10. `Map.Route` — 오프라인 GPX 경로 플래너 + 거리 계산기

**상세 기능**:
- 오프라인 지도 타일 (MBTiles 또는 PMTiles 번들 세계 지도)

- 지도 클릭으로 웨이포인트 추가 → 자동 선분 연결 거리 누적
- 고도 프로필 차트 (트랙 포함 GPX 기준, SRTM 내장 데이터 대체)
- 통계: 총 거리(km/mi), 최고/최저 고도, 소요 시간 추정 (속도 입력)

**기술 스택**:

- SharpGPX 또는 커스텀 GPX XML 파싱
- Haversine/Vincenty 거리 공식, SRTM30 DEM 번들

**수익 모델**: 무료 / 고해상도 지역 타일 팩 유료
**차별화**: Data.Map은 GeoJSON/TopoJSON 데이터 시각화. Map.Route는 GPX 기반 등산·사이클·러닝 경로 설계·분석 전문 도구

**기술**: 웨이브 스케줄러, 포탑 AI FSM, 해킹 타이머 시스템, 사이버펑크 파티클 이펙트
**매력**: 믿었던 아군 포탑이 배신하는 긴장감 — 방어선이 흔들리는 순간의 아드레날린
**차별화**: Tower.Guard(✅) 일반 TD. Tower.Fall(backlog) 역 TD(침략자 시점). Neon.Siege는 정방향 TD이지만 아군 통제권 상실이라는 3번째 요소가 추가

### 9. `Find.Cast` — 절차 생성 숨은그림찾기

**장르**: Casual / Puzzle / Observation
**핵심 메커닉**: 수십~수백 개의 오브젝트가 복잡하게 겹쳐진 장면에서 목록에 있는 아이템을 찾아 클릭하는 숨은그림찾기 게임. 장면이 절차적으로 생성되어 무한한 변형이 가능하다. Spot.Diff(★34차 틀린그림찾기)와 완전히 다른 "탐색" 메커닉이다.

**게임플레이**:
- 레벨 시작: 복잡한 장면(시장·정글·우주·서재 등) + 찾을 아이템 목록 5~15개
- 오브젝트: 100~300개 PNG 에셋을 무작위 배치·회전·크기 변환·레이어 정렬
- 타이머: 시간 내 전체 발견 시 별 3개 (찾지 못할수록 별 감소)
- 힌트 시스템: 30초 쿨다운 힌트 버튼 → 대상 방향 화살표 2초 표시
- 특수 모드: 이동하는 아이템(장면 내 움직임), 색상 일치 찾기
- 시즌 도전: 매일 새 장면 · 전세계 플레이어 클리어 시간 리더보드
- 커스텀 모드: 사용자 이미지 업로드 → 자동으로 오브젝트 추출 후 장면 생성

**기술**: 오브젝트 레이어 합성 (SkiaSharp), 클릭 히트 테스트 (픽셀 퍼펙트 / AABB 2단계)
**매력**: "어디있지…?" 발견 순간의 클릭 만족감 — 나이·성별 불문 접근성
**차별화**: Spot.Diff(★34차)는 두 이미지의 차이 찾기. Find.Cast는 단일 복잡 장면에서 목록 아이템 탐색 — 인지 메커닉이 완전히 다름

### 10. `Roll.Craft` — 구면 볼 롤링 미로 퍼즐

**장르**: Arcade / Puzzle / Physics
**핵심 메커닉**: 3D를 2D 탑다운으로 표현한 볼 롤링 미로 게임. 공을 기울이는 대신 공이 직접 WASD로 구른다. 자기 타일, 경사면, 아이스 타일, 바람 팬 등 다양한 특수 타일이 공의 물리를 변형한다. Tilt.Ball(backlog, 판 기울이기)과 달리 공을 직접 조종한다.

**게임플레이**:
- WASD로 공에 가속도 적용 (직접 이동 — 판 기울이기 아님)
- 특수 타일: 얼음(마찰=0 미끄러짐), 자석(인력·척력), 경사(중력 벡터 변환), 팬(일방향 가속)
- 목표: 탈출구까지 이동 + 코인 N개 수집
- 공 변형 아이템: 자성 공(자석 타일 역이용), 무거운 공(관성 증가), 고무 공(반발 계수 증가)
- 경사 타일 세트: 45° 꺾임으로 수직·수평 미로 전환
- 60스테이지 (5×5~12×12 격자, 난이도별 분류)
- 레벨 에디터: 타일 팔레트, 공 시작점, 코인·함정 배치 → 파일 공유

**기술**: 뉴턴 운동방정식 (가속도·마찰·탄성), 타일별 물리 효과 컴포넌트, SkiaSharp
**매력**: 관성과 특수 타일이 조합될수록 예상치 못한 경로 — "물리가 만든 반전"
**차별화**: Tilt.Ball(backlog)은 WASD로 플랫폼을 기울여 공을 간접 유도. Roll.Craft는 공을 직접 조종하며 타일 물리 조합에 집중 — 조종 메커닉과 퍼즐 설계 방향이 다름
