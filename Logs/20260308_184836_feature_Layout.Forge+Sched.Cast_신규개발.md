## 작업 내용

### Layout.Forge — 키보드 키 재배치 프로파일 에디터

- **경로**: `Applications/Tools/Dev/System/Layout.Forge`
- **기능**
  - ANSI 104키 키보드 UI (메인 블록 + 네비게이션 블록)
  - 키 클릭 후 드롭다운으로 재배치 대상 선택 (비활성화 포함)
  - Windows 레지스트리 `Scancode Map` 직접 읽기/쓰기 (`HKLM\SYSTEM\CurrentControlSet\Control\Keyboard Layout`)
  - 프로파일 JSON 저장/불러오기 (`%LocalAppData%\Layout.Forge\Profiles`)
  - 재배치된 키 강조 표시 (배지 텍스트, 색상 구분)
- **기술 스택**: WPF, net8.0-windows, UAC requireAdministrator, 레지스트리 COM-free 직접 접근
- **아이콘**: 키보드 모양 + 파란 강조 키

### Sched.Cast — Windows 작업 스케줄러 GUI 대체

- **경로**: `Applications/Tools/Dev/System/Sched.Cast`
- **기능**
  - SchedCast 전용 폴더(`\SchedCast`) 작업 목록 조회
  - 작업 실행/중지/삭제/활성화 전환
  - 신규 작업 등록 (한 번/매일/매주/로그온/시작 트리거 지원)
  - UAC 관리자 권한 필요 (Task Scheduler 쓰기)
- **기술 스택**: WPF, net8.0-windows, COM dynamic + schtasks.exe 하이브리드 (NuGet 없음)
  - `GetTasks()`: COM `Schedule.Service` dynamic으로 작업 읽기
  - 제어 명령: `schtasks.exe` `/run`, `/end`, `/delete`, `/change`
  - 등록: `BuildTaskXml()` → `schtasks /create /xml`
- **아이콘**: 달력 + 시계 + 체크마크

## 수정된 주요 버그

| 버그 | 원인 | 해결 |
|------|------|------|
| `TaskState` 컴파일 오류 | NuGet 제거 후 `SchedTaskState`로 이름 변경되었으나 TaskViewModel 미반영 | `TaskState` → `SchedTaskState` 교체 |
| `Marshal.GetActiveObject` 오류 | .NET 8에서 해당 API 제거 | 단순 `Activator.CreateInstance(t)` 방식으로 교체 |
| NuGet `Microsoft.Win32.TaskScheduler` 버전 불일치 | nuget.org에 2.2.0.3만 존재, 네임스페이스 불일치 | NuGet 완전 제거 후 COM dynamic + schtasks.exe 하이브리드로 재작성 |
