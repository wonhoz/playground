좋은 방향으로 개발하고 있네. 오히려 지금 구성을 보니까 **"지표 기반 탐색기"** 수준이 아니라 **실시간 시그널 엔진(Event Detection Engine)** 에 가까워.

현재 사용 중인

* ✅ Time Frame Window
* ✅ 볼린저 밴드(BB)
* ✅ RSI
* ✅ 거래량
* ✅ 이동평균선(GC/DC)

이 정도면 일반 개인투자자가 사용하는 수준은 이미 넘어섰어.

하지만 실제 단타 고수(주도주 스캘핑, 데이트레이딩, 세력주 추적)들이 많이 보는 것들은 생각보다 조금 다르다.

내가 추천하는 우선순위대로 정리해볼게.

---

# ★★★★★ (거의 필수)

## 1. VWAP (Volume Weighted Average Price)

이건 정말 추천한다.

미국에서는 거의 모든 기관이 보고

국내 단타 고수들도 많이 본다.

```
VWAP 위
→ 기관 평균 매수가보다 위

VWAP 아래
→ 기관 평균 매수가보다 아래
```

### 활용

바닥 반등

```
BB 하단 터치

↓

RSI 25

↓

VWAP 돌파

↓

거래량 증가

↓

매수
```

또는

```
VWAP 지지

↓

재상승
```

이게 엄청 많이 나온다.

---

## 2. OBV (On Balance Volume)

거래량 누적지표.

가격보다 선행하는 경우가 많다.

예)

```
가격

→ 횡보

OBV

→ 계속 상승
```

그러면

```
세력이 계속 모으는 중
```

인 경우가 많다.

---

## 3. MFI (Money Flow Index)

RSI + 거래량 합친 버전.

RSI보다 신뢰도가 조금 높다.

```
RSI

↓

가격만 사용

MFI

↓

가격 + 거래량
```

특히

```
MFI <20

↓

반등
```

잘 나온다.

---

## 4. ATR

Average True Range

변동성 측정.

예)

ATR이 갑자기 증가하면

```
추세 시작
```

가능성이 높다.

---

## 5. ADX

추세 강도.

많은 사람이 안 쓰는데

엄청 좋다.

```
ADX <20

횡보

ADX >25

추세 시작

ADX >40

강한 추세
```

골든크로스가 나와도

ADX가 낮으면

거짓 신호가 엄청 많다.

## 추가로 추천하는 '이벤트 기반 점수 시스템'

현재는 "조건 충족 시 알림" 방식이라면, 한 단계 발전시켜 **가중치 기반 스코어링**을 추천해.

예를 들어:

* BB 하단 터치: +15
* RSI < 30: +15
* MFI < 20: +10
* 거래량 Z-score > 3: +20
* RVOL > 2.0: +15
* VWAP 상향 돌파: +20
* EMA20 기울기 양전환: +10
* ADX > 25: +10
* RSI 다이버전스: +20

총점이 **70점 이상이면 "강한 매수 후보"**, **90점 이상이면 "최우선 알림"**처럼 분류하면 단순 조건식보다 노이즈를 크게 줄일 수 있다.

특히 너처럼 **KIS 실시간 분봉**을 받아 직접 계산하는 구조라면, 이런 **이벤트 기반 스코어 엔진 + 상태 머신(바닥 탐지 → 초기 반등 → 추세 확인 → 추격 구간)** 구조가 실제 단타 환경에서 확장성과 성능이 매우 좋다. 이렇게 설계하면 단순한 지표 알림이 아니라 "매매 타이밍을 단계별로 평가하는 엔진"으로 발전시킬 수 있다.
