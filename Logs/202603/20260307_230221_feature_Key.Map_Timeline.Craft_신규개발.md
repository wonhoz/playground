## Key.Map

**경로**: `Applications/Tools/Dev/Assets/Key.Map/`
**출력**: `bin/Applications/Tools/Dev/Assets/Key.Map.exe`

### 구현 내용
- ANSI 104-key 레이아웃 WPF Canvas 렌더링 (U=46px 그리드)
- 앱별 프리셋: VSCode, IntelliJ, Chrome, Windows, Vim (각 15~22개 단축키)
- 카테고리별 색상 구분: File(파랑), Edit(주황), View(보라), Run(녹색), Navigate(시안), Other(회색)
- 커스텀 단축키 추가/삭제, 단축키 입력 시 실시간 키보드 하이라이팅
- PNG 저장 (`RenderTargetBitmap`) / 인쇄 (`PrintDialog.PrintVisual` + `ScaleTransform`)
- 다크 테마 + DWM 타이틀바 + 멀티사이즈 아이콘 (256/48/32/16px)

### 주요 파일
| 파일 | 역할 |
|------|------|
| `Services/LayoutLibrary.cs` | ANSI 104-key 레이아웃 정의, `ParseShortcut()` |
| `Services/PresetLibrary.cs` | 5개 앱 프리셋 데이터 |
| `ViewModels/MainViewModel.cs` | INPC, 프리셋/단축키 CRUD |
| `MainWindow.xaml.cs` | Canvas 구축, 하이라이팅, PNG/인쇄 |

### 트러블슈팅
- `System.Windows.Printing` 네임스페이스 없음(ReachFramework 필요) → `PrintDialog.PrintVisual()` 방식으로 대체

## Timeline.Craft

**경로**: `Applications/Tools/Productivity/Visual/Timeline.Craft/`
**출력**: `bin/Applications/Tools/Productivity/Visual/Timeline.Craft.exe`

### 구현 내용
- 멀티 레인 WPF Canvas 타임라인 커스텀 컨트롤 (`TimelineView`)
- 드래그로 이벤트 생성(rubber-band) / 이동 / 리사이즈 (오른쪽 6px 핸들)
- 마일스톤 마커 (다이아몬드 형태)
- Ctrl+휠 줌 인/아웃 (2~120px/day)
- JSON 프로젝트 저장/불러오기 (`System.Text.Json`)
- PNG 내보내기 (`RenderTargetBitmap`)
- 이벤트 속성 편집 패널 (제목/시작일/종료일/색상/마일스톤/메모)
- 다크 테마 + DWM 타이틀바 + 멀티사이즈 아이콘 (256/48/32/16px)

### 주요 파일
| 파일 | 역할 |
|------|------|
| `Controls/TimelineView.cs` | Canvas 렌더링, 마우스 드래그 인터랙션 |
| `Models/TimelineEvent.cs` | INPC, 이벤트 데이터 |
| `Models/TimelineLane.cs` | 레인 정의 |
| `Models/TimelineProject.cs` | 프로젝트 직렬화 단위 |
| `Services/ProjectSerializer.cs` | JSON 저장/불러오기 |
| `ViewModels/MainViewModel.cs` | 줌/이벤트 CRUD, `RebuildRequested` 이벤트 |

### 트러블슈팅
- `Window.Resources` 중복 선언 → `BooleanToVisibilityConverter`를 첫 번째 블록에 통합
