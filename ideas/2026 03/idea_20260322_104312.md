### 1. `Barcode.Forge` — 1D/2D 바코드 올인원 생성기

**상세 기능**:
- 심볼로지 선택: Code-128·Code-39·EAN-13·EAN-8·UPC-A·UPC-E·ITF-14·QR·DataMatrix·PDF417·Aztec (ZXing.Net 활용)

- SkiaSharp (SVG/고해상도 PNG 렌더링)

**수익 모델**: 무료 오프라인 단독 실행
**차별화**: QR.Forge는 QR 코드 전용; Barcode.Forge는 QR을 포함한 20+ 산업 표준 심볼로지를 통합 지원. 배치 CSV 생성 + SVG 내보내기로 상업용 레이블 인쇄 워크플로우 완성.

### 2. `URL.Forge` — URL 빌더/파서/인스펙터 GUI

**카테고리**: Dev/Tools / 플랫폼: WinForms/WPF
**핵심 기능**: URL을 시각적으로 분해·편집·재조립하는 개발자 도구. URL의 각 구성요소(scheme, host, path, query params, fragment)를 독립적으로 편집하고, URL 인코딩·디코딩과 cURL/fetch 코드 스니펫 자동 생성 기능을 제공한다.

**상세 기능**:
- URL 파서: 붙여넣으면 즉시 scheme/userinfo/host/port/path/query/fragment 분해 표시
- 쿼리 파라미터 GUI 편집기: 키-값 테이블로 param 추가/삭제/편집 → URL 자동 재조립
- URL 인코딩/디코딩: percent-encoding, punycode 도메인, base64 URL-safe
- cURL/fetch/axios/HttpClient 코드 스니펫 자동 생성 (클립보드 복사)
- URL 정규화: trailing slash, 중복 슬래시, 대소문자 정규화
- 단축 URL 분석: 리디렉션 체인 추적 (최대 10 hop)
- 비교 모드: 두 URL 나란히 비교, 다른 구성요소 강조
- 히스토리: 최근 50개 URL 기록

**기술 스택**:
- System.Uri / UriBuilder (.NET 내장)
- HttpClient (리디렉션 추적)
- WPF DataGrid (쿼리 파라미터 편집)
- AvalonEdit 또는 TextBox (코드 스니펫 표시)

**수익 모델**: 무료 오프라인 단독 실행
**차별화**: Api.Probe는 HTTP 요청 전송 도구; URL.Forge는 URL 자체를 해부·재조립하는 전문 분석기. 쿼리 파라미터 GUI 편집과 코드 스니펫 생성이 핵심 차별점.

### 3. `Event.Count` — 멀티 이벤트 D-day 카운트다운 트레이

**카테고리**: Productivity / Tray / 플랫폼: WinForms 트레이
**핵심 기능**: 여러 개의 중요한 날짜(마감일, 기념일, 출시일, 시험 등)를 동시에 추적하는 D-day 카운트다운 트레이 앱. 트레이 아이콘 툴팁에 가장 임박한 이벤트를, 클릭 팝업에 전체 목록을 표시한다.

**상세 기능**:
- 이벤트 등록: 이름, 날짜, 색상 태그, 반복 여부(매년/매월)
- 트레이 아이콘 동적 변경: 가장 임박한 이벤트의 남은 일수 숫자를 아이콘에 직접 렌더링
- 툴팁: 상위 3개 이벤트의 D-day 수 표시
- 팝업 창: 전체 이벤트 목록, D-day 오름차순 정렬, 진행 바 표시
- 알림: D-30, D-7, D-3, D-1, D-day 당일 Windows Toast 알림
- 지난 이벤트: 경과 일수(+N일) 자동 표시, 아카이브 이동
- 내보내기: iCal(.ics) 형식으로 캘린더 앱에 등록 가능
- 다크 테마, 트레이 우클릭 메뉴로 빠른 이벤트 추가

**기술 스택**:
- WinForms NotifyIcon + ContextMenuStrip
- DwmSetWindowAttribute 다크 타이틀바
- Ical.Net (iCal 내보내기)
- System.Drawing (트레이 아이콘 동적 렌더링)
- JSON 단일 파일 로컬 저장

**수익 모델**: 무료 오프라인 단독 실행
**차별화**: Rush.Clock은 단일 출발 타이머; Sched.Cast는 Windows 작업 스케줄러 래퍼. Event.Count는 *여러 개의 중요한 날짜*를 동시에 추적하며 트레이 아이콘에 D-day 숫자를 직접 그린다는 점이 독특하다.

### 4. `Color.Name` — 색상 이름 사전

**카테고리**: Design/Tools / 플랫폼: WPF
**핵심 기능**: 어떤 색상이든 헥스/RGB/HSL로 입력하거나 화면에서 피킹하면, CSS·X11·Pantone·RAL·NCS·자연어 색상명 등 3만+ DB에서 가장 가까운 이름을 즉시 찾아준다. "이 색이 뭐라고 불리는지 모를 때" 를 위한 색상 이름 사전.

**상세 기능**:
- 입력: HEX/RGB/HSL/HSV/CMYK 모든 형식 지원, 화면 픽킹 (DWM 돋보기)
- 최근접 색상명 검색: ΔE2000 색차 기반, 상위 10개 후보 표시
- DB 카테고리 선택: CSS Named Colors (148개), X11 (256개), Pantone PMS (2,000+), RAL (200+), NCS (1,900+), 일반 한국어 색상명, 영어 자연어 색상명(XKCD 900개)
- 색상명 → 헥스 역방향 검색 ("남색" → #000080 또는 가장 가까운 매칭)
- 팔레트 비교: 선택한 색상과 동일 계열 색상명 그룹 탐색
- 내보내기: 색상 이름+HEX 텍스트, PNG 스와치
- 즐겨찾기: 자주 쓰는 색상-이름 쌍 저장

**기술 스택**:
- ColorMine 또는 직접 구현 ΔE2000 계산
- 내장 색상명 DB (JSON 파일, 3만+ 항목)
- Windows Desktop Duplication API (화면 픽킹)
- WPF ColorPicker 커스텀 컨트롤

**수익 모델**: 무료 오프라인 단독 실행
**차별화**: Color.Drop은 단순 화면 색상 피커; Color.Space는 색공간 수학 실험실; Palette.Cast는 팔레트 수집기. Color.Name은 색상의 *이름*을 찾는 역방향 사전이라는 점에서 완전히 다른 용도.

**수익 모델**: 무료 오프라인 단독 실행 (API 호출은 공개 레지스트리)

### 8. `NTP.Watch` — 시스템 시계 NTP 동기화 진단기

**카테고리**: System / Network / 플랫폼: WinForms 트레이
**핵심 기능**: Windows 시스템 시계의 NTP 동기화 상태를 실시간 모니터링한다. 시간 드리프트(오차), NTP 서버 응답 지연, 마지막 동기화 시각, 시계 정확도 추이 그래프를 제공해 시간 민감한 앱(인증서, JWT, OTP)의 문제를 조기에 진단한다.

**상세 기능**:
- 실시간 드리프트 측정: 시스템 시계 vs NTP 서버 응답 오프셋 (ms 정밀도)
- NTP 서버 목록: 현재 설정 서버 표시, 커스텀 서버 추가, 응답 지연 측정
- 히스토리 그래프: 24시간 드리프트 추이 SparkLine
- 트레이 아이콘: 오차 범위 색상 표시 (초록/주황/빨강)
- 알림: 드리프트 임계값 초과 시 Toast (예: 500ms 이상)
- 강제 동기화 트리거: w32tm /resync 원클릭 실행
- 동기화 이력: 마지막 N회 동기화 시각·오프셋·서버 기록
- 방화벽 체크: UDP 123 포트 접근 가능 여부 진단

**기술 스택**:
- System.Net.Sockets (NTP 패킷 직접 파싱, UDP 123)
- w32tm.exe 프로세스 래핑
- OxyPlot (드리프트 히스토리 그래프)
- WinForms NotifyIcon 트레이

**수익 모델**: 무료 오프라인 단독 실행
**차별화**: Zone.Watch는 세계시계; Boot.Map은 부팅 성능 타임라인. NTP.Watch는 시스템 시계 자체의 *정확도와 동기화 상태*를 진단하는 전문 도구로, 개발자·시스템 관리자 대상.

**기술**: 백트래킹 해법 탐색 (깊이 제한 DFS), SkiaSharp 타일 렌더링 (그림자·3D 스태킹 효과), 타일 접근 가능성 판정 알고리즘
**매력**: 느긋하고 명상적인 타일 제거 경험. 전통 미술 스타일의 타일이 시각적으로 아름다우며, 한 번 완성하면 다시 다른 패턴을 도전하고 싶어진다.
**차별화**: 기존 게임 목록에 없는 클래식 장르. Pixel.Cross (노노그램), Peg.Solo (말뚝 솔리테어), Card.Solo (트럼프 솔리테어)와 나란히 캐주얼 솔리테어 컬렉션을 구성.

### 7. `Grid.Paint` — 페인트 바이 넘버

**장르**: Casual / Creative / Relaxing
**핵심 메커닉**: 격자에 번호가 써진 칸을 해당 번호의 색으로 칠하면 숨겨진 이미지가 드러나는 디지털 페인트-바이-넘버. 사용자가 직접 이미지를 업로드해 퍼즐로 변환할 수도 있다.

**게임플레이**:
- 내장 퍼즐 컬렉션: 풍경·동물·인물·건축 100종 (20×20~60×60)
- 사용자 이미지 → 퍼즐 변환: k-Means 색상 군집화로 N색 번호 자동 할당
- 색상 팔레트 표시: 현재 선택한 번호의 색상, 미칠한 칸 수 카운터
- 칠하기: 클릭·드래그·동일 번호 전체 채우기(플러드 필)
- 확대/축소: 마우스 휠 줌, 미니맵
- 자동 채우기: 선택한 번호의 모든 칸 즉시 채우기
- 완성도 미리보기: 50% 진행 시 해당 영역 컬러 힌트
- 완성 애니메이션: 반짝임 + 완성된 그림 팝업
- 저장: 진행 상태 자동 저장, PNG 내보내기

**기술**: k-Means 색상 군집화 (이미지 팔레트 추출), WriteableBitmap (실시간 페인팅), SkiaSharp (결과 이미지 렌더링), SQLite (진행 상태 저장)
**매력**: 칸을 하나씩 칠하며 서서히 이미지가 나타나는 긴장감과 완성 순간의 감동. 자기 사진을 퍼즐로 변환하는 기능이 독특한 재미를 추가.
**차별화**: Pixel.Cross (노노그램: 줄 숫자 힌트로 채우기)와 달리 *번호=색상 직접 대응*으로 논리보다 집중·색칠 경험에 집중. 아트적 결과물이 남는다는 차별점.

### 8. `Sky.Scraper` — 스카이스크래퍼 논리 퍼즐

**장르**: Puzzle / Logic (공간 추론)
**핵심 메커닉**: N×N 격자에 1~N 높이의 건물을 배치하되, 행·열에 같은 높이 중복 금지(스도쿠 규칙) + 각 방향 외곽 숫자는 "해당 방향에서 보이는 건물 수"를 나타내는 이중 제약 논리 퍼즐. 공간적 직관과 논리가 결합된 독창적 퍼즐.

**게임플레이**:
- 격자 크기: 4×4(입문)~7×7(전문가)
- 입력: 높이 숫자 또는 터치 드래그, 연필 메모 기능
- 실시간 검증: 행/열 중복, 시야 불일치 즉시 표시
- 힌트 3종: 하나의 확정 셀 공개, 특정 행/열 가능값 필터링, 시야 제약 논리 설명 ("이 방향에서 1이 보이면 첫 칸이 N")
- 자동 생성: 유일해 보장 퍼즐 (백트래킹+제약 전파)
- 솔버: 단계별 논리 추론 과정 보여주기
- 튜토리얼: 인터랙티브 6단계 시야 규칙 설명
- 데일리 퍼즐: 날짜별 5×5 챌린지

**기술**: Constraint Propagation + Arc Consistency (AC-3) 솔버/생성기, WPF 격자 UI (외곽 숫자 힌트 렌더링), 유일해 검증 알고리즘
**매력**: "4방향 시야"라는 제약이 처음엔 낯설지만 익숙해지면 스도쿠보다 훨씬 우아한 추론의 쾌감을 준다. 전 세계 논리 퍼즐 커뮤니티에서 사랑받는 숨겨진 명작 장르.
**차별화**: Sudoku.Cast (스도쿠), Kakuro.Cast (합산 십자말)와 같은 논리 퍼즐 군이지만 *3D 공간 시야 제약*이라는 완전히 다른 메커닉. Logic.Zebra (아인슈타인 소거)와도 구별됨.

### 9. `Pirate.Map` — 보물 지도 추론 퍼즐

**장르**: Puzzle / Logic / Adventure
**핵심 메커닉**: 격자 지도 위에서 방향 화살표·거리 단서·"N칸 떨어짐" 힌트들을 논리적으로 조합해 숨겨진 보물의 위치를 추론하는 공간 논리 퍼즐. 다양한 단서 유형의 교집합으로 유일한 위치를 특정하는 과정이 핵심.

**게임플레이**:
- 격자 크기: 5×5~10×10 지도
- 단서 유형: 방향(북/남/동/서 방향 있음), 거리("3칸 이내"), 행/열 번호("3행"), 대각선 관계, 비교("A보다 북쪽"), NOT 단서("이 구역엔 없음")
- 마킹: 가능/불가능 격자에 체크·X 마킹
- 진행 상황: 활성 단서 하이라이트, 현재 불가 구역 음영
- 힌트: 사용되지 않은 단서 중 가장 유용한 단서 강조
- 130+ 내장 퍼즐: 해적/고대 유물/별자리 테마
- 자동 생성: 유일해 보장 퍼즐 (단서 수·유형 파라미터 조절)
- 스토리 모드: 10개 섬 연속 보물 찾기, 힌트를 NPC 대사로 제공

**기술**: Constraint Propagation 솔버, 유일해 검증 백트래킹, WPF 격자 인터랙티브 마킹 UI, SVG 테마 배경 지도
**매력**: 추리 소설처럼 단서가 하나씩 맞아 들어가며 범위가 좁혀지는 논리의 쾌감. 모험 테마가 퍼즐에 내러티브적 흥미를 더함.
**차별화**: Logic.Zebra (아인슈타인 표 소거), Ice.Slide (슬라이딩 블록)와 달리 *공간 방향+거리 단서 조합*이라는 독특한 추론 메커닉. 보물 찾기 스토리텔링 프레임.

### 10. `Fleet.Strike` — 배틀십 전함 전략 게임

**장르**: Strategy / Puzzle / 2P / Classic
**핵심 메커닉**: 그리드에 함선을 배치하고 상대 해역에 포격하여 모든 함선을 먼저 격침시키는 클래식 Battleship. AI 전략(Hunt/Target 알고리즘, Probability Density Map)을 시각화하여 전략 학습 도구로도 활용 가능.

**게임플레이**:
- 표준 10×10 격자, 함선 5종: 항공모함(5)·전함(4)·순양함(3)·잠수함(3)·구축함(2)
- 함선 배치: 드래그 배치, 회전, 무작위 배치
- 싱글플레이: Easy·Medium·Hard·Elite AI (확률 밀도 맵 기반)
- AI 시각화 모드: AI가 매 턴 어떤 확률 계산으로 좌표를 선택하는지 히트맵으로 표시
- 로컬 2P: 화면 전환 방식 (준비 단계에서 시선 차단)
- 특수 능력: 폭탄(3×3 영역), 레이더(5×5 영역 실루엣), 어뢰(행 전체) - 제한 횟수
- 전적 통계: 승률, 평균 포격 수, 가장 빠른 승리
- 고전 모드: 특수 능력 없는 순수 클래식 룰

**기술**: Monte Carlo Probability Density Map AI (함선 가능 배치 전체 열거→확률 맵), WPF 그리드 애니메이션 (명중/미스 이펙트), 2P 화면 전환 BlurEffect
**매력**: 정보 비대칭 속에서 확률적 사고로 상대를 추적하는 심리전. AI의 확률 맵을 보며 전략을 배울 수 있는 교육적 재미도 있음.
**차별화**: 기존 게임 목록에 없는 클래식 2인 해전 전략 게임. Hex.Storm (헥사 마이크로 RTS), Tower.Guard (타워 디펜스)와 다른 *정보 비대칭 추론 전략* 장르.
